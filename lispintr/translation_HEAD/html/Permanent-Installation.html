3.5 Install Code Permanently

関数を評価することによってインストールされた関数定義は、あなたがEmacsを閉じるまでインストールされたまま残ります。次回に新たなEmacsセッションを始めた際には、その関数定義を再度評価するまで関数はインストールされないでしょう。

いずれあなたは新たにEmacsセッションを開始する都度、コードを自動的にインストールしたいと思うようになるかもしれません。これを行うにはいくつかの手段があります:

  • あなただけのためのコードがあるなら、あなたの初期化ファイル.emacsにその関数定義のコードを記述することができる。Emacs開始時に自動的に.emacsファイルが評価されて、その中のすべての関数定義が自動的に評価される。
  • インストールしたい関数定義を1つ以上の関数用のファイルに記述して、load関数を使いファイル内の関数それぞれをインストールするようEmacsに評価させることもできる。Loading Filesを参照のこと。
  • あなたのサイト全体が使うであろうコードがあるなら、通常はEmacsのビルド時にロードされるsite-init.elと呼ばれるファイルに記述する。これによってあなたのマシンを使う誰もがそのコードを利用可能になる(Emacsディストリビューションに含まれるINSTALLを参照)。

最後に、Emacsを使う誰もが使いたいと思うかもしれないコードがあるなら、それをコンピューターネットワークに投稿したり、フリーソフトウェア財団(FSS: Free Software Foundation)にコピーを送付することができます(あなたがこれを行う際にはそのコードとコードのドキュメントにたいして実行、コピー、学習、変更、そしてそのコードの再配布を他の人が行える、あなたの作業が盗まれることを防ぐライセンスを付与してください)。コードのコピーをフリーソフトウェア財団に送付するなら、あなたと他の人たちが適切に保護されていれば、そのコードはEmacsの次回リリースに含まれることになるかもしれません。過去においてEmacsがどのように成長してきたかは、このような寄贈によるところが大きいのです。
