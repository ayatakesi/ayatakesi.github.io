074: @node Introduction
075: @unnumbered Introduction
076: 
077: This manual contains detailed information about various features that
078: are too specialized to be included in the printed Emacs manual.  It is
079: intended to be readable by anyone having a basic knowledge of Emacs.
080: However, certain sections may be intended for a more specialized
081: audience, such as Elisp authors.  This should be clearly pointed out
082: at the beginning of these sections.
083: 
084: @c Note to authors - you need to be careful about cross-references to
085: @c topics in the Emacs manual.  As a printed document, the xtra files
086: @c are separate from the Emacs manual; but as an info document, they
087: @c are part of the Emacs manual.  Hence you need to use something like:
088: @c @iftex
089: @c @ref{Comparing Files,,, emacs, the Emacs Manual},
090: @c @end iftex
091: @c @ifnottex
092: @c @ref{Comparing Files},
093: @c @end ifnottex
094: 
095: Certain packages (or collections of related features) have their own
096: manuals, separate from the main Emacs manual.  This manual is
097: intended as a complement, rather than an alternative, to reading those
098: additional manuals.  In a nutshell, it is a collection of smaller
099: specialized features (or extra detail about standard features), too
100: small or too obscure to justify their own manual, or inclusion in the
101: printed Emacs manual.  The chapters in this manual are, however,
102: included (at the relevant places) in the main Emacs manual when it is
103: formatted as an Info document.
104: 
105: Sections intended specifically for Elisp programmers can follow the
106: style of the Elisp manual.  Other sections should follow the style of
107: the Emacs manual.
