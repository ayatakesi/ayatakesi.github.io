0006: @appendix Command Line Arguments for Emacs Invocation
0007: @cindex command line arguments
0008: @cindex arguments (command line)
0009: @cindex options (command line)
0010: @cindex switches (command line)
0011: @cindex startup (command line arguments)
0012: @cindex invocation (command line arguments)
0013: @c FIXME: Document '--smid'?  --xfq
0014: 
0015:   Emacs supports command line arguments to request various actions
0016: when invoking Emacs.  These are for compatibility with other editors
0017: and for sophisticated activities.  We don't recommend using them for
0018: ordinary editing (@xref{Emacs Server}, for a way to access an existing
0019: Emacs job from the command line).
0020: 
0021:   Arguments starting with @samp{-} are @dfn{options}, and so is
0022: @samp{+@var{linenum}}.  All other arguments specify files to visit.
0023: Emacs visits the specified files while it starts up.  The last file
0024: specified on the command line becomes the current buffer; the other
0025: files are also visited in other buffers.  As with most programs, the
0026: special argument @samp{--} says that all subsequent arguments are file
0027: names, not options, even if they start with @samp{-}.
0028: 
0029:   Emacs command options can specify many things, such as the size and
0030: position of the X window Emacs uses, its colors, and so on.  A few
0031: options support advanced usage, such as running Lisp functions on files
0032: in batch mode.  The sections of this chapter describe the available
0033: options, arranged according to their purpose.
0034: 
0035:   There are two ways of writing options: the short forms that start with
0036: a single @samp{-}, and the long forms that start with @samp{--}.  For
0037: example, @samp{-d} is a short form and @samp{--display} is the
0038: corresponding long form.
0039: 
0040:   The long forms with @samp{--} are easier to remember, but longer to
0041: type.  However, you don't have to spell out the whole option name; any
0042: unambiguous abbreviation is enough.  When a long option takes an
0043: argument, you can use either a space or an equal sign to separate the
0044: option name and the argument.  Thus, for the option @samp{--display},
0045: you can write either @samp{--display sugar-bombs:0.0} or
0046: @samp{--display=sugar-bombs:0.0}.  We recommend an equal sign because
0047: it makes the relationship clearer, and the tables below always show an
0048: equal sign.
0049: 
0050: @cindex initial options (command line)
0051: @cindex action options (command line)
0052: @vindex command-line-args
0053:   Most options specify how to initialize Emacs, or set parameters for
0054: the Emacs session.  We call them @dfn{initial options}.  A few options
0055: specify things to do, such as loading libraries or calling Lisp
0056: functions.  These are called @dfn{action options}.  These and file
0057: names together are called @dfn{action arguments}.  The action
0058: arguments are stored as a list of strings in the variable
0059: @code{command-line-args}.  (Actually, when Emacs starts up,
0060: @code{command-line-args} contains all the arguments passed from the
0061: command line; during initialization, the initial arguments are removed
0062: from this list when they are processed, leaving only the action
0063: arguments.)

0190: @node Initial Options
0191: @appendixsec Initial Options
0192: 
0193:   The initial options specify parameters for the Emacs session.  This
0194: section describes the more general initial options; some other options
0195: specifically related to the X Window System appear in the following
0196: sections.
0197: 
0198:   Some initial options affect the loading of the initialization file.
0199: Normally, Emacs first loads @file{site-start.el} if it exists, then
0200: your own initialization file if it exists, and finally the default
0201: initialization file @file{default.el} if it exists (@pxref{Init
0202: File}).  Certain options prevent loading of some of these files or
0203: substitute other files for them.
0204:

0207: @opindex -chdir
0208: @itemx --chdir=@var{directory}
0209: @opindex --chdir
0210: @cindex change Emacs directory
0211: Change to @var{directory} before doing anything else.  This is mainly used
0212: by session management in X so that Emacs starts in the same directory as it
0213: stopped.  This makes desktop saving and restoring easier.
0214: 
0215: @item -t @var{device}
0216: @opindex -t
0217: @itemx --terminal=@var{device}
0218: @opindex --terminal
0219: @cindex device for Emacs terminal I/O
0220: Use @var{device} as the device for terminal input and output.  This
0221: option implies @samp{--no-window-system}.
0222: 
0223: @item -d @var{display}
0224: @opindex -d
0225: @itemx --display=@var{display}
0226: @opindex --display
0227: @cindex display for Emacs frame
0228: Use the X Window System and use the display named @var{display} to open
0229: the initial Emacs frame.  @xref{Display X}, for more details.
0230: 
0231: @item -nw
0232: @opindex -nw
0233: @itemx --no-window-system
0234: @opindex --no-window-system
0235: @cindex disable window system
0236: Don't communicate directly with the window system, disregarding the
0237: @env{DISPLAY} environment variable even if it is set.  This means that
0238: Emacs uses the terminal from which it was launched for all its display
0239: and input.
0240: 
0241: @cindex batch mode
0242: @item -batch
0243: @opindex --batch
0244: @itemx --batch
0245: Run Emacs in @dfn{batch mode}.  Batch mode is used for running
0246: programs written in Emacs Lisp from shell scripts, makefiles, and so
0247: on.  To invoke a Lisp program, use the @samp{-batch} option in
0248: conjunction with one or more of @samp{-l}, @samp{-f} or @samp{--eval}
0249: (@pxref{Action Arguments}).  @xref{Command Example}, for an example.
0250: 
0251: In batch mode, Emacs does not display the text being edited, and the
0252: standard terminal interrupt characters such as @kbd{C-z} and @kbd{C-c}
0253: have their usual effect.  Emacs functions that normally print a
0254: message in the echo area will print to either the standard output
0255: stream (@code{stdout}) or the standard error stream (@code{stderr})
0256: instead.  (To be precise, functions like @code{prin1}, @code{princ}
0257: and @code{print} print to @code{stdout}, while @code{message} and
0258: @code{error} print to @code{stderr}.)  Functions that normally read
0259: keyboard input from the minibuffer take their input from the
0260: terminal's standard input stream (@code{stdin}) instead.
0261: 
0262: @samp{--batch} implies @samp{-q} (do not load an initialization file),
0263: but @file{site-start.el} is loaded nonetheless.  It also causes Emacs
0264: to exit after processing all the command options.  In addition, it
0265: disables auto-saving except in buffers for which auto-saving is
0266: explicitly requested, and when saving files it omits the @code{fsync}
0267: system call unless otherwise requested.
0268:

0271: @cindex script mode
0272: Run Emacs in batch mode, like @samp{--batch}, and then read and

0274:

0277:

0281: 
0282: @noindent
0283: which will invoke Emacs with @samp{--script} and supply the name of
0284: the script file as @var{file}.  Emacs Lisp then treats the @samp{#!}
0285: on this first line as a comment delimiter.

0378: @node Command Example
0379: @appendixsec Command Argument Example
0380: 
0381:   Here is an example of using Emacs with arguments and options.  It
0382: assumes you have a Lisp program file called @file{hack-c.el} which, when
0383: loaded, performs some useful operation on the current buffer, expected
0384: to be a C program.
0385: 
0386: @example
0387: emacs --batch foo.c -l hack-c -f save-buffer >& log
0388: @end example
0389: 
0390: @noindent
0391: This says to visit @file{foo.c}, load @file{hack-c.el} (which makes
0392: changes in the visited file), save @file{foo.c} (note that
0393: @code{save-buffer} is the function that @kbd{C-x C-s} is bound to), and
0394: then exit back to the shell (because of @samp{--batch}).  @samp{--batch}
0395: also guarantees there will be no problem redirecting output to
0396: @file{log}, because Emacs will not assume that it has a display terminal
0397: to work with.

0399: @node Environment
0400: @appendixsec Environment Variables
0401: @cindex environment variables
0402: 
0403:   The @dfn{environment} is a feature of the operating system; it
0404: consists of a collection of variables with names and values.  Each
0405: variable is called an @dfn{environment variable}; environment variable
0406: names are case-sensitive, and it is conventional to use upper case
0407: letters only.  The values are all text strings.
0408: 
0409:   What makes the environment useful is that subprocesses inherit the
0410: environment automatically from their parent process.  This means you
0411: can set up an environment variable in your login shell, and all the
0412: programs you run (including Emacs) will automatically see it.
0413: Subprocesses of Emacs (such as shells, compilers, and version control
0414: programs) inherit the environment from Emacs, too.
0415: 
0416: @findex setenv
0417: @findex getenv
0418: @vindex initial-environment
0419:   Inside Emacs, the command @kbd{M-x getenv} reads the name of an
0420: environment variable, and prints its value in the echo area.  @kbd{M-x
0421: setenv} sets a variable in the Emacs environment, and @kbd{C-u M-x
0422: setenv} removes a variable.  (Environment variable substitutions with
0423: @samp{$} work in the value just as in file names; see @ref{File Names
0424: with $}.)  The variable @code{initial-environment} stores the initial
0425: environment inherited by Emacs.
0426: 
0427:   The way to set environment variables outside of Emacs depends on the
0428: operating system, and especially the shell that you are using.  For
0429: example, here's how to set the environment variable @env{ORGANIZATION}
0430: to @samp{not very much} using Bash:
0431: 
0432: @example
0433: export ORGANIZATION="not very much"
0434: @end example
0435: 
0436: @noindent
0437: and here's how to do it in csh or tcsh:
0438: 
0439: @example
0440: setenv ORGANIZATION "not very much"
0441: @end example
0442: 
0443:   When Emacs is using the X Window System, various environment
0444: variables that control X work for Emacs as well.  See the X
0445: documentation for more information.
0446:

0449: * Misc Variables::      Certain system-specific variables.
0450: * MS-Windows Registry:: An alternative to the environment on MS-Windows.

0453: @node General Variables
0454: @appendixsubsec General Variables
0455: 
0456:   Here is an alphabetical list of environment variables that have
0457: special meanings in Emacs.  Most of these variables are also used by
0458: some other programs.  Emacs does not require any of these environment
0459: variables to be set, but it uses their values if they are set.
0460: 
0461: @c This used to be @vtable, but that enters the variables alone into
0462: @c the Variable Index, which in some cases, like HOME, might be
0463: @c confused with keys by that name, and other cases, like NAME,
0464: @c might be confused with general-purpose phrases.
0465: @table @env
0466: @item CDPATH
0467: @vindex CDPATH@r{, environment variable}
0468: Used by the @code{cd} command to search for the directory you specify,
0469: when you specify a relative directory,
0470: @item DBUS_SESSION_BUS_ADDRESS
0471: @vindex DBUS_SESSION_BUS_ADDRESS@r{, environment variable}
0472: Used by D-Bus when Emacs is compiled with it.  Usually, there is no
0473: need to change it.  Setting it to a dummy address, like
0474: @samp{unix:path=/dev/null}, suppresses connections to the D-Bus session
0475: bus as well as autolaunching the D-Bus session bus if not running yet.
0476: @item EMACSDATA
0477: @vindex EMACSDATA@r{, environment variable}
0478: Directory for the architecture-independent files that come with Emacs.
0479: This is used to initialize the variable @code{data-directory}.
0480: @item EMACSDOC
0481: @vindex EMACSDOC@r{, environment variable}
0482: Directory for the documentation string file, which is used to
0483: initialize the Lisp variable @code{doc-directory}.
0484: @item EMACSLOADPATH
0485: @vindex EMACSLOADPATH@r{, environment variable}
0486: A colon-separated list of directories@footnote{Here and below,
0487: whenever we say ``colon-separated list of directories'', it pertains
0488: to Unix and GNU/Linux systems.  On MS-DOS and MS-Windows, the
0489: directories are separated by semi-colons instead, since DOS/Windows
0490: file names might include a colon after a drive letter.} to search for
0491: Emacs Lisp files.  If set, it modifies the usual initial value of the
0492: @code{load-path} variable (@pxref{Lisp Libraries}).  An empty element
0493: stands for the default value of @code{load-path}; e.g., using
0494: @samp{EMACSLOADPATH="/tmp:"} adds @file{/tmp} to the front of
0495: the default @code{load-path}.  To specify an empty element in the
0496: middle of the list, use 2 colons in a row, as in
0497: @samp{EMACSLOADPATH="/tmp::/foo"}.
0498: @item EMACSPATH
0499: @vindex EMACSPATH@r{, environment variable}
0500: A colon-separated list of directories to search for executable files.
0501: If set, Emacs uses this in addition to @env{PATH} (see below) when
0502: initializing the variable @code{exec-path} (@pxref{Shell}).
0503: @item EMAIL
0504: @vindex EMAIL@r{, environment variable}
0505: @vindex user-mail-address@r{, initialization}
0506: Your email address; used to initialize the Lisp variable
0507: @code{user-mail-address}, which the Emacs mail interface puts into the
0508: @samp{From} header of outgoing messages (@pxref{Mail Headers}).
0509: @item ESHELL
0510: @vindex ESHELL@r{, environment variable}
0511: Used for shell-mode to override the @env{SHELL} environment variable
0512: (@pxref{Interactive Shell}).
0513: @item HISTFILE
0514: @vindex HISTFILE@r{, environment variable}
0515: The name of the file that shell commands are saved in between logins.
0516: This variable defaults to @file{~/.bash_history} if you use Bash, to
0517: @file{~/.sh_history} if you use ksh, and to @file{~/.history}
0518: otherwise.
0519: @item HOME
0520: @vindex HOME@r{, environment variable}
0521: The location of your files in the directory tree; used for
0522: expansion of file names starting with a tilde (@file{~}).  On MS-DOS,
0523: it defaults to the directory from which Emacs was started, with
0524: @samp{/bin} removed from the end if it was present.  On Windows, the
0525: default value of @env{HOME} is the @file{Application Data}
0526: subdirectory of the user profile directory (normally, this is

0728: @node MS-Windows Registry
0729: @appendixsubsec The MS-Windows System Registry
0730: @pindex addpm, MS-Windows installation program
0731: @cindex registry, setting environment variables (MS-Windows)
0732: 
0733: On MS-Windows, the installation program @command{addpm.exe} adds
0734: values for @env{emacs_dir}, @env{EMACSLOADPATH}, @env{EMACSDATA},
0735: @env{EMACSPATH}, @env{EMACSDOC}, @env{SHELL} and @env{TERM} to the
0736: @file{HKEY_LOCAL_MACHINE} section of the system registry, under
0737: @file{/Software/GNU/Emacs}.  It does this because there is no standard
0738: place to set environment variables across different versions of
0739: Windows.  Running @command{addpm.exe} is no longer strictly necessary
0740: in recent versions of Emacs, but if you are upgrading from an older
0741: version, running @command{addpm.exe} ensures that you do not have
0742: older registry entries from a previous installation, which may not be
0743: compatible with the latest version of Emacs.
0744: 
0745: When Emacs starts, as well as checking the environment, it also checks
0746: the System Registry for those variables and for @env{HOME}, @env{LANG}
0747: and @env{PRELOAD_WINSOCK}.
0748: 
0749: To determine the value of those variables, Emacs goes through the
0750: following procedure.  First, the environment is checked.  If the
0751: variable is not found there, Emacs looks for registry keys by that
0752: name under @file{/Software/GNU/Emacs}; first in the
0753: @file{HKEY_CURRENT_USER} section of the registry, and if not found
0754: there, in the @file{HKEY_LOCAL_MACHINE} section.  Finally, if Emacs
0755: still cannot determine the values, compiled-in defaults are used.
0756: 
0757: In addition to the environment variables above, you can also add
0758: settings to the @file{/Software/GNU/Emacs} registry key to specify X
0759: resources (@pxref{X Resources}).  Most of the settings you can specify
0760: in your @file{.Xdefaults} file can be set from that registry key.

0762: @node Display X
0763: @appendixsec Specifying the Display Name
0764: @cindex display name (X Window System)
0765: @cindex @env{DISPLAY} environment variable
0766: 
0767:   The environment variable @env{DISPLAY} tells all X clients,
0768: including Emacs, where to display their windows.  Its value is set by
0769: default in ordinary circumstances, when you start an X server and run
0770: jobs locally.  You can specify the display yourself; one reason to do
0771: this is if you want to log into another system and run Emacs there,
0772: and have the window displayed at your local terminal.
0773: 
0774:   @env{DISPLAY} has the syntax
0775: @samp{@var{host}:@var{display}.@var{screen}}, where @var{host} is the
0776: host name of the X Window System server machine, @var{display} is an
0777: arbitrarily-assigned number that distinguishes your server (X
0778: terminal) from other servers on the same machine, and @var{screen} is
0779: a field that allows an X server to control multiple terminal screens.
0780: The period and the @var{screen} field are optional.  If included,
0781: @var{screen} is usually zero.
0782: 
0783:   For example, if your host is named @samp{glasperle} and your server is
0784: the first (or perhaps the only) server listed in the configuration, your
0785: @env{DISPLAY} is @samp{glasperle:0.0}.
0786: 
0787:   You can specify the display name explicitly when you run Emacs, either
0788: by changing the @env{DISPLAY} variable, or with the option @samp{-d
0789: @var{display}} or @samp{--display=@var{display}}.  Here is an example:
0790: 
0791: @smallexample
0792: emacs --display=glasperle:0 &
0793: @end smallexample
0794: 
0795:   You can inhibit the use of the X window system with the @samp{-nw}
0796: option.  Then Emacs uses its controlling text terminal for display.
0797: @xref{Initial Options}.
0798: 
0799:   Sometimes, security arrangements prevent a program on a remote system
0800: from displaying on your local system.  In this case, trying to run Emacs
0801: produces messages like this:
0802: 
0803: @smallexample
0804: Xlib:  connection to "glasperle:0.0" refused by server
0805: @end smallexample
0806: 
0807: @noindent
0808: You might be able to overcome this problem by using the @command{xhost}
0809: command on the local system to give permission for access from your
0810: remote machine.

0998: @noindent
0999: In the @samp{--geometry} option, @code{@r{@{}+-@r{@}}} means either a plus
1000:  sign or a minus sign.  A plus
1001: sign before @var{xoffset} means it is the distance from the left side of
1002: the screen; a minus sign means it counts from the right side.  A plus
1003: sign before @var{yoffset} means it is the distance from the top of the
1004: screen, and a minus sign there indicates the distance from the bottom.
1005: The values @var{xoffset} and @var{yoffset} may themselves be positive or
1006: negative, but that doesn't change their meaning, only their direction.
1007: 
1008:   Emacs uses the same units as @command{xterm} does to interpret the geometry.
1009: The @var{width} and @var{height} are measured in characters, so a large font
1010: creates a larger frame than a small font.  (If you specify a proportional
1011: font, Emacs uses its maximum bounds width as the width unit.)  The
1012: @var{xoffset} and @var{yoffset} are measured in pixels.
1013: 
1014:   You do not have to specify all of the fields in the geometry
1015: specification.  If you omit both @var{xoffset} and @var{yoffset}, the
1016: window manager decides where to put the Emacs frame, possibly by
1017: letting you place it with the mouse.  For example, @samp{164x55}
1018: specifies a window 164 columns wide, enough for two ordinary width
1019: windows side by side, and 55 lines tall.
1020: 
1021:   The default frame width is 80 characters and the default height is
1022: 40 lines.  You can omit either the width or the height or both.  If
1023: you start the geometry with an integer, Emacs interprets it as the
1024: width.  If you start with an @samp{x} followed by an integer, Emacs
1025: interprets it as the height.  Thus, @samp{81} specifies just the
1026: width; @samp{x45} specifies just the height.
1027: 
1028:   If you start the geometry with @samp{+} or @samp{-}, that introduces
1029: an offset, which means both sizes are omitted.  Thus, @samp{-3}
1030: specifies the @var{xoffset} only.  (If you give just one offset, it is
1031: always @var{xoffset}.)  @samp{+3-3} specifies both the @var{xoffset}
1032: and the @var{yoffset}, placing the frame near the bottom left of the
1033: screen.
1034: 
1035:   You can specify a default for any or all of the fields in your X
1036: resource file (@pxref{Resources}), and then override selected fields
1037: with a @samp{--geometry} option.
1038: 
1039:   Since the mode line and the echo area occupy the last 2 lines of the
1040: frame, the height of the initial text window is 2 less than the height
1041: specified in your geometry.  In non-X-toolkit versions of Emacs, the
1042: menu bar also takes one line of the specified number.  But in the X
1043: toolkit version, the menu bar is additional and does not count against
1044: the specified height.  The tool bar, if present, is also additional.
1045: 
1046:   Enabling or disabling the menu bar or tool bar alters the amount of
1047: space available for ordinary text.  Therefore, if Emacs starts up with
1048: a tool bar (which is the default), and handles the geometry
1049: specification assuming there is a tool bar, and then your
1050: initialization file disables the tool bar, you will end up with a
1051: frame geometry different from what you asked for.  To get the intended
1052: size with no tool bar, use an X resource to specify ``no tool bar''
1053: (@pxref{Table of Resources}); then Emacs will already know there's no
1054: tool bar when it processes the specified geometry.
1055: 
1056:   When using one of @samp{--fullscreen}, @samp{--maximized},
1057: @samp{--fullwidth} or @samp{--fullheight}, some window managers require
1058: you to set the variable @code{frame-resize-pixelwise} to a non-@code{nil}
1059: value to make a frame appear truly maximized or full-screen.
1060: 
1061:  Some window managers have options that can make them ignore both
1062: program-specified and user-specified positions.  If these are set,
1063: Emacs fails to position the window correctly.

1065: @node Borders X
1066: @appendixsec Internal and Outer Borders
1067: @cindex borders (X Window System)
1068: 
1069:   An Emacs frame has an internal border and an outer border.  The
1070: internal border is an extra strip of the background color around the
1071: text portion of the frame.  Emacs itself draws the internal border.  The
1072: outer border is drawn by X outside the tool and menu bars of the frame.
1073: There is also an external border which is drawn by the window manager.
1074: The size of the external border cannot be set from within Emacs.
1075: 
1076: @table @samp
1077: @item -ib @var{width}
1078: @opindex -ib
1079: @itemx --internal-border=@var{width}
1080: @opindex --internal-border
1081: @cindex internal border width, command-line argument
1082: Specify @var{width} as the width of the internal border (around the
1083: frame's text area), in pixels.
1084: 
1085: @item -bw @var{width}
1086: @opindex -bw
1087: @itemx --border-width=@var{width}
1088: @opindex --border-width
1089: @cindex main border width, command-line argument
1090: @cindex outer border width, command-line argument
1091: Specify @var{width} as the width of the outer border, in pixels.
1092: @end table
1093: 
1094:   When you specify the size of the frame, that does not count the
1095: borders.  The frame's position is measured from the outside edge of the
1096: external border.
1097: 
1098:   Use the @samp{-ib @var{n}} option to specify an internal border
1099: @var{n} pixels wide.  The default is 1.  Use @samp{-bw @var{n}} to
1100: specify the width of the outer border (though the window manager may not
1101: pay attention to what you specify).  The default width of the outer
1102: border is 2.

1104: @node Title X
1105: @appendixsec Frame Titles
1106: 
1107:   Each Emacs frame always has a title, which appears in window
1108: decorations and icons as the name of the frame.  The default title is
1109: of the form @samp{@var{invocation-name}@@@var{machine}} (if there is
1110: only one frame) or shows the selected window's buffer name (if there
1111: is more than one frame).
1112: 
1113:   You can specify a non-default title for the initial Emacs frame with
1114: a command line option:
1115: 
1116: @table @samp
1117: @item -T @var{title}
1118: @opindex -T
1119: @itemx --title=@var{title}
1120: @opindex --title
1121: @cindex frame title, command-line argument
1122: Specify @var{title} as the title for the initial Emacs frame.
1123: @end table
1124: 
1125:   The @samp{--name} option (@pxref{Resources}) also specifies the title
1126: for the initial Emacs frame.

1135: @opindex --iconic

1141: @opindex -nbi

1147: 
1148:   Most window managers allow you to iconify (or ``minimize'') an
1149: Emacs frame, hiding it from sight.  Some window managers replace
1150: iconified windows with tiny icons, while others remove them
1151: entirely from sight.  The @samp{-iconic} option tells Emacs to begin
1152: running in an iconified state, rather than showing a frame right away.
1153: The text frame doesn't appear until you deiconify (or ``un-minimize'')
1154: it.
1155: 
1156:   By default, Emacs uses an icon containing the Emacs logo.  On
1157: desktop environments such as Gnome, this icon is also displayed in
1158: other contexts, e.g., when switching into an Emacs frame.  The
1159: @samp{-nbi} or @samp{--no-bitmap-icon} option tells Emacs to let the
1160: window manager choose what sort of icon to use---usually just a small
1161: rectangle containing the frame's title.
