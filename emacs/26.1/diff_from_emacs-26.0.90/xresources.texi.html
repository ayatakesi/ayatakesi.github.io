005: @node X Resources
006: @appendix X Options and Resources
007: 
008:   You can customize some X-related aspects of Emacs behavior using X
009: resources, as is usual for programs that use X.
010: 
011:   When Emacs is compiled with GTK+ support, the appearance of various
012: graphical widgets, such as the menu-bar, scroll-bar, and dialog boxes,
013: is determined by
014: @ifnottex
015: GTK resources, which we will also describe.
016: @end ifnottex
017: @iftex
018: GTK resources.
019: @end iftex
020: When Emacs is built without GTK+ support, the appearance of these
021: widgets is determined by additional X resources.
022: 
023:   On MS-Windows, you can customize some of the same aspects using the
024: system registry (@pxref{MS-Windows Registry}).
025:

030: * Motif Resources::     X resources for Motif and LessTif menus.

034: @node Resources
035: @appendixsec X Resources
036: @cindex resources
037: @cindex X resources
038: @cindex @file{~/.Xdefaults} file
039: @cindex @file{~/.Xresources} file
040: 
041:   Programs running under the X Window System organize their user
042: options under a hierarchy of classes and resources.  You can specify
043: default values for these options in your @dfn{X resource file},
044: usually named @file{~/.Xdefaults} or @file{~/.Xresources}.  Changes in
045: this file do not take effect immediately, because the X server stores
046: its own list of resources; to update it, use the command
047: @command{xrdb}---for instance, @samp{xrdb ~/.Xdefaults}.
048: 
049: @cindex registry, setting resources (MS-Windows)
050:   (MS-Windows systems do not support X resource files; on such systems,
051: Emacs looks for X resources in the Windows Registry, first under the
052: key @samp{HKEY_CURRENT_USER\SOFTWARE\GNU\Emacs}, which affects only
053: the current user and override the system-wide settings, and then under
054: the key @samp{HKEY_LOCAL_MACHINE\SOFTWARE\GNU\Emacs}, which affects
055: all users of the system.  The menu and scroll bars are native widgets
056: on MS-Windows, so they are only customizable via the system-wide
057: settings in the Display Control Panel.  You can also set resources
058: using the @samp{-xrm} command line option, as explained below.)
059: 
060:   Each line in the X resource file specifies a value for one option or
061: for a collection of related options.  The order in which the lines
062: appear in the file does not matter.  Each resource specification
063: consists of a @dfn{program name} and a @dfn{resource name}.  Case
064: distinctions are significant in each of these names.  Here is an
065: example:
066: 
067: @example
068: emacs.cursorColor: dark green
069: @end example
070: 
071:   The program name is the name of the executable file to which the
072: resource applies.  For Emacs, this is normally @samp{emacs}.  To
073: specify a definition that applies to all instances of Emacs,
074: regardless of the name of the Emacs executable, use @samp{Emacs}.
075: 
076:   The resource name is the name of a program setting.  For instance,
077: Emacs recognizes a @samp{cursorColor} resource that controls the color
078: of the text cursor.
079: 
080:   Resources are grouped into named classes.  For instance, the
081: @samp{Foreground} class contains the @samp{cursorColor},
082: @samp{foreground} and @samp{pointerColor} resources (@pxref{Table of
083: Resources}).  Instead of using a resource name, you can use a class
084: name to specify the default value for all resources in that class,
085: like this:
086: 
087: @example
088: emacs.Foreground: dark green
089: @end example
090: 
091:   Emacs does not process X resources at all if you set the variable
092: @code{inhibit-x-resources} to a non-@code{nil} value.  If you invoke
093: Emacs with the @samp{-Q} (or @samp{--quick}) command-line option,
094: @code{inhibit-x-resources} is automatically set to @code{t}
095: (@pxref{Initial Options}).
096: 
097: @ifnottex
098:   In addition, you can use the following command-line options to
099: override the X resources file:
100: 
101: @table @samp
102: @item -name @var{name}
103: @opindex --name
104: @itemx --name=@var{name}
105: @cindex resource name, command-line argument
106: This option sets the program name of the initial Emacs frame to
107: @var{name}.  It also sets the title of the initial frame to
108: @var{name}.  This option does not affect subsequent frames.
109: 
110: If you don't specify this option, the default is to use the Emacs
111: executable's name as the program name.
112: 
113: For consistency, @samp{-name} also specifies the name to use for other
114: resource values that do not belong to any particular frame.
115: 
116: The resources that name Emacs invocations also belong to a class,
117: named @samp{Emacs}.  If you write @samp{Emacs} instead of
118: @samp{emacs}, the resource applies to all frames in all Emacs jobs,
119: regardless of frame titles and regardless of the name of the
120: executable file.
121: 
122: @item -xrm @var{resource-values}
123: @opindex --xrm
124: @itemx --xrm=@var{resource-values}
125: @cindex resource values, command-line argument
126: This option specifies X resource values for the present Emacs job.
127: 
128: @var{resource-values} should have the same format that you would use
129: inside a file of X resources.  To include multiple resource
130: specifications in @var{resource-values}, put a newline between them,
131: just as you would in a file.  You can also use @samp{#include
132: "@var{filename}"} to include a file full of resource specifications.
133: Resource values specified with @samp{-xrm} take precedence over all
134: other resource specifications.
135: @end table
136: @end ifnottex

139: @appendixsec Table of X Resources for Emacs
140: 
141:   This table lists the X resource names that Emacs recognizes,
142: excluding those that control the appearance of graphical widgets like
143: the menu bar:
144: 
145: @table @asis
146: @item @code{background} (class @code{Background})
147: Background color (@pxref{Colors}).
148: 
149: @item @code{bitmapIcon} (class @code{BitmapIcon})
150: Tell the window manager to display the Emacs icon if @samp{on}; don't
151: do so if @samp{off}.  @xref{Icons X}, for a description of the icon.
152: 
153: @ifnottex
154: @item @code{borderColor} (class @code{BorderColor})
155: Color of the frame's external border.  This has no effect if Emacs is
156: compiled with GTK+ support.
157: 
158: @item @code{borderWidth} (class @code{BorderWidth})
159: Width of the frame's external border, in pixels.  This has no effect
160: if Emacs is compiled with GTK+ support.
161: @end ifnottex
162: 
163: @item @code{cursorColor} (class @code{Foreground})
164: Text cursor color.  If this resource is specified when Emacs starts
165: up, Emacs sets its value as the background color of the @code{cursor}
166: face (@pxref{Faces}).
167: 
168: @item @code{cursorBlink} (class @code{CursorBlink})
169: If the value of this resource is @samp{off} or @samp{false} or
170: @samp{0} at startup, Emacs disables Blink Cursor mode (@pxref{Cursor
171: Display}).
172: 
173: @item @code{font} (class @code{Font})
174: Font name for the @code{default} face (@pxref{Fonts}).  You can also
175: specify a fontset name (@pxref{Fontsets}).
176: 
177: @item @code{fontBackend} (class @code{FontBackend})
178: Comma-delimited list of backend(s) to use for drawing fonts, in order
179: of precedence.  For instance, the value @samp{x,xft} tells Emacs to
180: draw fonts using the X core font driver, falling back on the Xft font
181: driver if that fails.  Normally, you should leave this resource unset,
182: in which case Emacs tries using all available font backends.
183: 
184: @item @code{foreground} (class @code{Foreground})
185: Default foreground color for text.
186: 
187: @item @code{geometry} (class @code{Geometry})
188: Window size and position.  The value should be a size and position
189: specification, of the same form as in the @samp{-g} or
190: @samp{--geometry} command-line option (@pxref{Window Size X}).
191: 
192: The size applies to all frames in the Emacs session, but the position
193: applies only to the initial Emacs frame (or, in the case of a resource
194: for a specific frame name, only that frame).
195: 
196: 
197: Be careful not to specify this resource as @samp{emacs*geometry}, as
198: that may affect individual menus as well as the main Emacs frame.
199: 
200: @item @code{fullscreen} (class @code{Fullscreen})
201: The desired fullscreen size.  The value can be one of @code{fullboth},
202: @code{maximized}, @code{fullwidth} or @code{fullheight}, which
203: correspond to the command-line options @samp{-fs}, @samp{-mm},
204: @samp{-fw}, and @samp{-fh} (@pxref{Window Size X}).  Note that this
205: applies to the initial frame only.
206: 
207: @ifnottex
208: @item @code{iconName} (class @code{Title})
209: Name to display in the icon.
210: 
211: @item @code{internalBorder} (class @code{BorderWidth})
212: Width of the internal frame border, in pixels.
213: @end ifnottex
214: 
215: @item @code{lineSpacing} (class @code{LineSpacing})
216: @cindex line spacing
217: Additional space between lines, in pixels.
218: 
219: @item @code{menuBar} (class @code{MenuBar})
220: @cindex menu bar
221: If the value of this resource is @samp{off} or @samp{false} or
222: @samp{0}, Emacs disables Menu Bar mode at startup (@pxref{Menu Bars}).
223: 
224: @ifnottex
225: @item @code{minibuffer} (class @code{Minibuffer})
226: If @samp{none}, Emacs will not make a minibuffer in this frame; it
227: will use a separate minibuffer frame instead.
228: 
229: @item @code{paneFont} (class @code{Font})
230: @cindex font for menus
231: Font name for menu pane titles, in non-toolkit versions of Emacs.
232: @end ifnottex
233: 
234: @item @code{pointerColor} (class @code{Foreground})
235: Color of the mouse cursor.  This has no effect in many graphical
236: desktop environments, as they do not let Emacs change the mouse cursor
237: this way.
238: 
239: @ifnottex
240: @item @code{privateColormap} (class @code{PrivateColormap})
241: If @samp{on}, use a private color map, in the case where the default
242: visual of class PseudoColor and Emacs is using it.
243: 
244: @item @code{reverseVideo} (class @code{ReverseVideo})
245: Switch foreground and background default colors if @samp{on}, use colors as
246: specified if @samp{off}.
247: 
248: @item @code{screenGamma} (class @code{ScreenGamma})
249: @cindex gamma correction
250: Gamma correction for colors, equivalent to the frame parameter
251: @code{screen-gamma}.
252: 
253: @item @code{scrollBar} (class @code{ScrollBar})
254: @cindex tool bar
255: If the value of this resource is @samp{off} or @samp{false} or
256: @samp{0}, Emacs disables Scroll Bar mode at startup (@pxref{Scroll Bars}).
257: 
258: @item @code{scrollBarWidth} (class @code{ScrollBarWidth})
259: @cindex scrollbar width
260: The scroll bar width in pixels, equivalent to the frame parameter
261: @code{scroll-bar-width}.  Do not set this resource if Emacs is
262: compiled with GTK+ support.
263: @end ifnottex
264: 
265: @ifnottex
266: @item @code{selectionFont} (class @code{SelectionFont})
267: Font name for pop-up menu items, in non-toolkit versions of Emacs.  (For
268: toolkit versions, see @ref{Lucid Resources}, also see @ref{Motif
269: Resources}.)
270: 
271: @item @code{selectionTimeout} (class @code{SelectionTimeout})
272: Number of milliseconds to wait for a selection reply.
273: If the selection owner doesn't reply in this time, we give up.
274: A value of 0 means wait as long as necessary.
275: 
276: @item @code{synchronous} (class @code{Synchronous})
277: @cindex debugging X problems
278: @cindex synchronous X mode
279: Run Emacs in synchronous mode if @samp{on}.  Synchronous mode is
280: useful for debugging X problems.
281: @end ifnottex
282: 
283: @item @code{title} (class @code{Title})
284: Name to display in the title bar of the initial Emacs frame.
285: 
286: @item @code{toolBar} (class @code{ToolBar})
287: @cindex tool bar
288: If the value of this resource is @samp{off} or @samp{false} or
289: @samp{0}, Emacs disables Tool Bar mode at startup (@pxref{Tool Bars}).
290: 
291: @item @code{useXIM} (class @code{UseXIM})
292: @cindex XIM
293: @cindex X input methods
294: @cindex input methods, X
295: Disable use of X input methods (XIM) if @samp{false} or @samp{off}.
296: This is only relevant if your Emacs is built with XIM support.  It
297: might be useful to turn off XIM on slow X client/server links.
298: 
299: @item @code{verticalScrollBars} (class @code{ScrollBars})
300: Give frames scroll bars if @samp{on}; don't have scroll bars if
301: @samp{off}.
302: 
303: @ifnottex
304: @item @code{visualClass} (class @code{VisualClass})
305: The @dfn{visual class} for X color display.  If specified, the value
306: should start with one of @samp{TrueColor}, @samp{PseudoColor},
307: @samp{DirectColor}, @samp{StaticColor}, @samp{GrayScale}, and
308: @samp{StaticGray}, followed by @samp{-@var{depth}}, where @var{depth}
309: is the number of color planes.
310: @end ifnottex
311: @end table
312: 
313:   You can also use X resources to customize individual Emacs faces
314: (@pxref{Faces}).  For example, setting the resource
315: @samp{@var{face}.attributeForeground} is equivalent to customizing the
316: @samp{foreground} attribute of the face @var{face}.  However, we
317: recommend customizing faces from within Emacs, instead of using X

321: @node Lucid Resources
322: @appendixsec Lucid Menu And Dialog X Resources
323: @cindex Menu X Resources (Lucid widgets)
324: @cindex Dialog X Resources (Lucid widgets)
325: @cindex Lucid Widget X Resources
326: 
327:   If Emacs is compiled with the X toolkit support using Lucid widgets,
328: you can use X resources to customize the appearance of the menu bar,
329: pop-up menus, and dialog boxes.  The resources for the menu bar fall
330: in the @samp{pane.menubar} class (following, as always, either the
331: name of the Emacs executable or @samp{Emacs} for all Emacs
332: invocations).  The resources for the pop-up menu are in the
333: @samp{menu*} class.  The resources for dialog boxes are in the
334: @samp{dialog*} class.
335: 
336:   For example, to display menu bar entries with the @samp{Courier-12}
337: font (@pxref{Fonts}), write this:
338: 
339: @example
340: Emacs.pane.menubar.font: Courier-12
341: @end example
342: 
343: @noindent
344: Lucid widgets can display multilingual text in your locale.  To enable
345: this, specify a @code{fontSet} resource instead of a @code{font}
346: resource.  @xref{Fontsets}.  If both @code{font} and @code{fontSet}
347: resources are specified, the @code{fontSet} resource is used.
348: 
349: Here is a list of resources for menu bars, pop-up menus, and dialogs:
350: 
351: @table @code
352: @item font
353: Font for menu item text.
354: @item fontSet
355: Fontset for menu item text.
356: @item foreground
357: Foreground color.
358: @item background
359: Background color.
360: @item buttonForeground
361: Foreground color for a selected item.
362: @ifnottex
363: @item horizontalSpacing
364: Horizontal spacing in pixels between items.  Default is 3.
365: @item verticalSpacing
366: Vertical spacing in pixels between items.  Default is 2.
367: @item arrowSpacing
368: Horizontal spacing between the arrow (which indicates a submenu) and
369: the associated text.  Default is 10.
370: @item shadowThickness
371: Thickness of shadow lines for 3D buttons, arrows, and other graphical
372: elements.  Default is 1.
373: @end ifnottex
374: @item margin
375: Margin of the menu bar, in characters.  Default is 1.
376: @end table

378: @node Motif Resources
379: @appendixsec Motif Menu X Resources
380: @cindex Menu X Resources (Motif widgets)
381: @cindex Motif Widget X Resources
382: 
383:   If Emacs is compiled with the X toolkit support using Motif or
384: LessTif widgets, you can use X resources to customize the appearance
385: of the menu bar, pop-up menus, and dialog boxes.  However, the
386: resources are organized differently from Lucid widgets.
387: 
388:   The resource names for the menu bar are in the @samp{pane.menubar}
389: class, and they must be specified in this form:
390: 
391: @smallexample
392: Emacs.pane.menubar.@var{subwidget}.@var{resource}:  @var{value}
393: @end smallexample
394: 
395: @noindent
396: For pop-up menus, the resources are in the @samp{menu*} class, instead
397: of @samp{pane.menubar}.  For dialog boxes, they are in @samp{dialog}.
398: In each case, each individual menu string is a subwidget; the
399: subwidget's name is the same as the menu item string.  For example,
400: the @samp{File} menu in the menu bar is a subwidget named
401: @samp{emacs.pane.menubar.File}.
402: 
403:   Typically, you want to specify the same resources for the whole menu
404: bar.  To do this, use @samp{*} instead of a specific subwidget name.
405: For example, to specify the font @samp{8x16} for all menu bar items,
406: including submenus, write this:
407: 
408: @smallexample
409: Emacs.pane.menubar.*.fontList:  8x16
410: @end smallexample
411: 
412:   Each item in a submenu also has its own name for X resources; for
413: example, the @samp{File} submenu has an item named @samp{Save (current
414: buffer)}.  A resource specification for a submenu item looks like
415: this:
416: 
417: @smallexample
418: Emacs.pane.menubar.popup_*.@var{menu}.@var{item}.@var{resource}: @var{value}
419: @end smallexample
420: 
421: @noindent
422: For example, here's how to specify the font for the @samp{Save (current
423: buffer)} item:
424: 
425: @smallexample
426: Emacs.pane.menubar.popup_*.File.Save (current buffer).fontList: 8x16
427: @end smallexample
428: 
429: @noindent
430: For an item in a second-level submenu, such as @samp{Complete Word}
431: under @samp{Spell Checking} under @samp{Tools}, the resource fits this
432: template:
433: 
434: @smallexample
435: Emacs.pane.menubar.popup_*.popup_*.@var{menu}.@var{resource}: @var{value}
436: @end smallexample
437: 
438: @noindent
439: For example,
440: 
441: @smallexample
442: Emacs.pane.menubar.popup_*.popup_*.Spell Checking.Complete Word: @var{value}
443: @end smallexample
444: 
445: @noindent
446: (This should be one long line.)
447: 
448:   If you want the submenu items to look different from the menu bar
449: itself, you must first specify the resource for all of them, then
450: override the value for submenus alone.  Here is an example:
451: 
452: @smallexample
453: Emacs.pane.menubar.*.fontList:  8x16
454: Emacs.pane.menubar.popup_*.fontList: 8x16
455: @end smallexample
456: 
457:   To specify resources for the LessTif file-selection box, use
458: @samp{fsb*}, like this:
459: 
460: @example
461: Emacs.fsb*.fontList: 8x16
462: @end example
463: 
464:   Here is a list of resources for LessTif menu bars and pop-up menus:
465: 
466: @table @code
467: @item armColor
468: The color to show in an armed button.
469: @item fontList
470: The font to use.
471: @item marginBottom
472: @itemx marginHeight
473: @itemx marginLeft
474: @itemx marginRight
475: @itemx marginTop
476: @itemx marginWidth
477: Amount of space to leave around the item, within the border.
478: @item borderWidth
479: The width of the border around the menu item, on all sides.
480: @item shadowThickness
481: The width of the border shadow.
482: @item bottomShadowColor
483: The color for the border shadow, on the bottom and the right.
484: @item topShadowColor
485: The color for the border shadow, on the top and the left.
486: @end table

489: @node GTK resources
490: @appendixsec GTK resources
491: @cindex GTK+ resources
492: @cindex resource files for GTK
493: @cindex @file{~/.gtkrc-2.0} file
494: @cindex @file{~/.emacs.d/gtkrc} file
495: 
496:   If Emacs is compiled with GTK+ toolkit support, the simplest way to
497: customize its GTK+ widgets (e.g., menus, dialogs, tool bars and
498: scroll bars) is to choose an appropriate GTK+ theme, for example with
499: the GNOME theme selector.
500: 
501:   In GTK+ version 2, you can also use @dfn{GTK+ resources} to
502: customize the appearance of GTK+ widgets used by Emacs.  These
503: resources are specified in either the file @file{~/.emacs.d/gtkrc}
504: (for Emacs-specific GTK+ resources), or @file{~/.gtkrc-2.0} (for
505: general GTK+ resources).  We recommend using @file{~/.emacs.d/gtkrc},
506: since GTK+ seems to ignore @file{~/.gtkrc-2.0} when running GConf with
507: GNOME@.  Note, however, that some GTK themes may override
508: customizations in @file{~/.emacs.d/gtkrc}; there is nothing we can do
509: about this.  GTK+ resources do not affect aspects of Emacs unrelated
510: to GTK+ widgets, such as fonts and colors in the main Emacs window;
511: those are governed by normal X resources (@pxref{Resources}).
512: 
513:   The following sections describe how to customize GTK+ resources for
514: Emacs.  For details about GTK+ resources, see the GTK+ API document at
515: @uref{http://developer.gnome.org/gtk2/stable/gtk2-Resource-Files.html}.
516: 
517:   In GTK+ version 3, GTK+ resources have been replaced by a completely
518: different system.  The appearance of GTK+ widgets is now determined by
519: CSS-like style files: @file{gtk-3.0/gtk.css} in the GTK+ installation
520: directory, and @file{~/.themes/@var{theme}/gtk-3.0/gtk.css} for local
521: style settings (where @var{theme} is the name of the current GTK+
522: theme).  Therefore, the description of GTK+ resources in this section
523: does not apply to GTK+ 3.  For details about the GTK+ 3 styling
524: system, see
525: @uref{http://developer.gnome.org/gtk3/3.0/GtkCssProvider.html}.
526:

534: @node GTK Resource Basics
535: @appendixsubsec GTK Resource Basics
536: 
537:   In a GTK+ 2 resource file (usually @file{~/.emacs.d/gtkrc}), the
538: simplest kinds of resource settings simply assign a value to a
539: variable.  For example, putting the following line in the resource
540: file changes the font on all GTK+ widgets to @samp{courier-12}:
541: 
542: @smallexample
543: gtk-font-name = "courier 12"
544: @end smallexample
545: 
546: @noindent
547: Note that in this case the font name must be supplied as a GTK font
548: pattern (also called a @dfn{Pango font name}), not as a
549: Fontconfig-style font name or XLFD@.  @xref{Fonts}.
550: 
551:   To customize widgets you first define a @dfn{style}, and then apply
552: the style to the widgets.  Here is an example that sets the font for
553: menus (@samp{#} characters indicate comments):
554: 
555: @smallexample
556: # @r{Define the style @samp{my_style}.}
557: style "my_style"
558: @{
559:   font_name = "helvetica bold 14"
560: @}
561: 
562: # @r{Specify that widget type @samp{*emacs-menuitem*} uses @samp{my_style}.}
563: widget "*emacs-menuitem*" style "my_style"
564: @end smallexample
565: 
566: @noindent
567: The widget name in this example contains wildcards, so the style is
568: applied to all widgets matching @samp{*emacs-menuitem*}.  The widgets
569: are named by the way they are contained, from the outer widget to the
570: inner widget.  Here is another example that applies @samp{my_style}
571: specifically to the Emacs menu bar:
572: 
573: @smallexample
574: widget "Emacs.pane.menubar.*" style "my_style"
575: @end smallexample
576: 
577:   Here is a more elaborate example, showing how to change the parts of
578: the scroll bar:
579: 
580: @smallexample
581: style "scroll"

592: @node GTK Widget Names
593: @appendixsubsec GTK widget names
594: @cindex GTK widget names
595: 
596:   A GTK+ widget is specified by a @dfn{widget name} and a @dfn{widget
597: class}.  The widget name refers to a specific widget
598: (e.g., @samp{emacs-menuitem}), while the widget class refers to a
599: collection of similar widgets (e.g., @samp{GtkMenuItem}).  A widget
600: always has a class, but need not have a name.
601: 
602:   @dfn{Absolute names} are sequences of widget names or widget
603: classes, corresponding to hierarchies of widgets embedded within
604: other widgets.  For example, if a @code{GtkWindow} named @code{top}
605: contains a @code{GtkVBox} named @code{box}, which in turn contains
606: a @code{GtkMenuBar} called @code{menubar}, the absolute class name
607: of the menu-bar widget is @code{GtkWindow.GtkVBox.GtkMenuBar}, and
608: its absolute widget name is @code{top.box.menubar}.
609: 
610:   GTK+ resource files can contain two types of commands for specifying
611: widget appearances:
612: 
613: @table @code
614: @item widget
615: specifies a style for widgets based on the class name, or just the
616: class.
617: 
618: @item widget_class
619: specifies a style for widgets based on the class name.
620: @end table
621: 
622: @noindent
623: See the previous subsection for examples of using the @code{widget}
624: command; the @code{widget_class} command is used similarly.  Note that
625: the widget name/class and the style must be enclosed in double-quotes,
626: and these commands must be at the top level in the GTK+ resource file.
627: 
628:   As previously noted, you may specify a widget name or class with
629: shell wildcard syntax: @samp{*} matches zero or more characters and
630: @samp{?} matches one character.  This example assigns a style to all
631: widgets:
632: 
633: @smallexample
634: widget "*" style "my_style"
635: @end smallexample

673:   For example, here are two different ways to set the menu bar style:

678: @end smallexample

680:   For GTK+ dialogs, Emacs uses a widget named @code{emacs-dialog}, of
681: class @code{GtkDialog}.  For file selection, Emacs uses a widget named
682: @code{emacs-filedialog}, of class @code{GtkFileSelection}.

684:   Because the widgets for pop-up menus and dialogs are free-standing
685: windows and not contained in the @code{Emacs} widget, their GTK+
686: absolute names do not start with @samp{Emacs}.  To customize these
687: widgets, use wildcards like this:
688: 
689: @smallexample

691: widget "*emacs-filedialog* style "my_file_style"
692: widget "*emacs-menuitem* style "my_menu_style"
693: @end smallexample

704: 
705:   Here is an example of two GTK+ style declarations:
706: 
707: @smallexample
708: pixmap_path "/usr/share/pixmaps:/usr/include/X11/pixmaps"
709: 
710: style "default"
711: @{
712:   font_name = "helvetica 12"
713: 
714:   bg[NORMAL] = @{ 0.83, 0.80, 0.73 @}
715:   bg[SELECTED] = @{ 0.0, 0.55, 0.55 @}
716:   bg[INSENSITIVE] = @{ 0.77, 0.77, 0.66 @}
717:   bg[ACTIVE] = @{ 0.0, 0.55, 0.55 @}
718:   bg[PRELIGHT] = @{ 0.0, 0.55, 0.55 @}
719: 
720:   fg[NORMAL] = "black"
721:   fg[SELECTED] = @{ 0.9, 0.9, 0.9 @}
722:   fg[ACTIVE] = "black"
723:   fg[PRELIGHT] = @{ 0.9, 0.9, 0.9 @}
724: 
725:   base[INSENSITIVE] = "#777766"
726:   text[INSENSITIVE] = @{ 0.60, 0.65, 0.57 @}
727: 
728:   bg_pixmap[NORMAL] = "background.xpm"
729:   bg_pixmap[INSENSITIVE] = "background.xpm"
730:   bg_pixmap[ACTIVE] = "background.xpm"
731:   bg_pixmap[PRELIGHT] = "<none>"
732: 
733: @}
734: 
735: style "ruler" = "default"
736: @{
737:   font_name = "helvetica 8"
738: @}
739: 
740: @end smallexample
741: 
742:   The style @samp{ruler} inherits from @samp{default}.  This way you can build
743: on existing styles.  The syntax for fonts and colors is described below.
744: 
745:   As this example shows, it is possible to specify several values for
746: foreground and background depending on the widget's @dfn{state}.  The
747: possible states are:
748: 
749: @table @code
750: @item NORMAL
751: This is the default state for widgets.
752: @item ACTIVE
753: This is the state for a widget that is ready to do something.  It is
754: also for the trough of a scroll bar, i.e., @code{bg[ACTIVE] = "red"}
755: sets the scroll bar trough to red.  Buttons that have been armed
756: (pressed but not released yet) are in this state.
757: @item PRELIGHT
758: This is the state for a widget that can be manipulated, when the mouse
759: pointer is over it---for example when the mouse is over the thumb in
760: the scroll bar or over a menu item.  When the mouse is over a button
761: that is not pressed, the button is in this state.
762: @item SELECTED
763: This is the state for data that has been selected by the user.  It can
764: be selected text or items selected in a list.  This state is not used
765: in Emacs.
766: @item INSENSITIVE
767: This is the state for widgets that are visible, but they can not be
768: manipulated in the usual way---for example, buttons that can't be
769: pressed, and disabled menu items.  To display disabled menu items in
770: yellow, use @code{fg[INSENSITIVE] = "yellow"}.
771: @end table
772: 
773:   Here are the things that can go in a style declaration:
774: 
775: @table @code
776: @item bg[@var{state}] = @var{color}
777: This specifies the background color for the widget.  Note that
778: editable text doesn't use @code{bg}; it uses @code{base} instead.
779: 
780: @item base[@var{state}] = @var{color}
781: This specifies the background color for editable text.  In Emacs, this
782: color is used for the background of the text fields in the file
783: dialog.
784: 
785: @item bg_pixmap[@var{state}] = "@var{pixmap}"
786: This specifies an image background (instead of a background color).
787: @var{pixmap} should be the image file name.  GTK can use a number of
788: image file formats, including XPM, XBM, GIF, JPEG and PNG@.  If you
789: want a widget to use the same image as its parent, use
790: @samp{<parent>}.  If you don't want any image, use @samp{<none>}.
791: @samp{<none>} is the way to cancel a background image inherited from a
792: parent style.
793: 
794: You can't specify the file by its absolute file name.  GTK looks for
795: the pixmap file in directories specified in @code{pixmap_path}.
796: @code{pixmap_path} is a colon-separated list of directories within
797: double quotes, specified at the top level in a @file{gtkrc} file
798: (i.e., not inside a style definition; see example above):
799: 
800: @smallexample
801: pixmap_path "/usr/share/pixmaps:/usr/include/X11/pixmaps"
802: @end smallexample
803: 
804: @item fg[@var{state}] = @var{color}
805: This specifies the foreground color for widgets to use.  It is the
806: color of text in menus and buttons, and the color for the arrows in
807: the scroll bar.  For editable text, use @code{text}.
808: 
809: @item text[@var{state}] = @var{color}
810: This is the color for editable text.  In Emacs, this color is used for the
811: text fields in the file dialog.
812: 
813: @item font_name = "@var{font}"
814: This specifies the font for text in the widget.  @var{font} is a
815: GTK-style (or Pango) font name, like @samp{Sans Italic 10}.
816: @xref{Fonts}.  The names are case insensitive.
817: @end table
818: 
819:   There are three ways to specify a color: a color name, an RGB
820: triplet, or a GTK-style RGB triplet.  @xref{Colors}, for a description
821: of color names and RGB triplets.  Color names should be enclosed with
822: double quotes, e.g., @samp{"red"}.  RGB triplets should be written
823: without double quotes, e.g., @samp{#ff0000}.  GTK-style RGB triplets
824: have the form @w{@code{@{ @var{r}, @var{g}, @var{b} @}}}, where
825: @var{r}, @var{g} and @var{b} are either integers in the range 0--65535
826: or floats in the range 0.0--1.0.
