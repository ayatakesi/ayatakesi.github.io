0007: @chapter Commands for Human Languages
0008: @cindex text
0009: @cindex manipulating text
0010: 
0011:   This chapter describes Emacs commands that act on @dfn{text}, by
0012: which we mean sequences of characters in a human language (as opposed
0013: to, say, a computer programming language).  These commands act in ways
0014: that take into account the syntactic and stylistic conventions of
0015: human languages: conventions involving words, sentences, paragraphs,
0016: and capital letters.  There are also commands for @dfn{filling}, which
0017: means rearranging the lines of a paragraph to be approximately equal
0018: in length.  These commands, while intended primarily for editing text,
0019: are also often useful for editing programs.
0020: 
0021:   Emacs has several major modes for editing human-language text.  If
0022: the file contains ordinary text, use Text mode, which customizes Emacs
0023: in small ways for the syntactic conventions of text.  Outline mode
0024: provides special commands for operating on text with an outline
0025: structure.  Org mode extends Outline mode and turn Emacs into a
0026: full-fledged organizer: you can manage TODO lists, store notes and
0027: publish them in many formats.

0086: @node Words
0087: @section Words
0088: @cindex words
0089: @cindex Meta commands and words
0090: 
0091:   Emacs defines several commands for moving over or operating on
0092: words:
0093: 
0094: @table @kbd
0095: @item M-f
0096: Move forward over a word (@code{forward-word}).
0097: @item M-b
0098: Move backward over a word (@code{backward-word}).
0099: @item M-d
0100: Kill up to the end of a word (@code{kill-word}).
0101: @item M-@key{DEL}
0102: Kill back to the beginning of a word (@code{backward-kill-word}).
0103: @item M-@@
0104: Mark the end of the next word (@code{mark-word}).
0105: @item M-t
0106: Transpose two words or drag a word across others
0107: (@code{transpose-words}).
0108: @end table
0109: 
0110:   Notice how these keys form a series that parallels the character-based
0111: @kbd{C-f}, @kbd{C-b}, @kbd{C-d}, @key{DEL} and @kbd{C-t}.  @kbd{M-@@} is
0112: cognate to @kbd{C-@@}, which is an alias for @kbd{C-@key{SPC}}.
0113: 
0114: @kindex M-f
0115: @kindex M-b
0116: @findex forward-word
0117: @findex backward-word
0118:   The commands @kbd{M-f} (@code{forward-word}) and @kbd{M-b}
0119: (@code{backward-word}) move forward and backward over words.  These
0120: @key{META}-based key sequences are analogous to the key sequences
0121: @kbd{C-f} and @kbd{C-b}, which move over single characters.  The
0122: analogy extends to numeric arguments, which serve as repeat counts.
0123: @kbd{M-f} with a negative argument moves backward, and @kbd{M-b} with
0124: a negative argument moves forward.  Forward motion stops right after
0125: the last letter of the word, while backward motion stops right before
0126: the first letter.
0127: 
0128: @kindex M-d
0129: @findex kill-word
0130:   @kbd{M-d} (@code{kill-word}) kills the word after point.  To be
0131: precise, it kills everything from point to the place @kbd{M-f} would
0132: move to.  Thus, if point is in the middle of a word, @kbd{M-d} kills
0133: just the part after point.  If some punctuation comes between point
0134: and the next word, it is killed along with the word.  (If you wish to
0135: kill only the next word but not the punctuation before it, simply do
0136: @kbd{M-f} to get the end, and kill the word backwards with
0137: @kbd{M-@key{DEL}}.)  @kbd{M-d} takes arguments just like @kbd{M-f}.
0138: 
0139: @findex backward-kill-word
0140: @kindex M-DEL
0141:   @kbd{M-@key{DEL}} (@code{backward-kill-word}) kills the word before
0142: point.  It kills everything from point back to where @kbd{M-b} would
0143: move to.  For instance, if point is after the space in @w{@samp{FOO,
0144: BAR}}, it kills @w{@samp{FOO, }}.  If you wish to kill just
0145: @samp{FOO}, and not the comma and the space, use @kbd{M-b M-d} instead
0146: of @kbd{M-@key{DEL}}.
0147: 
0148: @c Don't index M-t and transpose-words here, they are indexed in
0149: @c fixit.texi, in the node "Transpose".
0150: @c @kindex M-t
0151: @c @findex transpose-words
0152:   @kbd{M-t} (@code{transpose-words}) exchanges the word before or
0153: containing point with the following word.  The delimiter characters between
0154: the words do not move.  For example, @w{@samp{FOO, BAR}} transposes into
0155: @w{@samp{BAR, FOO}} rather than @samp{@w{BAR FOO,}}.  @xref{Transpose}, for
0156: more on transposition.
0157: 
0158: @kindex M-@@
0159: @findex mark-word
0160:   To operate on words with an operation which acts on the region, use
0161: the command @kbd{M-@@} (@code{mark-word}).  This command sets the mark
0162: where @kbd{M-f} would move to.  @xref{Marking Objects}, for more
0163: information about this command.
0164: 
0165:   The word commands' understanding of word boundaries is controlled by
0166: the syntax table.  Any character can, for example, be declared to be a
0167: word delimiter.  @xref{Syntax Tables,, Syntax Tables, elisp, The Emacs
0168: Lisp Reference Manual}.
0169: 
0170:   In addition, see @ref{Position Info} for the @kbd{M-=}
0171: (@code{count-words-region}) and @kbd{M-x count-words} commands, which
0172: count and report the number of words in the region or buffer.

0174: @node Sentences
0175: @section Sentences
0176: @cindex sentences
0177: @cindex manipulating sentences
0178: 
0179:   The Emacs commands for manipulating sentences and paragraphs are
0180: mostly on Meta keys, like the word-handling commands.
0181: 
0182: @table @kbd
0183: @item M-a
0184: Move back to the beginning of the sentence (@code{backward-sentence}).
0185: @item M-e
0186: Move forward to the end of the sentence (@code{forward-sentence}).
0187: @item M-k
0188: Kill forward to the end of the sentence (@code{kill-sentence}).
0189: @item C-x @key{DEL}
0190: Kill back to the beginning of the sentence (@code{backward-kill-sentence}).
0191: @end table
0192: 
0193: @kindex M-a
0194: @kindex M-e
0195: @findex backward-sentence
0196: @findex forward-sentence
0197:   The commands @kbd{M-a} (@code{backward-sentence}) and @kbd{M-e}
0198: (@code{forward-sentence}) move to the beginning and end of the current
0199: sentence, respectively.  Their bindings were chosen to resemble
0200: @kbd{C-a} and @kbd{C-e}, which move to the beginning and end of a
0201: line.  Unlike them, @kbd{M-a} and @kbd{M-e} move over successive
0202: sentences if repeated.
0203: 
0204:   Moving backward over a sentence places point just before the first
0205: character of the sentence; moving forward places point right after the
0206: punctuation that ends the sentence.  Neither one moves over the
0207: whitespace at the sentence boundary.
0208: 
0209: @kindex M-k
0210: @findex kill-sentence
0211:   Just as @kbd{C-a} and @kbd{C-e} have a kill command, @kbd{C-k}, to
0212: go with them, @kbd{M-a} and @kbd{M-e} have a corresponding kill
0213: command: @kbd{M-k} (@code{kill-sentence}) kills from point to the end
0214: of the sentence.  With a positive numeric argument @var{n}, it kills
0215: the next @var{n} sentences; with a negative argument @minus{}@var{n},
0216: it kills back to the beginning of the @var{n}th preceding sentence.
0217: 
0218: @kindex C-x DEL
0219: @findex backward-kill-sentence
0220:   The @kbd{C-x @key{DEL}} (@code{backward-kill-sentence}) kills back
0221: to the beginning of a sentence.
0222: 
0223:   The sentence commands assume that you follow the American typist's
0224: convention of putting two spaces at the end of a sentence.  That is, a
0225: sentence ends wherever there is a @samp{.}, @samp{?} or @samp{!}
0226: followed by the end of a line or two spaces, with any number of
0227: @samp{)}, @samp{]}, @samp{'}, or @samp{"} characters allowed in
0228: between.  A sentence also begins or ends wherever a paragraph begins
0229: or ends.  It is useful to follow this convention, because it allows
0230: the Emacs sentence commands to distinguish between periods that end a
0231: sentence and periods that indicate abbreviations.
0232: 
0233: @vindex sentence-end-double-space
0234:   If you want to use just one space between sentences, you can set the
0235: variable @code{sentence-end-double-space} to @code{nil} to make the
0236: sentence commands stop for single spaces.  However, this has a
0237: drawback: there is no way to distinguish between periods that end
0238: sentences and those that indicate abbreviations.  For convenient and
0239: reliable editing, we therefore recommend you follow the two-space
0240: convention.  The variable @code{sentence-end-double-space} also
0241: affects filling (@pxref{Fill Commands}).
0242: 
0243: @vindex sentence-end
0244:   The variable @code{sentence-end} controls how to recognize the end
0245: of a sentence.  If non-@code{nil}, its value should be a regular
0246: expression, which is used to match the last few characters of a
0247: sentence, together with the whitespace following the sentence
0248: (@pxref{Regexps}).  If the value is @code{nil}, the default, then
0249: Emacs computes sentence ends according to various criteria such as the
0250: value of @code{sentence-end-double-space}.
0251: 
0252: @vindex sentence-end-without-period
0253:   Some languages, such as Thai, do not use periods to indicate the end
0254: of a sentence.  Set the variable @code{sentence-end-without-period} to
0255: @code{t} in such cases.

0257: @node Paragraphs
0258: @section Paragraphs
0259: @cindex paragraphs
0260: @cindex manipulating paragraphs
0261: 
0262:   The Emacs commands for manipulating paragraphs are also on Meta keys.
0263: 
0264: @table @kbd
0265: @item M-@{
0266: Move back to previous paragraph beginning (@code{backward-paragraph}).
0267: @item M-@}
0268: Move forward to next paragraph end (@code{forward-paragraph}).
0269: @item M-h
0270: Put point and mark around this or next paragraph (@code{mark-paragraph}).
0271: @end table
0272: 
0273: @kindex M-@{
0274: @kindex M-@}
0275: @findex backward-paragraph
0276: @findex forward-paragraph
0277:   @kbd{M-@{} (@code{backward-paragraph}) moves to the beginning of the
0278: current or previous paragraph (see below for the definition of a
0279: paragraph).  @kbd{M-@}} (@code{forward-paragraph}) moves to the end of
0280: the current or next paragraph.  If there is a blank line before the
0281: paragraph, @kbd{M-@{} moves to the blank line.
0282: 
0283: @kindex M-h
0284: @findex mark-paragraph
0285:   When you wish to operate on a paragraph, type @kbd{M-h}
0286: (@code{mark-paragraph}) to set the region around it.  For example,
0287: @kbd{M-h C-w} kills the paragraph around or after point.  @kbd{M-h}
0288: puts point at the beginning and mark at the end of the paragraph point
0289: was in.  If point is between paragraphs (in a run of blank lines, or
0290: at a boundary), @kbd{M-h} sets the region around the paragraph
0291: following point.  If there are blank lines preceding the first line of
0292: the paragraph, one of these blank lines is included in the region.  If
0293: the region is already active, the command sets the mark without
0294: changing point, and each subsequent @kbd{M-h} further advances the
0295: mark by one paragraph.
0296: 
0297:   The definition of a paragraph depends on the major mode.  In
0298: Fundamental mode, as well as Text mode and related modes, a paragraph
0299: is separated from neighboring paragraphs by one or more
0300: @dfn{blank lines}---lines that are either empty, or consist solely of
0301: space, tab and/or formfeed characters.  In programming language modes,
0302: paragraphs are usually defined in a similar way, so that you can use
0303: the paragraph commands even though there are no paragraphs as such in
0304: a program.
0305: 
0306:   Note that an indented line is @emph{not} itself a paragraph break in
0307: Text mode.  If you want indented lines to separate paragraphs, use
0308: Paragraph-Indent Text mode instead.  @xref{Text Mode}.
0309: 
0310:   If you set a fill prefix, then paragraphs are delimited by all lines
0311: which don't start with the fill prefix.  @xref{Filling}.
0312: 
0313: @vindex paragraph-start
0314: @vindex paragraph-separate
0315:   The precise definition of a paragraph boundary is controlled by the
0316: variables @code{paragraph-separate} and @code{paragraph-start}.  The
0317: value of @code{paragraph-start} is a regular expression that should
0318: match lines that either start or separate paragraphs
0319: (@pxref{Regexps}).  The value of @code{paragraph-separate} is another
0320: regular expression that should match lines that separate paragraphs
0321: without being part of any paragraph (for example, blank lines).  Lines
0322: that start a new paragraph and are contained in it must match only
0323: @code{paragraph-start}, not @code{paragraph-separate}.  For example,
0324: in Fundamental mode, @code{paragraph-start} is @w{@code{"\f\\|[
0325: \t]*$"}}, and @code{paragraph-separate} is @w{@code{"[ \t\f]*$"}}.

0327: @node Pages
0328: @section Pages
0329: 
0330: @cindex pages
0331: @cindex formfeed character
0332:   Within some text files, text is divided into @dfn{pages} delimited
0333: by the @dfn{formfeed character} (@acronym{ASCII} code 12, also denoted
0334: as @samp{control-L}), which is displayed in Emacs as the escape
0335: sequence @samp{^L} (@pxref{Text Display}).  Traditionally, when such
0336: text files are printed to hardcopy, each formfeed character forces a
0337: page break.  Most Emacs commands treat it just like any other
0338: character, so you can insert it with @kbd{C-q C-l}, delete it with
0339: @key{DEL}, etc.  In addition, Emacs provides commands to move over
0340: pages and operate on them.
0341: 
0342: @table @kbd
0343: @item M-x what-page
0344: Display the page number of point, and the line number within that page.
0345: @item C-x [
0346: Move point to previous page boundary (@code{backward-page}).
0347: @item C-x ]
0348: Move point to next page boundary (@code{forward-page}).
0349: @item C-x C-p
0350: Put point and mark around this page (or another page) (@code{mark-page}).
0351: @item C-x l
0352: Count the lines in this page (@code{count-lines-page}).
0353: @end table
0354: 
0355: @findex what-page
0356:   @kbd{M-x what-page} counts pages from the beginning of the file, and
0357: counts lines within the page, showing both numbers in the echo area.
0358: 
0359: @kindex C-x [
0360: @kindex C-x ]
0361: @findex forward-page
0362: @findex backward-page
0363:   The @kbd{C-x [} (@code{backward-page}) command moves point to immediately
0364: after the previous page delimiter.  If point is already right after a page
0365: delimiter, it skips that one and stops at the previous one.  A numeric
0366: argument serves as a repeat count.  The @kbd{C-x ]} (@code{forward-page})
0367: command moves forward past the next page delimiter.
0368: 
0369: @kindex C-x C-p
0370: @findex mark-page
0371:   The @kbd{C-x C-p} command (@code{mark-page}) puts point at the
0372: beginning of the current page (after that page delimiter at the
0373: front), and the mark at the end of the page (after the page delimiter
0374: at the end).
0375: 
0376:   @kbd{C-x C-p C-w} is a handy way to kill a page to move it
0377: elsewhere.  If you move to another page delimiter with @kbd{C-x [} and
0378: @kbd{C-x ]}, then yank the killed page, all the pages will be properly
0379: delimited once again.  The reason @kbd{C-x C-p} includes only the
0380: following page delimiter in the region is to ensure that.
0381: 
0382:   A numeric argument to @kbd{C-x C-p} specifies which page to go to,
0383: relative to the current one.  Zero means the current page, one
0384: the next page, and @minus{}1 the previous one.
0385: 
0386: @kindex C-x l
0387: @findex count-lines-page
0388:   The @kbd{C-x l} command (@code{count-lines-page}) is good for deciding
0389: where to break a page in two.  It displays in the echo area the total number
0390: of lines in the current page, and then divides it up into those preceding
0391: the current line and those following, as in
0392: 
0393: @example
0394: Page has 96 (72+25) lines
0395: @end example
0396: 
0397: @noindent
0398:   Notice that the sum is off by one; this is correct if point is not at the
0399: beginning of a line.
0400: 
0401: @vindex page-delimiter
0402:   The variable @code{page-delimiter} controls where pages begin.  Its
0403: value is a regular expression that matches the beginning of a line
0404: that separates pages (@pxref{Regexps}).  The normal value of this
0405: variable is @code{"^\f"}, which matches a formfeed character at the
0406: beginning of a line.

0408: @node Quotation Marks
0409: @section Quotation Marks
0410: @cindex Quotation marks
0411: @cindex Electric Quote mode
0412: @cindex mode, Electric Quote
0413: @cindex curly quotes
0414: @cindex curved quotes
0415: @cindex guillemets
0416: @findex electric-quote-mode
0417:   One common way to quote is the typewriter convention, which quotes
0418: using straight apostrophes @t{'like this'} or double-quotes @t{"like
0419: this"}.  Another common way is the curved quote convention, which uses
0420: left and right single or double quotation marks @t{‘like this’} or
0421: @t{“like this”}.  In text files, typewriter quotes are simple and
0422: portable; curved quotes are less ambiguous and typically look nicer.
0423: 
0424: @vindex electric-quote-chars
0425:   Electric Quote mode makes it easier to type curved quotes.  As you
0426: type characters it optionally converts @t{`} to @t{‘}, @t{'} to @t{’},
0427: @t{``} to @t{“}, and @t{''} to @t{”}.  It's possible to change the
0428: default quotes listed above, by customizing the variable
0429: @code{electric-quote-chars}, a list of four characters, where the
0430: items correspond to the left single quote, the right single quote, the
0431: left double quote and the right double quote, respectively, whose
0432: default value is @code{'(?‘ ?’ ?“ ?”)}.
0433: 
0434: @vindex electric-quote-paragraph
0435: @vindex electric-quote-comment
0436: @vindex electric-quote-string
0437:   You can customize the behavior of Electric Quote mode by customizing
0438: variables that control where it is active.  It is active in text
0439: paragraphs if @code{electric-quote-paragraph} is non-@code{nil}, in
0440: programming-language comments if @code{electric-quote-comment} is
0441: non-@code{nil}, and in programming-language strings if
0442: @code{electric-quote-string} is non-@code{nil}.  The default is
0443: @code{nil} for @code{electric-quote-string} and @code{t} for the other
0444: variables.
0445: 
0446:   Electric Quote mode is disabled by default.  To toggle it, type
0447: @kbd{M-x electric-quote-mode}.  To toggle it in a single buffer, use
0448: @kbd{M-x electric-quote-local-mode}.  To suppress it for a single use,
0449: type @kbd{C-q `} or @kbd{C-q '} instead of @kbd{`} or @kbd{'}.  To
0450: insert a curved quote even when Electric Quote is disabled or
0451: inactive, you can type @kbd{C-x 8 [} for @t{‘}, @kbd{C-x 8 ]} for
0452: @t{’}, @kbd{C-x 8 @{} for @t{“}, and @kbd{C-x 8 @}} for @t{”}.
0453: @xref{Inserting Text}.  Note that the value of
0454: @code{electric-quote-chars} does not affect these keybindings, they
0455: are not keybindings of @code{electric-quote-mode} but bound in
0456: @code{global-map}.

0462:   @dfn{Filling} text means breaking it up into lines that fit a
0463: specified width.  Emacs does filling in two ways.  In Auto Fill mode,

0475: @node Auto Fill
0476: @subsection Auto Fill Mode
0477: @cindex Auto Fill mode
0478: @cindex mode, Auto Fill
0479: 
0480:   @dfn{Auto Fill} mode is a buffer-local minor mode (@pxref{Minor
0481: Modes}) in which lines are broken automatically when they become too
0482: wide.  Breaking happens only when you type a @key{SPC} or @key{RET}.
0483: 
0484: @table @kbd
0485: @item M-x auto-fill-mode
0486: Enable or disable Auto Fill mode.
0487: @item @key{SPC}
0488: @itemx @key{RET}
0489: In Auto Fill mode, break lines when appropriate.
0490: @end table
0491: 
0492: @findex auto-fill-mode
0493:   The mode command @kbd{M-x auto-fill-mode} toggles Auto Fill mode in
0494: the current buffer.  With a positive numeric argument, it enables Auto
0495: Fill mode, and with a negative argument it disables it.  If
0496: @code{auto-fill-mode} is called from Lisp with an omitted or
0497: @code{nil} argument, it enables Auto Fill mode.  To enable Auto Fill
0498: mode automatically in certain major modes, add @code{auto-fill-mode}
0499: to the mode hooks (@pxref{Major Modes}).  When Auto Fill mode is
0500: enabled, the mode indicator @samp{Fill} appears in the mode line
0501: (@pxref{Mode Line}).
0502: 
0503:   Auto Fill mode breaks lines automatically at spaces whenever they
0504: get longer than the desired width.  This line breaking occurs only
0505: when you type @key{SPC} or @key{RET}.  If you wish to insert a space
0506: or newline without permitting line-breaking, type @kbd{C-q @key{SPC}}
0507: or @kbd{C-q C-j} respectively.  Also, @kbd{C-o} inserts a newline

0546: @kindex M-q
0547: @findex fill-paragraph
0548:   The command @kbd{M-q} (@code{fill-paragraph}) @dfn{fills} the
0549: current paragraph.  It redistributes the line breaks within the
0550: paragraph, and deletes any excess space and tab characters occurring
0551: within the paragraph, in such a way that the lines end up fitting
0552: within a certain maximum width.
0553: 
0554: @findex fill-region
0555:   Normally, @kbd{M-q} acts on the paragraph where point is, but if
0556: point is between paragraphs, it acts on the paragraph after point.  If
0557: the region is active, it acts instead on the text in the region.  You
0558: can also call @kbd{M-x fill-region} to specifically fill the text in
0559: the region.
0560: 
0561: @findex fill-region-as-paragraph
0562:   @kbd{M-q} and @code{fill-region} use the usual Emacs criteria for
0563: finding paragraph boundaries (@pxref{Paragraphs}).  For more control,
0564: you can use @kbd{M-x fill-region-as-paragraph}, which refills
0565: everything between point and mark as a single paragraph.  This command
0566: deletes any blank lines within the region, so separate blocks of text
0567: end up combined into one block.

0569: @cindex justification
0570:   A numeric argument to @kbd{M-q} tells it to @dfn{justify} the text
0571: as well as filling it.  This means that extra spaces are inserted to
0572: make the right margin line up exactly at the fill column.  To remove
0573: the extra spaces, use @kbd{M-q} with no argument.  (Likewise for
0574: @code{fill-region}.)

0576: @vindex fill-column
0577: @kindex C-x f
0578: @findex set-fill-column
0579:   The maximum line width for filling is specified by the buffer-local
0580: variable @code{fill-column}.  The default value (@pxref{Locals}) is
0581: 70.  The easiest way to set @code{fill-column} in the current buffer
0582: is to use the command @kbd{C-x f} (@code{set-fill-column}).  With a
0583: numeric argument, it uses that as the new fill column.  With just
0584: @kbd{C-u} as argument, it sets @code{fill-column} to the current
0585: horizontal position of point.

0587: @kindex M-o M-s @r{(Text mode)}
0588: @cindex centering
0589: @findex center-line
0590:   The command @kbd{M-o M-s} (@code{center-line}) centers the current line
0591: within the current fill column.  With an argument @var{n}, it centers
0592: @var{n} lines individually and moves past them.  This binding is
0593: made by Text mode and is available only in that and related modes
0594: (@pxref{Text Mode}).
0595: 
0596:   By default, Emacs considers a period followed by two spaces or by a
0597: newline as the end of a sentence; a period followed by just one space
0598: indicates an abbreviation, not the end of a sentence.  Accordingly,
0599: the fill commands will not break a line after a period followed by
0600: just one space.  If you set the variable
0601: @code{sentence-end-double-space} to @code{nil}, the fill commands will
0602: break a line after a period followed by one space, and put just one
0603: space after each period.  @xref{Sentences}, for other effects and
0604: possible drawbacks of this.

0606: @vindex colon-double-space
0607:   If the variable @code{colon-double-space} is non-@code{nil}, the
0608: fill commands put two spaces after a colon.

0610: @vindex fill-nobreak-predicate
0611:   To specify additional conditions where line-breaking is not allowed,
0612: customize the abnormal hook variable @code{fill-nobreak-predicate}

0616: break the line there.  Functions you can use there include:
0617: @code{fill-single-word-nobreak-p} (don't break after the first word of
0618: a sentence or before the last); @code{fill-single-char-nobreak-p}
0619: (don't break after a one-letter word); and @code{fill-french-nobreak-p}
0620: (don't break after @samp{(} or before @samp{)}, @samp{:} or @samp{?}).

0622: @node Fill Prefix
0623: @subsection The Fill Prefix
0624: 
0625: @cindex fill prefix
0626:   The @dfn{fill prefix} feature allows paragraphs to be filled so that
0627: each line starts with a special string of characters (such as a
0628: sequence of spaces, giving an indented paragraph).  You can specify a
0629: fill prefix explicitly; otherwise, Emacs tries to deduce one
0630: automatically (@pxref{Adaptive Fill}).
0631: 
0632: @table @kbd
0633: @item C-x .
0634: Set the fill prefix (@code{set-fill-prefix}).
0635: @item M-q
0636: Fill a paragraph using current fill prefix (@code{fill-paragraph}).
0637: @item M-x fill-individual-paragraphs
0638: Fill the region, considering each change of indentation as starting a
0639: new paragraph.
0640: @item M-x fill-nonuniform-paragraphs
0641: Fill the region, considering only paragraph-separator lines as starting
0642: a new paragraph.
0643: @end table
0644: 
0645: @kindex C-x .
0646: @findex set-fill-prefix
0647:   To specify a fill prefix for the current buffer, move to a line that
0648: starts with the desired prefix, put point at the end of the prefix,
0649: and type @w{@kbd{C-x .}}@: (@code{set-fill-prefix}).  (That's a period
0650: after the @kbd{C-x}.)  To turn off the fill prefix, specify an empty
0651: prefix: type @w{@kbd{C-x .}}@: with point at the beginning of a line.
0652: 
0653:   When a fill prefix is in effect, the fill commands remove the fill
0654: prefix from each line of the paragraph before filling, and insert it
0655: on each line after filling.  (The beginning of the first line of the
0656: paragraph is left unchanged, since often that is intentionally
0657: different.)  Auto Fill mode also inserts the fill prefix automatically
0658: when it makes a new line (@pxref{Auto Fill}).  The @kbd{C-o} command
0659: inserts the fill prefix on new lines it creates, when you use it at
0660: the beginning of a line (@pxref{Blank Lines}).  Conversely, the
0661: command @kbd{M-^} deletes the prefix (if it occurs) after the newline
0662: that it deletes (@pxref{Indentation}).
0663: 
0664:   For example, if @code{fill-column} is 40 and you set the fill prefix
0665: to @samp{;; }, then @kbd{M-q} in the following text
0666: 
0667: @example
0668: ;; This is an
0669: ;; example of a paragraph
0670: ;; inside a Lisp-style comment.
0671: @end example
0672: 
0673: @noindent
0674: produces this:
0675: 
0676: @example
0677: ;; This is an example of a paragraph
0678: ;; inside a Lisp-style comment.
0679: @end example
0680: 
0681:   Lines that do not start with the fill prefix are considered to start
0682: paragraphs, both in @kbd{M-q} and the paragraph commands; this gives
0683: good results for paragraphs with hanging indentation (every line
0684: indented except the first one).  Lines which are blank or indented once
0685: the prefix is removed also separate or start paragraphs; this is what
0686: you want if you are writing multi-paragraph comments with a comment
0687: delimiter on each line.
0688: 
0689: @findex fill-individual-paragraphs
0690:   You can use @kbd{M-x fill-individual-paragraphs} to set the fill
0691: prefix for each paragraph automatically.  This command divides the
0692: region into paragraphs, treating every change in the amount of
0693: indentation as the start of a new paragraph, and fills each of these
0694: paragraphs.  Thus, all the lines in one paragraph have the same
0695: amount of indentation.  That indentation serves as the fill prefix for
0696: that paragraph.
0697: 
0698: @findex fill-nonuniform-paragraphs
0699:   @kbd{M-x fill-nonuniform-paragraphs} is a similar command that divides
0700: the region into paragraphs in a different way.  It considers only
0701: paragraph-separating lines (as defined by @code{paragraph-separate}) as
0702: starting a new paragraph.  Since this means that the lines of one
0703: paragraph may have different amounts of indentation, the fill prefix
0704: used is the smallest amount of indentation of any of the lines of the
0705: paragraph.  This gives good results with styles that indent a paragraph's
0706: first line more or less that the rest of the paragraph.
0707: 
0708: @vindex fill-prefix
0709:   The fill prefix is stored in the variable @code{fill-prefix}.  Its value
0710: is a string, or @code{nil} when there is no fill prefix.  This is a
0711: per-buffer variable; altering the variable affects only the current buffer,
0712: but there is a default value which you can change as well.  @xref{Locals}.
0713: 
0714:   The @code{indentation} text property provides another way to control
0715: the amount of indentation paragraphs receive.  @xref{Enriched
0716: Indentation}.

0718: @node Adaptive Fill
0719: @subsection Adaptive Filling
0720: 
0721: @cindex adaptive filling
0722:   The fill commands can deduce the proper fill prefix for a paragraph
0723: automatically in certain cases: either whitespace or certain punctuation
0724: characters at the beginning of a line are propagated to all lines of the
0725: paragraph.
0726: 
0727:   If the paragraph has two or more lines, the fill prefix is taken from
0728: the paragraph's second line, but only if it appears on the first line as
0729: well.
0730: 
0731:   If a paragraph has just one line, fill commands @emph{may} take a
0732: prefix from that line.  The decision is complicated because there are
0733: three reasonable things to do in such a case:
0734: 
0735: @itemize @bullet
0736: @item
0737: Use the first line's prefix on all the lines of the paragraph.
0738: 
0739: @item
0740: Indent subsequent lines with whitespace, so that they line up under the
0741: text that follows the prefix on the first line, but don't actually copy
0742: the prefix from the first line.
0743: 
0744: @item
0745: Don't do anything special with the second and following lines.
0746: @end itemize
0747: 
0748:   All three of these styles of formatting are commonly used.  So the
0749: fill commands try to determine what you would like, based on the prefix
0750: that appears and on the major mode.  Here is how.
0751: 
0752: @vindex adaptive-fill-first-line-regexp
0753:   If the prefix found on the first line matches
0754: @code{adaptive-fill-first-line-regexp}, or if it appears to be a
0755: comment-starting sequence (this depends on the major mode), then the
0756: prefix found is used for filling the paragraph, provided it would not
0757: act as a paragraph starter on subsequent lines.
0758: 
0759:   Otherwise, the prefix found is converted to an equivalent number of
0760: spaces, and those spaces are used as the fill prefix for the rest of the
0761: lines, provided they would not act as a paragraph starter on subsequent
0762: lines.
0763: 
0764:   In Text mode, and other modes where only blank lines and page
0765: delimiters separate paragraphs, the prefix chosen by adaptive filling
0766: never acts as a paragraph starter, so it can always be used for filling.
0767: 
0768: @vindex adaptive-fill-mode
0769: @vindex adaptive-fill-regexp
0770:   The variable @code{adaptive-fill-regexp} determines what kinds of line
0771: beginnings can serve as a fill prefix: any characters at the start of
0772: the line that match this regular expression are used.  If you set the
0773: variable @code{adaptive-fill-mode} to @code{nil}, the fill prefix is
0774: never chosen automatically.
0775: 
0776: @vindex adaptive-fill-function
0777:   You can specify more complex ways of choosing a fill prefix
0778: automatically by setting the variable @code{adaptive-fill-function} to a
0779: function.  This function is called with point after the left margin of a
0780: line, and it should return the appropriate fill prefix based on that
0781: line.  If it returns @code{nil}, @code{adaptive-fill-regexp} gets
0782: a chance to find a prefix.

0939: line really kills all the following invisible text associated with the

0964: @node Outline Format
0965: @subsection Format of Outlines
0966: 
0967: @cindex heading lines (Outline mode)
0968: @cindex body lines (Outline mode)
0969:   Outline mode assumes that the lines in the buffer are of two types:
0970: @dfn{heading lines} and @dfn{body lines}.  A heading line represents a
0971: topic in the outline.  Heading lines start with one or more asterisk
0972: (@samp{*}) characters; the number of asterisks determines the depth of
0973: the heading in the outline structure.  Thus, a heading line with one
0974: @samp{*} is a major topic; all the heading lines with two @samp{*}s
0975: between it and the next one-@samp{*} heading are its subtopics; and so
0976: on.  Any line that is not a heading line is a body line.  Body lines
0977: belong with the preceding heading line.  Here is an example:
0978: 
0979: @example
0980: * Food
0981: This is the body,
0982: which says something about the topic of food.
0983: 
0984: ** Delicious Food
0985: This is the body of the second-level header.
0986: 
0987: ** Distasteful Food
0988: This could have
0989: a body too, with
0990: several lines.
0991: 
0992: *** Dormitory Food
0993: 
0994: * Shelter
0995: Another first-level topic with its header line.
0996: @end example
0997: 
0998:   A heading line together with all following body lines is called
0999: collectively an @dfn{entry}.  A heading line together with all following
1000: deeper heading lines and their body lines is called a @dfn{subtree}.
1001: 
1002: @vindex outline-regexp
1003:   You can customize the criterion for distinguishing heading lines by
1004: setting the variable @code{outline-regexp}.  (The recommended ways to
1005: do this are in a major mode function or with a file local variable.)
1006: Any line whose beginning has a match for this regexp is considered a
1007: heading line.  Matches that start within a line (not at the left
1008: margin) do not count.
1009: 
1010:   The length of the matching text determines the level of the heading;
1011: longer matches make a more deeply nested level.  Thus, for example, if
1012: a text formatter has commands @samp{@@chapter}, @samp{@@section} and
1013: @samp{@@subsection} to divide the document into chapters and sections,
1014: you could make those lines count as heading lines by setting
1015: @code{outline-regexp} to @samp{"@@chap\\|@@\\(sub\\)*section"}.  Note
1016: the trick: the two words @samp{chapter} and @samp{section} are equally
1017: long, but by defining the regexp to match only @samp{chap} we ensure
1018: that the length of the text matched on a chapter heading is shorter,
1019: so that Outline mode will know that sections are contained in
1020: chapters.  This works as long as no other command starts with
1021: @samp{@@chap}.
1022: 
1023: @vindex outline-level
1024:   You can explicitly specify a rule for calculating the level of a
1025: heading line by setting the variable @code{outline-level}.  The value
1026: of @code{outline-level} should be a function that takes no arguments
1027: and returns the level of the current heading.  The recommended ways to
1028: set this variable are in a major mode command or with a file local
1029: variable.

1080: revealing parts of the buffer, based on the outline structure.  These
1081: commands are not undoable; their effects are simply not recorded by
1082: the undo mechanism, so you can undo right past them (@pxref{Undo}).

1085: point is on a heading line, that is the current heading line; if point
1086: is on a body line, the current heading line is the nearest preceding
1087: header line.

1092: (@code{outline-hide-entry}).
1093: @item C-c C-e

1095: (@code{outline-show-entry}).

1098: heading itself (@code{outline-hide-subtree}).
1099: @item C-c C-s
1100: Make everything under the current heading visible, including body,
1101: subheadings, and their bodies (@code{outline-show-subtree}).
1102: @item C-c C-l
1103: Make the body of the current heading line, and of all its subheadings,
1104: invisible (@code{outline-hide-leaves}).
1105: @item C-c C-k
1106: Make all subheadings of the current heading line, at all levels,
1107: visible (@code{outline-show-branches}).
1108: @item C-c C-i
1109: Make immediate subheadings (one level down) of the current heading
1110: line visible (@code{outline-show-children}).

1165: level headings.  It also reveals body lines preceding the first
1166: heading in the buffer.

1220: @node Foldout
1221: @subsection Folding Editing
1222: 
1223: @cindex folding editing
1224:   The Foldout package extends Outline mode and Outline minor mode with
1225: folding commands.  The idea of folding is that you zoom in on a
1226: nested portion of the outline, while hiding its relatives at higher
1227: levels.
1228: 
1229:   Consider an Outline mode buffer with all the text and subheadings under
1230: level-1 headings hidden.  To look at what is hidden under one of these
1231: headings, you could use @kbd{C-c C-e} (@kbd{M-x outline-show-entry})
1232: to expose the body, or @kbd{C-c C-i} to expose the child (level-2)
1233: headings.
1234: 
1235: @kindex C-c C-z
1236: @findex foldout-zoom-subtree
1237:   With Foldout, you use @kbd{C-c C-z} (@kbd{M-x foldout-zoom-subtree}).
1238: This exposes the body and child subheadings, and narrows the buffer so
1239: that only the @w{level-1} heading, the body and the level-2 headings are
1240: visible.  Now to look under one of the level-2 headings, position the
1241: cursor on it and use @kbd{C-c C-z} again.  This exposes the level-2 body
1242: and its level-3 child subheadings and narrows the buffer again.  Zooming
1243: in on successive subheadings can be done as much as you like.  A string
1244: in the mode line shows how deep you've gone.
1245: 
1246:   When zooming in on a heading, to see only the child subheadings specify
1247: a numeric argument: @kbd{C-u C-c C-z}.  The number of levels of children
1248: can be specified too (compare @kbd{M-x outline-show-children}), e.g.,
1249: @w{@kbd{M-2 C-c C-z}} exposes two levels of child subheadings.
1250: Alternatively, the body can be specified with a negative argument:
1251: @w{@kbd{M-- C-c C-z}}.  The whole subtree can be expanded, similarly to
1252: @kbd{C-c C-s} (@kbd{M-x outline-show-subtree}), by specifying a zero
1253: argument: @w{@kbd{M-0 C-c C-z}}.
1254: 
1255:   While you're zoomed in, you can still use Outline mode's exposure and
1256: hiding functions without disturbing Foldout.  Also, since the buffer is
1257: narrowed, global editing actions will only affect text under the
1258: zoomed-in heading.  This is useful for restricting changes to a
1259: particular chapter or section of your document.
1260: 
1261: @kindex C-c C-x
1262: @findex foldout-exit-fold
1263:   To unzoom (exit) a fold, use @kbd{C-c C-x} (@kbd{M-x foldout-exit-fold}).
1264: This hides all the text and subheadings under the top-level heading and
1265: returns you to the previous view of the buffer.  Specifying a numeric
1266: argument exits that many levels of folds.  Specifying a zero argument
1267: exits all folds.
1268: 
1269:   To cancel the narrowing of a fold without hiding the text and
1270: subheadings, specify a negative argument.  For example, @w{@kbd{M--2 C-c
1271: C-x}} exits two folds and leaves the text and subheadings exposed.
1272:

1355: executed.

1360: buffer cycles the visibility of the entire outline structure, between
1361: (i) showing only top-level heading lines, (ii) showing all heading
1362: lines but no body lines, and (iii) showing everything.

1492:   Emacs provides special major modes for editing files written in
1493: @TeX{} and its related formats.  @TeX{} is a powerful text formatter
1494: written by Donald Knuth; like GNU Emacs, it is free software.
1495: @LaTeX{} is a simplified input format for @TeX{}, implemented using
1496: @TeX{} macros.  Doc@TeX{} is a special file format in which the
1497: @LaTeX{} sources are written, combining sources with documentation.

1503:   @TeX{} mode has four variants: Plain @TeX{} mode, @LaTeX{} mode,
1504: Doc@TeX{} mode, and Sli@TeX{} mode.  These distinct major modes differ
1505: only slightly, and are designed for editing the four different
1506: formats.  Emacs selects the appropriate mode by looking at the
1507: contents of the buffer.  (This is done by the @code{tex-mode} command,
1508: which is normally called automatically when you visit a @TeX{}-like
1509: file.  @xref{Choosing Modes}.)  If the contents are insufficient to
1510: determine this, Emacs chooses the mode specified by the variable
1511: @code{tex-default-mode}; its default value is @code{latex-mode}.  If
1512: Emacs does not guess right, you can select the correct variant of
1513: @TeX{} mode using the command @kbd{M-x plain-tex-mode}, @kbd{M-x
1514: latex-mode}, @kbd{M-x slitex-mode}, or @kbd{doctex-mode}.

1579: @findex tex-insert-quote
1580: @kindex " @r{(@TeX{} mode)}
1581:   In @TeX{}, the character @samp{"} is not normally used; instead,
1582: quotations begin with @samp{``} and end with @samp{''}.  @TeX{} mode
1583: therefore binds the @kbd{"} key to the @code{tex-insert-quote}
1584: command.  This inserts @samp{``} after whitespace or an open brace,
1585: @samp{"} after a backslash, and @samp{''} after any other character.
1586: 
1587:   As a special exception, if you type @kbd{"} when the text before
1588: point is either @samp{``} or @samp{''}, Emacs replaces that preceding
1589: text with a single @samp{"} character.  You can therefore type
1590: @kbd{""} to insert @samp{"}, should you ever need to do so.  (You can
1591: also use @kbd{C-q "} to insert this character.)
1592: 
1593:   In @TeX{} mode, @samp{$} has a special syntax code which attempts to
1594: understand the way @TeX{} math mode delimiters match.  When you insert a
1595: @samp{$} that is meant to exit math mode, the position of the matching
1596: @samp{$} that entered math mode is displayed for a second.  This is the
1597: same feature that displays the open brace that matches a close brace that
1598: is inserted.  However, there is no way to tell whether a @samp{$} enters
1599: math mode or leaves it; so when you insert a @samp{$} that enters math
1600: mode, the previous @samp{$} position is shown as if it were a match, even
1601: though they are actually unrelated.
1602: 
1603: @findex tex-insert-braces
1604: @kindex C-c @{ @r{(@TeX{} mode)}
1605: @findex up-list
1606: @kindex C-c @} @r{(@TeX{} mode)}
1607:   @TeX{} uses braces as delimiters that must match.  Some users prefer
1608: to keep braces balanced at all times, rather than inserting them
1609: singly.  Use @kbd{C-c @{} (@code{tex-insert-braces}) to insert a pair of
1610: braces.  It leaves point between the two braces so you can insert the
1611: text that belongs inside.  Afterward, use the command @kbd{C-c @}}
1612: (@code{up-list}) to move forward past the close brace.
1613: 
1614: @findex tex-validate-region
1615: @findex tex-terminate-paragraph
1616: @kindex C-j @r{(@TeX{} mode)}
1617:   There are two commands for checking the matching of braces.
1618: @kbd{C-j} (@code{tex-terminate-paragraph}) checks the paragraph before
1619: point, and inserts two newlines to start a new paragraph.  It outputs
1620: a message in the echo area if any mismatch is found.  @kbd{M-x
1621: tex-validate-region} checks a region, paragraph by paragraph.  The
1622: errors are listed in an @file{*Occur*} buffer; you can use the usual
1623: Occur mode commands in that buffer, such as @kbd{C-c C-c}, to visit a
1624: particular mismatch (@pxref{Other Repeating Search}).
1625: 
1626:   Note that Emacs commands count square brackets and parentheses in
1627: @TeX{} mode, not just braces.  This is not strictly correct for the
1628: purpose of checking @TeX{} syntax.  However, parentheses and square
1629: brackets are likely to be used in text as matching delimiters, and it
1630: is useful for the various motion commands and automatic match display
1631: to work with them.

1711: @findex tex-buffer
1712: @kindex C-c C-b @r{(@TeX{} mode)}
1713: @findex tex-view
1714: @kindex C-c C-v @r{(@TeX{} mode)}
1715: @findex tex-print
1716: @kindex C-c C-p @r{(@TeX{} mode)}
1717:   To pass the current buffer through @TeX{}, type @kbd{C-c C-b}
1718: (@code{tex-buffer}).  The formatted output goes in a temporary file,
1719: normally a @file{.dvi} file.  Afterwards, you can type @kbd{C-c C-v}
1720: (@code{tex-view}) to launch an external program, such as
1721: @command{xdvi}, to view this output file.  You can also type @kbd{C-c
1722: C-p} (@code{tex-print}) to print a hardcopy of the output file.

1724: @cindex @env{TEXINPUTS} environment variable
1725: @vindex tex-directory
1726:   By default, @kbd{C-c C-b} runs @TeX{} in the current directory.  The
1727: output of @TeX{} also goes in this directory.  To run @TeX{} in a
1728: different directory, change the variable @code{tex-directory} to
1729: the desired directory.  If your environment variable @env{TEXINPUTS}
1730: contains relative names, or if your files contains
1731: @samp{\input} commands with relative file names, then
1732: @code{tex-directory} @emph{must} be @code{"."} or you will get the
1733: wrong results.  Otherwise, it is safe to specify some other directory,
1734: such as @code{"/tmp"}.

1736: @vindex tex-run-command
1737: @vindex latex-run-command
1738: @vindex tex-dvi-view-command
1739: @vindex tex-dvi-print-command
1740: @vindex tex-print-file-extension
1741:   The buffer's @TeX{} variant determines what shell command @kbd{C-c
1742: C-b} actually runs.  In Plain @TeX{} mode, it is specified by the
1743: variable @code{tex-run-command}, which defaults to @code{"tex"}.  In
1744: @LaTeX{} mode, it is specified by @code{latex-run-command}, which
1745: defaults to @code{"latex"}.  The shell command that @kbd{C-c C-v} runs
1746: to view the @file{.dvi} output is determined by the variable
1747: @code{tex-dvi-view-command}, regardless of the @TeX{} variant.  The
1748: shell command that @kbd{C-c C-p} runs to print the output is
1749: determined by the variable @code{tex-dvi-print-command}.  The variable
1750: @code{tex-print-file-extension} can be set to the required file
1751: extension for viewing and printing @TeX{}-compiled files.  For
1752: example, you can set it to @file{.pdf}, and update
1753: @code{tex-dvi-view-command} and @code{tex-dvi-print-command}
1754: accordingly.
1755: 
1756:   Normally, Emacs automatically appends the output file name to the
1757: shell command strings described in the preceding paragraph.  For
1758: example, if @code{tex-dvi-view-command} is @code{"xdvi"}, @kbd{C-c
1759: C-v} runs @command{xdvi @var{output-file-name}}.  In some cases,
1760: however, the file name needs to be embedded in the command, e.g., if
1761: you need to provide the file name as an argument to one command whose
1762: output is piped to another.  You can specify where to put the file
1763: name with @samp{*} in the command string.  For example,
1764:

1766: (setq tex-dvi-print-command "dvips -f * | lpr")

1770: @kindex C-c C-k @r{(@TeX{} mode)}
1771: @findex tex-recenter-output-buffer
1772: @kindex C-c C-l @r{(@TeX{} mode)}
1773:   The terminal output from @TeX{}, including any error messages,
1774: appears in a buffer called @file{*tex-shell*}.  If @TeX{} gets an
1775: error, you can switch to this buffer and feed it input (this works as
1776: in Shell mode; @pxref{Interactive Shell}).  Without switching to this
1777: buffer you can scroll it so that its last line is visible by typing
1778: @kbd{C-c C-l}.

1782: @kbd{C-c C-r} also kills any @TeX{} process still running.

1784: @findex tex-region
1785: @kindex C-c C-r @r{(@TeX{} mode)}
1786:   You can also pass an arbitrary region through @TeX{} by typing
1787: @kbd{C-c C-r} (@code{tex-region}).  This is tricky, however, because
1788: most files of @TeX{} input contain commands at the beginning to set
1789: parameters and define macros, without which no later part of the file
1790: will format correctly.  To solve this problem, @kbd{C-c C-r} allows
1791: you to designate a part of the file as containing essential commands;
1792: it is included before the specified region as part of the input to
1793: @TeX{}.  The designated part of the file is called the @dfn{header}.
1794: 
1795: @cindex header (@TeX{} mode)
1796:   To indicate the bounds of the header in Plain @TeX{} mode, you insert two
1797: special strings in the file.  Insert @samp{%**start of header} before the
1798: header, and @samp{%**end of header} after it.  Each string must appear
1799: entirely on one line, but there may be other text on the line before or
1800: after.  The lines containing the two strings are included in the header.
1801: If @samp{%**start of header} does not appear within the first 100 lines of
1802: the buffer, @kbd{C-c C-r} assumes that there is no header.
1803: 
1804:   In @LaTeX{} mode, the header begins with @samp{\documentclass} or
1805: @samp{\documentstyle} and ends with @samp{\begin@{document@}}.  These
1806: are commands that @LaTeX{} requires you to use in any case, so nothing
1807: special needs to be done to identify the header.

1809: @findex tex-file
1810: @kindex C-c C-f @r{(@TeX{} mode)}
1811:   The commands (@code{tex-buffer}) and (@code{tex-region}) do all of their
1812: work in a temporary directory, and do not have available any of the auxiliary
1813: files needed by @TeX{} for cross-references; these commands are generally
1814: not suitable for running the final copy in which all of the cross-references
1815: need to be correct.
1816: 
1817:   When you want the auxiliary files for cross references, use @kbd{C-c
1818: C-f} (@code{tex-file}) which runs @TeX{} on the current buffer's file,
1819: in that file's directory.  Before running @TeX{}, it offers to save any
1820: modified buffers.  Generally, you need to use (@code{tex-file}) twice to
1821: get the cross-references right.
1822: 
1823: @vindex tex-start-options
1824:   The value of the variable @code{tex-start-options} specifies
1825: options for the @TeX{} run.
1826: 
1827: @vindex tex-start-commands
1828:   The value of the variable @code{tex-start-commands} specifies @TeX{}
1829: commands for starting @TeX{}.  The default value causes @TeX{} to run
1830: in nonstop mode.  To run @TeX{} interactively, set the variable to
1831: @code{""}.
1832: 
1833: @vindex tex-main-file
1834:   Large @TeX{} documents are often split into several files---one main
1835: file, plus subfiles.  Running @TeX{} on a subfile typically does not
1836: work; you have to run it on the main file.  In order to make
1837: @code{tex-file} useful when you are editing a subfile, you can set the
1838: variable @code{tex-main-file} to the name of the main file.  Then
1839: @code{tex-file} runs @TeX{} on that file.
1840: 
1841:   The most convenient way to use @code{tex-main-file} is to specify it
1842: in a local variable list in each of the subfiles.  @xref{File
1843: Variables}.
1844: 
1845: @findex tex-bibtex-file
1846: @kindex C-c TAB @r{(@TeX{} mode)}
1847: @vindex tex-bibtex-command
1848:   For @LaTeX{} files, you can use Bib@TeX{} to process the auxiliary
1849: file for the current buffer's file.  Bib@TeX{} looks up bibliographic
1850: citations in a data base and prepares the cited references for the
1851: bibliography section.  The command @kbd{C-c @key{TAB}}
1852: (@code{tex-bibtex-file}) runs the shell command
1853: (@code{tex-bibtex-command}) to produce a @samp{.bbl} file for the
1854: current buffer's file.  Generally, you need to do @kbd{C-c C-f}
1855: (@code{tex-file}) once to generate the @samp{.aux} file, then do
1856: @kbd{C-c @key{TAB}} (@code{tex-bibtex-file}), and then repeat @kbd{C-c C-f}
1857: (@code{tex-file}) twice more to get the cross-references correct.

2643: @node Cell Justification
2644: @subsection Cell Justification
2645: @cindex justification in text-based tables
2646: 
2647:   The command @kbd{M-x table-justify} imposes @dfn{justification} on
2648: one or more cells in a text-based table.  Justification determines how
2649: the text in the cell is aligned, relative to the edges of the cell.
2650: Each cell in a table can be separately justified.
2651: 
2652: @findex table-justify
2653:   @kbd{M-x table-justify} first prompts for what to justify; the
2654: options are @samp{cell} (just the current cell), @samp{column} (all
2655: cells in the current table column) and @samp{row} (all cells in the
2656: current table row).  The command then prompts for the justification
2657: style; the options are @code{left}, @code{center}, @code{right},
2658: @code{top}, @code{middle}, @code{bottom}, or @code{none} (meaning no
2659: vertical justification).
2660: 
2661:   Horizontal and vertical justification styles are specified
2662: independently, and both types can be in effect simultaneously; for
2663: instance, you can call @kbd{M-x table-justify} twice, once to specify
2664: @code{right} justification and once to specify @code{bottom}
2665: justification, to align the contents of a cell to the bottom right.
2666: 
2667: @vindex table-detect-cell-alignment
2668:   The justification style is stored in the buffer as a text property,
2669: and is lost when you kill the buffer or exit Emacs.  However, the
2670: table recognition commands, such as @kbd{M-x table-recognize}
2671: (@pxref{Table Recognition}), attempt to determine and re-apply each
2672: cell's justification style, by examining its contents.  To disable
2673: this feature, change the variable @code{table-detect-cell-alignment}
2674: to @code{nil}.

2703: @node Table Conversion
2704: @subsection Converting Between Plain Text and Tables
2705: @cindex text to table
2706: @cindex table to text
2707: 
2708: @findex table-capture
2709:   The command @kbd{M-x table-capture} captures plain text in a region
2710: and turns it into a table.  Unlike @kbd{M-x table-recognize}
2711: (@pxref{Table Recognition}), the original text does not need to have a
2712: table appearance; it only needs to have a logical table-like
2713: structure.
2714: 
2715:   For example, suppose we have the following numbers, which are
2716: divided into three lines and separated horizontally by commas:
2717: 
2718: @example
2719: 1, 2, 3, 4
2720: 5, 6, 7, 8
2721: , 9, 10
2722: @end example
2723: 
2724: @noindent
2725: Invoking @kbd{M-x table-capture} on that text produces this table:
2726: 
2727: @example
2728: +-----+-----+-----+-----+
2729: |1    |2    |3    |4    |
2730: +-----+-----+-----+-----+
2731: |5    |6    |7    |8    |
2732: +-----+-----+-----+-----+
2733: |     |9    |10   |     |
2734: +-----+-----+-----+-----+
2735: @end example
2736: 
2737: @findex table-release
2738:   @kbd{M-x table-release} does the opposite: it converts a table back
2739: to plain text, removing its cell borders.
2740: 
2741:   One application of this pair of commands is to edit a text in
2742: layout.  Look at the following three paragraphs (the latter two are
2743: indented with header lines):
2744: 
2745: @example
2746: table-capture is a powerful command.
2747: Here are some things it can do:
2748: 
2749: Parse Cell Items   Using row and column delimiter regexps,
2750:                    it parses the specified text area and
2751:                    extracts cell items into a table.
2752: @end example
2753: 
2754: @noindent
2755: Applying @code{table-capture} to a region containing the above text,
2756: with empty strings for the column and row delimiter regexps, creates a
2757: table with a single cell like the following one.
2758: 
2759: @smallexample
2760: @group
2761: +----------------------------------------------------------+
2762: |table-capture is a powerful command.                      |
2763: |Here are some things it can do:                           |
2764: |                                                          |
2765: |Parse Cell Items   Using row and column delimiter regexps,|
2766: |                   it parses the specified text area and  |
2767: |                   extracts cell items into a table.      |
2768: +----------------------------------------------------------+
2769: @end group
2770: @end smallexample
2771: 
2772: @noindent
2773: We can then use the cell splitting commands (@pxref{Cell Commands}) to
2774: subdivide the table so that each paragraph occupies a cell:
2775: 
2776: @smallexample
2777: +----------------------------------------------------------+
2778: |table-capture is a powerful command.                      |
2779: |Here are some things it can do:                           |
2780: +-----------------+----------------------------------------+
2781: |Parse Cell Items | Using row and column delimiter regexps,|
2782: |                 | it parses the specified text area and  |
2783: |                 | extracts cell items into a table.      |
2784: +-----------------+----------------------------------------+
2785: @end smallexample
2786: 
2787: @noindent
2788: Each cell can now be edited independently without affecting the layout
2789: of other cells.  When finished, we can invoke @kbd{M-x table-release}
2790: to convert the table back to plain text.

2824: @node Two-Column
2825: @section Two-Column Editing
2826: @cindex two-column editing
2827: @cindex splitting columns
2828: @cindex columns, splitting
2829: 
2830:   Two-column mode lets you conveniently edit two side-by-side columns
2831: of text.  It uses two side-by-side windows, each showing its own
2832: buffer.  There are three ways to enter two-column mode:
2833: 
2834: @table @asis
2835: @item @kbd{@key{F2} 2} or @kbd{C-x 6 2}
2836: @kindex F2 2
2837: @kindex C-x 6 2
2838: @findex 2C-two-columns
2839: Enter two-column mode with the current buffer on the left, and on the
2840: right, a buffer whose name is based on the current buffer's name
2841: (@code{2C-two-columns}).  If the right-hand buffer doesn't already
2842: exist, it starts out empty; the current buffer's contents are not
2843: changed.
2844: 
2845: This command is appropriate when the current buffer is empty or contains
2846: just one column and you want to add another column.
2847: 
2848: @item @kbd{@key{F2} s} or @kbd{C-x 6 s}
2849: @kindex F2 s
2850: @kindex C-x 6 s
2851: @findex 2C-split
2852: Split the current buffer, which contains two-column text, into two
2853: buffers, and display them side by side (@code{2C-split}).  The current
2854: buffer becomes the left-hand buffer, but the text in the right-hand
2855: column is moved into the right-hand buffer.  The current column
2856: specifies the split point.  Splitting starts with the current line and
2857: continues to the end of the buffer.
2858: 
2859: This command is appropriate when you have a buffer that already contains
2860: two-column text, and you wish to separate the columns temporarily.
2861: 
2862: @item @kbd{@key{F2} b @var{buffer} @key{RET}}
2863: @itemx @kbd{C-x 6 b @var{buffer} @key{RET}}
2864: @kindex F2 b
2865: @kindex C-x 6 b
2866: @findex 2C-associate-buffer
2867: Enter two-column mode using the current buffer as the left-hand buffer,
2868: and using buffer @var{buffer} as the right-hand buffer
2869: (@code{2C-associate-buffer}).
2870: @end table
2871: 
2872:   @kbd{@key{F2} s} or @kbd{C-x 6 s} looks for a column separator, which
2873: is a string that appears on each line between the two columns.  You can
2874: specify the width of the separator with a numeric argument to
2875: @kbd{@key{F2} s}; that many characters, before point, constitute the
2876: separator string.  By default, the width is 1, so the column separator
2877: is the character before point.
2878: 
2879:   When a line has the separator at the proper place, @kbd{@key{F2} s}
2880: puts the text after the separator into the right-hand buffer, and
2881: deletes the separator.  Lines that don't have the column separator at
2882: the proper place remain unsplit; they stay in the left-hand buffer, and
2883: the right-hand buffer gets an empty line to correspond.  (This is the
2884: way to write a line that spans both columns while in two-column
2885: mode: write it in the left-hand buffer, and put an empty line in the
2886: right-hand buffer.)
2887: 
2888: @kindex F2 RET
2889: @kindex C-x 6 RET
2890: @findex 2C-newline
2891:   The command @kbd{C-x 6 @key{RET}} or @kbd{@key{F2} @key{RET}}
2892: (@code{2C-newline}) inserts a newline in each of the two buffers at
2893: corresponding positions.  This is the easiest way to add a new line to
2894: the two-column text while editing it in split buffers.
2895: 
2896: @kindex F2 1
2897: @kindex C-x 6 1
2898: @findex 2C-merge
2899:   When you have edited both buffers as you wish, merge them with
2900: @kbd{@key{F2} 1} or @kbd{C-x 6 1} (@code{2C-merge}).  This copies the
2901: text from the right-hand buffer as a second column in the other buffer.
2902: To go back to two-column editing, use @kbd{@key{F2} s}.
2903: 
2904: @kindex F2 d
2905: @kindex C-x 6 d
2906: @findex 2C-dissociate
2907:   Use @kbd{@key{F2} d} or @kbd{C-x 6 d} to dissociate the two buffers,
2908: leaving each as it stands (@code{2C-dissociate}).  If the other buffer,
2909: the one not current when you type @kbd{@key{F2} d}, is empty,
2910: @kbd{@key{F2} d} kills it.
