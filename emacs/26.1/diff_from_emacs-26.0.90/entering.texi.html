015: @node Entering Emacs
016: @section Entering Emacs
017: @cindex entering Emacs
018: @cindex starting Emacs
019: 
020:   The usual way to invoke Emacs is with the shell command
021: @command{emacs}.  From a terminal window running in the X Window
022: System, you can run Emacs in the background with @command{emacs &};
023: this way, Emacs won't tie up the terminal window, so you can use it to
024: run other shell commands.
025: 
026: @cindex startup screen
027:   When Emacs starts up, the initial frame displays a special buffer
028: named @samp{*GNU Emacs*}.  This @dfn{startup screen} contains
029: information about Emacs and @dfn{links} to common tasks that are
030: useful for beginning users.  For instance, activating the @samp{Emacs
031: Tutorial} link opens the Emacs tutorial; this does the same thing as
032: the command @kbd{C-h t} (@code{help-with-tutorial}).  To activate a
033: link, either move point onto it and type @kbd{@key{RET}}, or click on
034: it with @kbd{mouse-1} (the left mouse button).
035: 
036:   Using a command line argument, you can tell Emacs to visit one or
037: more files as soon as it starts up.  For example, @command{emacs
038: foo.txt} starts Emacs with a buffer displaying the contents of the
039: file @samp{foo.txt}.  This feature exists mainly for compatibility
040: with other editors, which are designed to be launched from the shell
041: for short editing sessions.  If you call Emacs this way, the initial
042: frame is split into two windows---one showing the specified file, and
043: the other showing the startup screen.  @xref{Windows}.
044: 
045:   Generally, it is unnecessary and wasteful to start Emacs afresh each
046: time you want to edit a file.  The recommended way to use Emacs is to
047: start it just once, just after you log in, and do all your editing in
048: the same Emacs session.  @xref{Files}, for information on visiting
049: more than one file.  If you use Emacs this way, the Emacs session
050: accumulates valuable context, such as the kill ring, registers, undo
051: history, and mark ring data, which together make editing more
052: convenient.  These features are described later in the manual.
053: 
054:   To edit a file from another program while Emacs is running, you can
055: use the @command{emacsclient} helper program to open a file in the
056: existing Emacs session.  @xref{Emacs Server}.
057: 
058:   Emacs accepts other command line arguments that tell it to load
059: certain Lisp files, where to put the initial frame, and so forth.
060: @xref{Emacs Invocation}.
061: 
062: @vindex inhibit-startup-screen
063:   If the variable @code{inhibit-startup-screen} is non-@code{nil},
064: Emacs does not display the startup screen.  In that case, if one or
065: more files were specified on the command line, Emacs simply displays
066: those files; otherwise, it displays a buffer named @file{*scratch*},
067: which can be used to evaluate Emacs Lisp expressions interactively.
068: @xref{Lisp Interaction}.  You can set the variable
069: @code{inhibit-startup-screen} using the Customize facility
070: (@pxref{Easy Customization}), or by editing your initialization file
071: (@pxref{Init File}).@footnote{Setting @code{inhibit-startup-screen} in
072: @file{site-start.el} doesn't work, because the startup screen is set
073: up before reading @file{site-start.el}.  @xref{Init File}, for
074: information about @file{site-start.el}.}
075: 
076:   You can also force Emacs to display a file or directory at startup
077: by setting the variable @code{initial-buffer-choice} to a string
078: naming that file or directory.  The value of
079: @code{initial-buffer-choice} may also be a function (of no arguments)
080: that should return a buffer which is then displayed.
081: @ignore
082: @c I do not think this should be mentioned.  AFAICS it is just a dodge
083: @c around inhibit-startup-screen not being settable on a site-wide basis.
084: @code{initial-buffer-choice} may also be @code{t} in which case the
085: @file{*scratch*} buffer will be shown.
086: @end ignore
087: If @code{initial-buffer-choice} is non-@code{nil}, then if you specify
088: any files on the command line, Emacs still visits them, but does not
089: display them initially.

091: @node Exiting
092: @section Exiting Emacs
093: @cindex exiting
094: @cindex killing Emacs
095: @cindex leaving Emacs
096: @cindex quitting Emacs
097: 
098: @table @kbd
099: @item C-x C-c
100: Kill Emacs (@code{save-buffers-kill-terminal}).
101: @item C-z
102: On a text terminal, suspend Emacs; on a graphical display,
103: iconify (or ``minimize'') the selected frame (@code{suspend-frame}).
104: @end table
105: 
106: @kindex C-x C-c
107: @findex save-buffers-kill-terminal
108:   @dfn{Killing} Emacs means terminating the Emacs program.  To do
109: this, type @kbd{C-x C-c} (@code{save-buffers-kill-terminal}).  A
110: two-character key sequence is used to make it harder to type by
111: accident.  If there are any modified file-visiting buffers when you
112: type @kbd{C-x C-c}, Emacs first offers to save these buffers.  If you
113: do not save them all, it asks for confirmation again, since the
114: unsaved changes will be lost.  Emacs also asks for confirmation if any
115: subprocesses are still running, since killing Emacs will also kill the
116: subprocesses (@pxref{Shell}).
117: 
118:   @kbd{C-x C-c} behaves specially if you are using Emacs as a server.
119: If you type it from a client frame, it closes the client
120: connection.  @xref{Emacs Server}.
121: 
122:   Emacs can, optionally, record certain session information when you
123: kill it, such as the files you were visiting at the time.  This
124: information is then available the next time you start Emacs.
125: @xref{Saving Emacs Sessions}.
126: 
127: @vindex confirm-kill-emacs
128:   If the value of the variable @code{confirm-kill-emacs} is
129: non-@code{nil}, @kbd{C-x C-c} assumes that its value is a predicate
130: function, and calls that function.  If the result of the function call
131: is non-@code{nil}, the session is killed, otherwise Emacs continues to
132: run.  One convenient function to use as the value of
133: @code{confirm-kill-emacs} is the function @code{yes-or-no-p}.  The
134: default value of @code{confirm-kill-emacs} is @code{nil}.
135: 
136: @vindex confirm-kill-processes
137:   If the value of the variable @code{confirm-kill-processes} is
138: @code{nil}, @kbd{C-x C-c} does not ask for confirmation before killing
139: subprocesses started by Emacs.  The value is @code{t} by default.
140: 
141:   To further customize what happens when Emacs is exiting, see
142: @ref{Killing Emacs,,, elisp, The GNU Emacs Lisp Reference Manual}.
143: 
144: @findex kill-emacs
145:   To kill Emacs without being prompted about saving, type @kbd{M-x
146: kill-emacs}.
147: 
148: @kindex C-z
149: @findex suspend-frame
150: @cindex minimizing
151: @cindex iconifying
152: @cindex suspending
153:   @kbd{C-z} runs the command @code{suspend-frame}.  On a graphical
154: display, this command @dfn{minimizes} (or @dfn{iconifies}) the
155: selected Emacs frame, hiding it in a way that lets you bring it back
156: later (exactly how this hiding occurs depends on the window system).
157: On a text terminal, the @kbd{C-z} command @dfn{suspends} Emacs,
158: stopping the program temporarily and returning control to the parent
159: process (usually a shell); in most shells, you can resume Emacs after
160: suspending it with the shell command @command{%emacs}.
161: 
162:   Text terminals usually listen for certain special characters whose
163: meaning is to kill or suspend the program you are running.  @b{This
164: terminal feature is turned off while you are in Emacs.}  The meanings
165: of @kbd{C-z} and @kbd{C-x C-c} as keys in Emacs were inspired by the
166: use of @kbd{C-z} and @kbd{C-c} on several operating systems as the
167: characters for stopping or killing a program, but that is their only
168: relationship with the operating system.  You can customize these keys
169: to run any commands of your choice (@pxref{Keymaps}).
