006: @node Antinews
007: @appendix Emacs 25 Antinews
008: @c Update the emacs.texi Antinews menu entry with the above version number.
009: 
010:   For those users who live backwards in time, here is information
011: about downgrading to Emacs version 25.2.  We hope you will enjoy the
012: greater simplicity that results from the absence of many @w{Emacs
013: @value{EMACSVER}} features.
014: 
015: @itemize @bullet
016: @item
017: Emacs no longer defaults to requiring the GnuTLS library when you
018: build it.  Those who want the TLS functionality built-in will have to
019: explicitly request it at build time---or forever hold their peace.  We
020: decided that having the TLS functionality doesn't justify annoying
021: users or package builders with error messages about libgnutls absence.
022: We also decided that if you do build with GnuTLS, we will allow
023: versions of the library older than 2.12.2, as that version will become
024: less and less available/popular as you move farther back in time.
025: 
026: @item
027: For similar reasons, we've reverted back to building our own version
028: of @command{movemail} that retrieves POP3 mail as clear text via
029: insecure channels.  As you move back in time, the availability of
030: secure alternatives to POP3 will diminish, and we are only keen to
031: support that.  We've also removed the @option{--with-mailutils}
032: configure-time option, as it no longer makes sense for the observable
033: past.
034: 
035: @item
036: We have removed support for @command{systemd} and similar services: we
037: no longer provide a user init file for enabling Emacs support via
038: those services, and we removed from the Emacs server the
039: socket-launching support important for Emacs client operation under
040: these services.  Again, these services will lose popularity as you
041: move back in time, so the code supporting them will be just dead code,
042: bloating Emacs unnecessarily.
043: 
044: @item
045: Reproducible builds of Emacs are no longer supported, as past
046: development will make that unnecessary.
047: 
048: @item
049: The @option{--fg-daemon} is gone, leaving only @option{--daemon}.  No
050: need to procrastinate on the dilemma whether you do or don't want the
051: new shiny ``headless Emacs'' thingy.  Hail, simplicity!
052: 
053: @item
054: As text terminals supporting true color will lose ground as you move
055: back in time, we've removed support for 24-bit colors on text
056: terminals.  If you want colors on a text terminal, you should be fine
057: with just 8 of them.  (Truth being told, we think text terminals
058: should be monochrome, but you will have to keep downgrading to older
059: Emacs versions to have that feature back.)
060: 
061: @item
062: Emacs 25.2 no longer supports magic signatures of the form
063: @samp{#!/usr/bin/env @var{interpreter}} in scripts.  Moving back in
064: time means you are getting closer to the ideal of the original Unix
065: design where all the interpreters lived in a single directory
066: @file{/bin}, so this fancy feature is simply becoming unnecessary
067: ballast.
068: 
069: @item
070: The double-buffering feature of Emacs display on X has been removed.
071: We decided that its complexity and a few random surprising
072: side-effects aren't justified by the gains, even though those gains
073: were hailed in some quarters.  Yes, Emacs 25.2 will flicker in some
074: use cases, but we are sure Emacs users will be able to suck it, a they
075: have been doing for years.  Since this feature is gone, we've also
076: removed the @code{inhibit-double-buffering} frame parameter, which is
077: now unnecessary.
078: 
079: @item
080: Non-breaking hyphens and ASCII characters displayed instead of
081: unsupported quote characters are now again displayed using the
082: @code{escape-glyph} face.  We think having a single face instead of 3
083: different ones will make Emacs customization a much simpler job for
084: users.  For the same reason, we've removed the
085: @code{header-line-highlight} face, leaving just @code{highlight} for
086: any element of the Emacs display besides the mode line.
087: 
088: @item
089: You can no longer disable attempts of recovery from fatal exceptions
090: such as C stack overflows and fatal signals.  Since the recovery
091: included in Emacs is reliable enough, we decided there was no reason
092: to put your edits in danger of becoming lost when these situations
093: happen.  The variables @code{'attempt-stack-overflow-recovery} and
094: @code{attempt-orderly-shutdown-on-fatal-signal} are therefore removed.
095: 
096: @item
097: The @code{list-timers} command was removed, as we decided timers are
098: not user-level feature, and therefore users should not be allowed to
099: mess with them.  Ask an Emacs Lisp guru near you for help if you have
100: a runaway timer in your session.  (Of course, as you move back in
101: time, such runaway timers will become less and less frequent, and
102: actually timers might start shutting down automatically, as they
103: cannot cope with time reversal.)
104: 
105: @item
106: Horizontal scrolling using the mouse or touchpad has been removed.  In
107: the past, wide monitors will become less popular, so horizontal
108: scrolling will no longer be needed.  Removal of the mouse support for
109: horizontal scrolling is the first step towards its complete removal in
110: prior Emacs versions.
111: 
112: @item
113: We have found the @option{--tramp} option of @command{emacsclient} too
114: risky and too complicated, so we removed it to simplify the client
115: code and its usage.
116: 
117: @item
118: The @code{display-raw-bytes-as-hex} variable is gone, so raw bytes can
119: only be displayed as octal escapes.  Emacs users should be able to
120: convert from octal to any other base in their sleep!
121: 
122: @item
123: Displaying line numbers for a buffer is only possibly using add-on
124: features, such as @code{linum-mode}, which can only display the
125: numbers in the display margins.  Line-number display using these
126: features is also slow, as we firmly believe such a feature is
127: un-Emacsy and should not have been included in Emacs to begin with.
128: Consequently, @code{display-line-numbers-mode} was removed.
129: 
130: @item
131: On our permanent quest for simplifying Emacs, we've removed the
132: support for passing command-line arguments and options to Emacs via
133: the @option{--alternate-editor} option of @command{emacsclient} and
134: @env{ALTERNATE_EDITOR} environment variable.  There's only one True
135: Emacs---the one that comes up when invoked as @kbd{emacs}, no need for
136: all those fancy options!
137: 
138: @item
139: The complication known as ``single-line horizontal scrolling'' is no
140: longer with you in Emacs 25.2.  This feature was a bow to ``other
141: editors''; instead, let those other editors bow to Emacs by hscrolling
142: the entire window at all times.  Repeat after me: ``The Emacs way is
143: the Only Way!''
144: 
145: @item
146: The fancy case conversions of non-ASCII characters used in several
147: locales, like Turkish and Greek, are removed, leaving the relations
148: between upper and lower letter-case simple again, as they were in
149: 7-bit ASCII.  Likewise with ligatures that turn into multiple
150: characters when their letter-case changes---gone.
151: 
152: @item
153: Enchant is no longer supported by @code{ispell-buffer} and similar
154: spell-checking commands.  As Enchant will gradually disappear while
155: you move back in time, its support will become unnecessary anyway.
156: 
157: @item
158: Tramp lost its support for Google Drive repositories.  Cloud storage
159: is on its way to extinction as you move back in time, thus making this
160: feature redundant.
161: 
162: @item
163: Several commands, deemed to be unnecessary complications, have been
164: removed.  Examples include @code{replace-buffer-contents} and
165: @code{apropos-local-variable}.
166: 
167: @item
168: To keep up with decreasing computer memory capacity and disk space, many
169: other functions and files have been eliminated in Emacs 25.2.
170: @end itemize
