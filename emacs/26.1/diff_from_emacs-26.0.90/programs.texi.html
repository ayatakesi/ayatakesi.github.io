0051: @section Major Modes for Programming Languages
0052: @cindex modes for programming languages
0053: 
0054:   Emacs has specialized major modes (@pxref{Major Modes}) for many
0055: programming languages.  A programming language mode typically
0056: specifies the syntax of expressions, the customary rules for
0057: indentation, how to do syntax highlighting for the language, and how
0058: to find the beginning or end of a function definition.  It often has
0059: features for compiling and debugging programs as well.  The major mode
0060: for each language is named after the language; for instance, the major
0061: mode for the C programming language is @code{c-mode}.

0135: @section Top-Level Definitions, or Defuns
0136: 
0137:   In Emacs, a major definition at the top level in the buffer, such as

0139: Emacs we use it for all languages.

0149: @node Left Margin Paren
0150: @subsection Left Margin Convention
0151: 
0152: @cindex open-parenthesis in leftmost column
0153: @cindex ( in leftmost column
0154:   Many programming-language modes assume by default that any opening
0155: delimiter found at the left margin is the start of a top-level
0156: definition, or defun.  Therefore, @strong{don't put an opening
0157: delimiter at the left margin unless it should have that significance}.
0158: For instance, never put an open-parenthesis at the left margin in a
0159: Lisp file unless it is the start of a top-level list.
0160: 
0161:   The convention speeds up many Emacs operations, which would
0162: otherwise have to scan back to the beginning of the buffer to analyze
0163: the syntax of the code.
0164: 
0165:   If you don't follow this convention, not only will you have trouble
0166: when you explicitly use the commands for motion by defuns; other
0167: features that use them will also give you trouble.  This includes the
0168: indentation commands (@pxref{Program Indent}) and Font Lock mode
0169: (@pxref{Font Lock}).
0170: 
0171:   The most likely problem case is when you want an opening delimiter
0172: at the start of a line inside a string.  To avoid trouble, put an
0173: escape character (@samp{\}, in C and Emacs Lisp, @samp{/} in some
0174: other Lisp dialects) before the opening delimiter.  This will not
0175: affect the contents of the string, but will prevent that opening
0176: delimiter from starting a defun.  Here's an example:
0177: 
0178: @example
0179:   (insert "Foo:
0180: \(bar)
0181: ")
0182: @end example
0183: 
0184:   To help you catch violations of this convention, Font Lock mode
0185: highlights confusing opening delimiters (those that ought to be
0186: quoted) in bold red.
0187: 
0188: @vindex open-paren-in-column-0-is-defun-start
0189:   If you need to override this convention, you can do so by setting
0190: the variable @code{open-paren-in-column-0-is-defun-start}.
0191: If this user option is set to @code{t} (the default), opening
0192: parentheses or braces at column zero always start defuns.  When it is
0193: @code{nil}, defuns are found by searching for parens or braces at the
0194: outermost level.
0195: 
0196:   Usually, you should leave this option at its default value of
0197: @code{t}.  If your buffer contains parentheses or braces in column
0198: zero which don't start defuns, and it is somehow impractical to remove
0199: these parentheses or braces, it might be helpful to set the option to
0200: @code{nil}.  Be aware that this might make scrolling and display in
0201: large buffers quite sluggish.  Furthermore, the parentheses and braces
0202: must be correctly matched throughout the buffer for it to work
0203: properly.

0205: @node Moving by Defuns
0206: @subsection Moving by Defuns
0207: @cindex defuns
0208: 
0209:   These commands move point or set up the region based on top-level
0210: major definitions, also called @dfn{defuns}.
0211: 
0212: @table @kbd
0213: @item C-M-a
0214: Move to beginning of current or preceding defun
0215: (@code{beginning-of-defun}).
0216: @item C-M-e
0217: Move to end of current or following defun (@code{end-of-defun}).
0218: @item C-M-h
0219: Put region around whole current or following defun (@code{mark-defun}).
0220: @end table
0221: 
0222: @cindex move to beginning or end of function
0223: @cindex function, move to beginning or end
0224: @kindex C-M-a
0225: @kindex C-M-e
0226: @kindex C-M-h
0227: @findex beginning-of-defun
0228: @findex end-of-defun
0229: @findex mark-defun
0230:   The commands to move to the beginning and end of the current defun
0231: are @kbd{C-M-a} (@code{beginning-of-defun}) and @kbd{C-M-e}
0232: (@code{end-of-defun}).  If you repeat one of these commands, or use a
0233: positive numeric argument, each repetition moves to the next defun in
0234: the direction of motion.
0235: 
0236:   @kbd{C-M-a} with a negative argument @minus{}@var{n} moves forward
0237: @var{n} times to the next beginning of a defun.  This is not exactly
0238: the same place that @kbd{C-M-e} with argument @var{n} would move to;
0239: the end of this defun is not usually exactly the same place as the
0240: beginning of the following defun.  (Whitespace, comments, and perhaps
0241: declarations can separate them.)  Likewise, @kbd{C-M-e} with a
0242: negative argument moves back to an end of a defun, which is not quite
0243: the same as @kbd{C-M-a} with a positive argument.
0244: 
0245: @kindex C-M-h @r{(C mode)}
0246: @findex c-mark-function
0247:   To operate on the current defun, use @kbd{C-M-h}
0248: (@code{mark-defun}), which sets the mark at the end of the current
0249: defun and puts point at its beginning.  @xref{Marking Objects}.  This
0250: is the easiest way to get ready to kill the defun in order to move it
0251: to a different place in the file.  If the defun is directly preceded
0252: by comments (with no intervening blank lines), they are marked, too.
0253: If you use the command while point is between defuns, it uses the
0254: following defun.  If you use the command while the mark is already
0255: active, it extends the end of the region to include one more defun.
0256: With a prefix argument, it marks that many defuns or extends the
0257: region by the appropriate number of defuns.  With negative prefix
0258: argument it marks defuns in the opposite direction and also changes
0259: the direction of selecting for subsequent uses of @code{mark-defun}.
0260: 
0261:   In C mode, @kbd{C-M-h} runs the function @code{c-mark-function},
0262: which is almost the same as @code{mark-defun}; the difference is that
0263: it backs up over the argument declarations, function name and returned
0264: data type so that the entire C function is inside the region.  This is
0265: an example of how major modes adjust the standard key bindings so that
0266: they do their standard jobs in a way better fitting a particular
0267: language.  Other major modes may replace any or all of these key
0268: bindings for that purpose.

0281: @findex imenu
0282:   If you type @kbd{M-x imenu}, it reads the name of a definition using
0283: the minibuffer, then moves point to that definition.  You can use
0284: completion to specify the name; the command always displays the whole
0285: list of valid names.
0286: 
0287: @findex imenu-add-menubar-index
0288:   Alternatively, you can bind the command @code{imenu} to a mouse
0289: click.  Then it displays mouse menus for you to select a definition
0290: name.  You can also add the buffer's index to the menu bar by calling

0292: item available for all buffers in a certain major mode, you can do
0293: this by adding @code{imenu-add-menubar-index} to its mode hook.  But
0294: if you have done that, you will have to wait a little while each time
0295: you visit a file in that mode, while Emacs finds all the definitions
0296: in that buffer.

0299:   When you change the contents of a buffer, if you add or delete

0302: Rescanning happens automatically if you set @code{imenu-auto-rescan} to

0366: @node Basic Indent
0367: @subsection Basic Program Indentation Commands
0368: 
0369: @table @kbd
0370: @item @key{TAB}
0371: Adjust indentation of current line (@code{indent-for-tab-command}).
0372: @item @key{RET}
0373: Insert a newline, then adjust indentation of following line
0374: (@code{newline}).
0375: @end table

0377: @kindex TAB @r{(programming modes)}
0378: @findex c-indent-command
0379: @findex indent-line-function
0380: @findex indent-for-tab-command
0381:   The basic indentation command is @key{TAB}
0382: (@code{indent-for-tab-command}), which was documented in
0383: @ref{Indentation}.  In programming language modes, @key{TAB} indents
0384: the current line, based on the indentation and syntactic content of
0385: the preceding lines; if the region is active, @key{TAB} indents each
0386: line within the region, not just the current line.
0387: 
0388:   The command @key{RET} (@code{newline}), which was documented in
0389: @ref{Inserting Text}, does the same as @kbd{C-j} followed by
0390: @key{TAB}: it inserts a new line, then adjusts the line's indentation.
0391: 
0392:   When indenting a line that starts within a parenthetical grouping,
0393: Emacs usually places the start of the line under the preceding line
0394: within the group, or under the text after the parenthesis.  If you
0395: manually give one of these lines a nonstandard indentation (e.g., for
0396: aesthetic purposes), the lines below will follow it.
0397: 
0398:   The indentation commands for most programming language modes assume
0399: that a open-parenthesis, open-brace or other opening delimiter at the
0400: left margin is the start of a function.  If the code you are editing
0401: violates this assumption---even if the delimiters occur in strings or
0402: comments---you must set @code{open-paren-in-column-0-is-defun-start}
0403: to @code{nil} for indentation to work properly.  @xref{Left Margin
0404: Paren}.

0406: @node Multi-line Indent
0407: @subsection Indenting Several Lines
0408: 
0409:   Sometimes, you may want to reindent several lines of code at a time.
0410: One way to do this is to use the mark; when the mark is active and the
0411: region is non-empty, @key{TAB} indents every line in the region.
0412: Alternatively, the command @kbd{C-M-\} (@code{indent-region}) indents
0413: every line in the region, whether or not the mark is active
0414: (@pxref{Indentation Commands}).
0415: 
0416:   In addition, Emacs provides the following commands for indenting
0417: large chunks of code:
0418: 
0419: @table @kbd
0420: @item C-M-q
0421: Reindent all the lines within one parenthetical grouping.
0422: @item C-u @key{TAB}
0423: Shift an entire parenthetical grouping rigidly sideways so that its
0424: first line is properly indented.
0425: @item M-x indent-code-rigidly
0426: Shift all the lines in the region rigidly sideways, but do not alter
0427: lines that start inside comments and strings.
0428: @end table
0429: 
0430: @kindex C-M-q
0431: @findex indent-pp-sexp
0432:   To reindent the contents of a single parenthetical grouping,
0433: position point before the beginning of the grouping and type
0434: @kbd{C-M-q}.  This changes the relative indentation within the
0435: grouping, without affecting its overall indentation (i.e., the
0436: indentation of the line where the grouping starts).  The function that
0437: @kbd{C-M-q} runs depends on the major mode; it is
0438: @code{indent-pp-sexp} in Lisp mode, @code{c-indent-exp} in C mode,
0439: etc.  To correct the overall indentation as well, type @key{TAB}
0440: first.
0441: 
0442: @kindex C-u TAB
0443:   If you like the relative indentation within a grouping but not the
0444: indentation of its first line, move point to that first line and type
0445: @kbd{C-u @key{TAB}}.  In Lisp, C, and some other major modes,
0446: @key{TAB} with a numeric argument reindents the current line as usual,
0447: then reindents by the same amount all the lines in the parenthetical
0448: grouping starting on the current line.  It is clever, though, and does
0449: not alter lines that start inside strings.  Neither does it alter C
0450: preprocessor lines when in C mode, but it does reindent any
0451: continuation lines that may be attached to them.
0452: 
0453: @findex indent-code-rigidly
0454:   The command @kbd{M-x indent-code-rigidly} rigidly shifts all the
0455: lines in the region sideways, like @code{indent-rigidly} does
0456: (@pxref{Indentation Commands}).  It doesn't alter the indentation of
0457: lines that start inside a string, unless the region also starts inside
0458: that string.  The prefix arg specifies the number of columns to
0459: indent.

0461: @node Lisp Indent
0462: @subsection Customizing Lisp Indentation
0463: @cindex customizing Lisp indentation
0464: 
0465:   The indentation pattern for a Lisp expression can depend on the function
0466: called by the expression.  For each Lisp function, you can choose among
0467: several predefined patterns of indentation, or define an arbitrary one with
0468: a Lisp program.
0469: 
0470:   The standard pattern of indentation is as follows: the second line of the
0471: expression is indented under the first argument, if that is on the same
0472: line as the beginning of the expression; otherwise, the second line is
0473: indented underneath the function name.  Each following line is indented
0474: under the previous line whose nesting depth is the same.
0475: 
0476: @vindex lisp-indent-offset
0477:   If the variable @code{lisp-indent-offset} is non-@code{nil}, it overrides
0478: the usual indentation pattern for the second line of an expression, so that
0479: such lines are always indented @code{lisp-indent-offset} more columns than
0480: the containing list.
0481: 
0482: @vindex lisp-body-indent
0483:   Certain functions override the standard pattern.  Functions whose
0484: names start with @code{def} treat the second lines as the start of
0485: a @dfn{body}, by indenting the second line @code{lisp-body-indent}
0486: additional columns beyond the open-parenthesis that starts the
0487: expression.
0488: 
0489: @cindex @code{lisp-indent-function} property
0490:   You can override the standard pattern in various ways for individual
0491: functions, according to the @code{lisp-indent-function} property of
0492: the function name.  This is normally done for macro definitions, using
0493: the @code{declare} construct.  @xref{Defining Macros,,, elisp, the
0494: Emacs Lisp Reference Manual}.

0540: @node Custom C Indent
0541: @subsection Customizing C Indentation
0542: @cindex style (for indentation)
0543: 
0544:   C mode and related modes use a flexible mechanism for customizing
0545: indentation.  C mode indents a source line in two steps: first it
0546: classifies the line syntactically according to its contents and
0547: context; second, it determines the indentation offset associated by
0548: your selected @dfn{style} with the syntactic construct and adds this
0549: onto the indentation of the @dfn{anchor statement}.
0550: 
0551: @table @kbd
0552: @item C-c . @var{style} @key{RET}
0553: Select a predefined style @var{style} (@code{c-set-style}).
0554: @end table
0555: 
0556:   A @dfn{style} is a named collection of customizations that can be
0557: used in C mode and the related modes.  @ref{Styles,,, ccmode, The CC
0558: Mode Manual}, for a complete description.  Emacs comes with several
0559: predefined styles, including @code{gnu}, @code{k&r}, @code{bsd},
0560: @code{stroustrup}, @code{linux}, @code{python}, @code{java},
0561: @code{whitesmith}, @code{ellemtel}, and @code{awk}.  Some of these
0562: styles are primarily intended for one language, but any of them can be
0563: used with any of the languages supported by these modes.  To find out
0564: what a style looks like, select it and reindent some code, e.g., by
0565: typing @kbd{C-M-q} at the start of a function definition.
0566: 
0567: @kindex C-c . @r{(C mode)}
0568: @findex c-set-style
0569:   To choose a style for the current buffer, use the command @w{@kbd{C-c
0570: .}}.  Specify a style name as an argument (case is not significant).
0571: This command affects the current buffer only, and it affects only
0572: future invocations of the indentation commands; it does not reindent
0573: the code already in the buffer.  To reindent the whole buffer in the
0574: new style, you can type @kbd{C-x h C-M-\}.
0575: 
0576: @vindex c-default-style
0577:   You can also set the variable @code{c-default-style} to specify the
0578: default style for various major modes.  Its value should be either the
0579: style's name (a string) or an alist, in which each element specifies
0580: one major mode and which indentation style to use for it.  For
0581: example,
0582: 
0583: @example
0584: (setq c-default-style
0585:       '((java-mode . "java")
0586:         (awk-mode . "awk")
0587:         (other . "gnu")))
0588: @end example
0589: 
0590: @noindent
0591: specifies explicit choices for Java and AWK modes, and the default
0592: @samp{gnu} style for the other C-like modes.  (These settings are
0593: actually the defaults.)  This variable takes effect when you select
0594: one of the C-like major modes; thus, if you specify a new default
0595: style for Java mode, you can make it take effect in an existing Java
0596: mode buffer by typing @kbd{M-x java-mode} there.
0597: 
0598:   The @code{gnu} style specifies the formatting recommended by the GNU
0599: Project for C; it is the default, so as to encourage use of our
0600: recommended style.
0601: 
0602:   @xref{Indentation Engine Basics,,, ccmode, the CC Mode Manual}, and
0603: @ref{Customizing Indentation,,, ccmode, the CC Mode Manual}, for more
0604: information on customizing indentation for C and related modes,
0605: including how to override parts of an existing style and how to define
0606: your own styles.
0607: 
0608: @findex c-guess

0613: c-guess-install}.  @xref{Guessing the Style,,, ccmode, the CC Mode

0642: @node Expressions
0643: @subsection Expressions with Balanced Parentheses
0644: 
0645: @cindex sexp
0646: @cindex expression
0647: @cindex balanced expression
0648:   Each programming language mode has its own definition of a
0649: @dfn{balanced expression}.  Balanced expressions typically include
0650: individual symbols, numbers, and string constants, as well as pieces
0651: of code enclosed in a matching pair of delimiters.  The following
0652: commands deal with balanced expressions (in Emacs, such expressions
0653: are referred to internally as @dfn{sexps}@footnote{The word ``sexp''
0654: is used to refer to an expression in Lisp.}).
0655: 
0656: @table @kbd
0657: @item C-M-f
0658: Move forward over a balanced expression (@code{forward-sexp}).
0659: @item C-M-b
0660: Move backward over a balanced expression (@code{backward-sexp}).
0661: @item C-M-k
0662: Kill balanced expression forward (@code{kill-sexp}).
0663: @item C-M-t
0664: Transpose expressions (@code{transpose-sexps}).
0665: @item C-M-@@
0666: @itemx C-M-@key{SPC}
0667: Put mark after following expression (@code{mark-sexp}).
0668: @end table
0669: 
0670: @kindex C-M-f
0671: @kindex C-M-b
0672: @findex forward-sexp
0673: @findex backward-sexp
0674:   To move forward over a balanced expression, use @kbd{C-M-f}
0675: (@code{forward-sexp}).  If the first significant character after point
0676: is an opening delimiter (e.g., @samp{(}, @samp{[} or @samp{@{} in C),
0677: this command moves past the matching closing delimiter.  If the
0678: character begins a symbol, string, or number, the command moves over
0679: that.
0680: 
0681:   The command @kbd{C-M-b} (@code{backward-sexp}) moves backward over a
0682: balanced expression---like @kbd{C-M-f}, but in the reverse direction.
0683: If the expression is preceded by any prefix characters (single-quote,
0684: backquote and comma, in Lisp), the command moves back over them as
0685: well.
0686: 
0687:   @kbd{C-M-f} or @kbd{C-M-b} with an argument repeats that operation
0688: the specified number of times; with a negative argument means to move
0689: in the opposite direction.  In most modes, these two commands move
0690: across comments as if they were whitespace.  Note that their keys,
0691: @kbd{C-M-f} and @kbd{C-M-b}, are analogous to @kbd{C-f} and @kbd{C-b},
0692: which move by characters (@pxref{Moving Point}), and @kbd{M-f} and
0693: @kbd{M-b}, which move by words (@pxref{Words}).
0694: 
0695: @cindex killing expressions
0696: @kindex C-M-k
0697: @findex kill-sexp
0698:   To kill a whole balanced expression, type @kbd{C-M-k}
0699: (@code{kill-sexp}).  This kills the text that @kbd{C-M-f} would move
0700: over.
0701: 
0702: @cindex transposition of expressions
0703: @kindex C-M-t
0704: @findex transpose-sexps
0705:   @kbd{C-M-t} (@code{transpose-sexps}) switches the positions of the
0706: previous balanced expression and the next one.  It is analogous to the
0707: @kbd{C-t} command, which transposes characters (@pxref{Transpose}).
0708: An argument to @kbd{C-M-t} serves as a repeat count, moving the
0709: previous expression over that many following ones.  A negative
0710: argument moves the previous balanced expression backwards across those
0711: before it.  An argument of zero, rather than doing nothing, transposes
0712: the balanced expressions ending at or after point and the mark.
0713: 
0714: @kindex C-M-@@
0715: @kindex C-M-SPC
0716: @findex mark-sexp
0717:   To operate on balanced expressions with a command which acts on the
0718: region, type @kbd{C-M-@key{SPC}} (@code{mark-sexp}).  This sets the
0719: mark where @kbd{C-M-f} would move to.  While the mark is active, each
0720: successive call to this command extends the region by shifting the
0721: mark by one expression.  Positive or negative numeric arguments move
0722: the mark forward or backward by the specified number of expressions.
0723: The alias @kbd{C-M-@@} is equivalent to @kbd{C-M-@key{SPC}}.
0724: @xref{Marking Objects}, for more information about this and related
0725: commands.
0726: 
0727:   In languages that use infix operators, such as C, it is not possible
0728: to recognize all balanced expressions because there can be multiple
0729: possibilities at a given position.  For example, C mode does not treat
0730: @samp{foo + bar} as a single expression, even though it @emph{is} one
0731: C expression; instead, it recognizes @samp{foo} as one expression and
0732: @samp{bar} as another, with the @samp{+} as punctuation between them.
0733: However, C mode recognizes @samp{(foo + bar)} as a single expression,
0734: because of the parentheses.

0750: programs, but can be useful for editing any text containing

0755: or a comment.  If you invoke them from inside a string or comment, the

0771: @findex forward-list
0772: @findex backward-list

0774: @kbd{C-M-p} (@code{backward-list}) move forward or backward over one
0775: (or @var{n}) parenthetical groupings.
0776: 
0777: @kindex C-M-u
0778: @findex backward-up-list
0779:   @kbd{C-M-n} and @kbd{C-M-p} try to stay at the same level in the

0793: @node Matching
0794: @subsection Matching Parentheses
0795: @cindex matching parentheses
0796: @cindex parentheses, displaying matches
0797: 
0798:   Emacs has a number of @dfn{parenthesis matching} features, which
0799: make it easy to see how and whether parentheses (or other delimiters)
0800: match up.
0801: 
0802:   Whenever you type a self-inserting character that is a closing
0803: delimiter, Emacs briefly indicates the location of the matching
0804: opening delimiter, provided that is on the screen.  If it is not on
0805: the screen, Emacs displays some of the text near it in the echo area.
0806: Either way, you can tell which grouping you are closing off.  If the
0807: opening delimiter and closing delimiter are mismatched---such as in
0808: @samp{[x)}---a warning message is displayed in the echo area.
0809: 
0810: @vindex blink-matching-paren
0811: @vindex blink-matching-paren-distance
0812: @vindex blink-matching-delay
0813:   Three variables control the display of matching parentheses:
0814: 
0815: @itemize @bullet
0816: @item
0817: @code{blink-matching-paren} turns the feature on or off: @code{nil}
0818: disables it, but the default is @code{t} to enable it.  Set it to
0819: @code{jump} to make indication work by momentarily moving the cursor
0820: to the matching opening delimiter.  Set it to @code{jump-offscreen} to
0821: make the cursor jump, even if the opening delimiter is off screen.
0822: 
0823: @item
0824: @code{blink-matching-delay} says how many seconds to keep indicating
0825: the matching opening delimiter.  This may be an integer or
0826: floating-point number; the default is 1.
0827: 
0828: @item
0829: @code{blink-matching-paren-distance} specifies how many characters
0830: back to search to find the matching opening delimiter.  If the match
0831: is not found in that distance, Emacs stops scanning and nothing is
0832: displayed.  The default is 102400.
0833: @end itemize

0835: @cindex Show Paren mode
0836: @cindex highlighting matching parentheses
0837: @findex show-paren-mode
0838:   Show Paren mode, a global minor mode, provides a more powerful kind
0839: of automatic matching.  Whenever point is before an opening delimiter
0840: or after a closing delimiter, the delimiter, its matching delimiter,
0841: and optionally the text between them are highlighted.  To toggle Show
0842: Paren mode, type @kbd{M-x show-paren-mode}.  To customize it, type
0843: @kbd{M-x customize-group @key{RET} paren-showing}.  The customizable
0844: options which control the operation of this mode include:
0845: 
0846: @itemize @bullet
0847: @item
0848: @code{show-paren-highlight-open-paren} controls whether to highlight
0849: an open paren when point stands just before it, and hence its position
0850: is marked by the cursor anyway.  The default is non-@code{nil} (yes).
0851: 
0852: @item
0853: @code{show-paren-style} controls whether just the two parens, or also
0854: the space between them get highlighted.  The valid options here are
0855: @code{parenthesis} (show the matching paren), @code{expression}
0856: (highlight the entire expression enclosed by the parens), and
0857: @code{mixed} (highlight the matching paren if it is visible, the
0858: expression otherwise).
0859: 
0860: @item
0861: @code{show-paren-when-point-inside-paren}, when non-@code{nil}, causes
0862: highlighting also when point is on the inside of a parenthesis.
0863: 
0864: @item
0865: @code{show-paren-when-point-in-periphery}, when non-@code{nil}, causes
0866: highlighting also when point is in whitespace at the beginning or end
0867: of a line, and there is a paren at, respectively, the first or last,
0868: or the last, non-whitespace position on the line.

0917: provides special commands for editing and inserting comments.  It can
0918: also do spell checking on comments with Flyspell Prog mode

0927: will indent the comment to the appropriate position.

0930: ;; This function is just an example.

0932: (defun foo (x)

0934:   ;; The following line adds one.

0950: 
0951: @table @asis
0952: @item @kbd{M-;}
0953: Insert or realign comment on current line; if the region is active,
0954: comment or uncomment the region instead (@code{comment-dwim}).
0955: @item @kbd{C-x C-;}
0956: Comment or uncomment the current line (@code{comment-line}).
0957: @item @kbd{C-u M-;}
0958: Kill comment on current line (@code{comment-kill}).
0959: @item @kbd{C-x ;}
0960: Set comment column (@code{comment-set-column}).
0961: @item @kbd{C-M-j}
0962: @itemx @kbd{M-j}
0963: Like @key{RET} followed by inserting and aligning a comment
0964: (@code{comment-indent-new-line}).  @xref{Multi-Line Comments}.
0965: @item @kbd{M-x comment-region}
0966: @itemx @kbd{C-c C-c} (in C-like modes)
0967: Add comment delimiters to all the lines in the region.
0968: @end table
0969: 
0970: @kindex M-;
0971: @findex comment-dwim
0972:   The command to create or align a comment is @kbd{M-;}
0973: (@code{comment-dwim}).  The word ``dwim'' is an acronym for ``Do What
0974: I Mean''; it indicates that this command can be used for many
0975: different jobs relating to comments, depending on the situation where
0976: you use it.
0977: 
0978:   When a region is active (@pxref{Mark}), @kbd{M-;} either adds
0979: comment delimiters to the region, or removes them.  If every line in
0980: the region is already a comment, it uncomments each of those lines
0981: by removing their comment delimiters.  Otherwise, it adds comment
0982: delimiters to enclose the text in the region.
0983: 
0984:   If you supply a prefix argument to @kbd{M-;} when a region is
0985: active, that specifies the number of comment delimiters to add or
0986: delete.  A positive argument @var{n} adds @var{n} delimiters, while a
0987: negative argument @var{-n} removes @var{n} delimiters.
0988: 
0989:   If the region is not active, and there is no existing comment on the
0990: current line, @kbd{M-;} adds a new comment to the current line.  If
0991: the line is blank (i.e., empty or containing only whitespace
0992: characters), the comment is indented to the same position where
0993: @key{TAB} would indent to (@pxref{Basic Indent}).  If the line is
0994: non-blank, the comment is placed after the last non-whitespace
0995: character on the line; normally, Emacs tries putting it at the column
0996: specified by the variable @code{comment-column} (@pxref{Options for
0997: Comments}), but if the line already extends past that column, it puts
0998: the comment at some suitable position, usually separated from the
0999: non-comment text by at least one space.  In each case, Emacs places
1000: point after the comment's starting delimiter, so that you can start
1001: typing the comment text right away.
1002: 
1003:   You can also use @kbd{M-;} to align an existing comment.  If a line
1004: already contains the comment-start string, @kbd{M-;} realigns it to
1005: the conventional alignment and moves point after the comment's
1006: starting delimiter.  As an exception, comments starting in column 0
1007: are not moved.  Even when an existing comment is properly aligned,
1008: @kbd{M-;} is still useful for moving directly to the start of the
1009: comment text.
1010: 
1011: @findex comment-line
1012: @kindex C-x C-;
1013:   @kbd{C-x C-;} (@code{comment-line}) comments or uncomments complete
1014: lines.  When a region is active (@pxref{Mark}), @kbd{C-x C-;} either
1015: comments or uncomments the lines in the region.  If the region is not
1016: active, this command comments or uncomments the line point is on.
1017: With a positive prefix argument @var{n}, it operates on @var{n} lines
1018: starting with the current one; with a negative @var{n}, it affects
1019: @var{n} preceding lines.  After invoking this command with a negative
1020: argument, successive invocations with a positive argument will operate
1021: on preceding lines as if the argument were negated.
1022: 
1023: @findex comment-kill
1024: @kindex C-u M-;
1025:   @kbd{C-u M-;} (@code{comment-dwim} with a prefix argument) kills any
1026: comment on the current line, along with the whitespace before it.
1027: Since the comment is saved to the kill ring, you can reinsert it on
1028: another line by moving to the end of that line, doing @kbd{C-y}, and
1029: then @kbd{M-;} to realign the comment.  You can achieve the same
1030: effect as @kbd{C-u M-;} by typing @kbd{M-x comment-kill}
1031: (@code{comment-dwim} actually calls @code{comment-kill} as a
1032: subroutine when it is given a prefix argument).
1033: 
1034: @kindex C-c C-c (C mode)
1035: @findex comment-region
1036: @findex uncomment-region
1037:   The command @kbd{M-x comment-region} is equivalent to calling
1038: @kbd{M-;} on an active region, except that it always acts on the

1193: @node Man Page
1194: @subsection Man Page Lookup
1195: 
1196: @cindex man page
1197:   On Unix, the main form of on-line documentation was the @dfn{manual
1198: page} or @dfn{man page}.  In the GNU operating system, we aim to
1199: replace man pages with better-organized manuals that you can browse
1200: with Info (@pxref{Misc Help}).  This process is not finished, so it is
1201: still useful to read manual pages.
1202: 
1203: @findex man
1204:   You can read the man page for an operating system command, library
1205: function, or system call, with the @kbd{M-x man} command.  This
1206: prompts for a topic, with completion (@pxref{Completion}), and runs
1207: the @command{man} program to format the corresponding man page.  If
1208: the system permits, it runs @command{man} asynchronously, so that you
1209: can keep on editing while the page is being formatted.  The result
1210: goes in a buffer named @file{*Man @var{topic}*}.  These buffers use a
1211: special major mode, Man mode, that facilitates scrolling and jumping
1212: to other manual pages.  For details, type @kbd{C-h m} while in a Man
1213: mode buffer.
1214: 
1215: @cindex sections of manual pages
1216:   Each man page belongs to one of ten or more @dfn{sections}, each
1217: named by a digit or by a digit and a letter.  Sometimes there are man
1218: pages with the same name in different sections.  To read a man page
1219: from a specific section, type @samp{@var{topic}(@var{section})} or
1220: @samp{@var{section} @var{topic}} when @kbd{M-x man} prompts for the
1221: topic.  For example, the man page for the C library function
1222: @code{chmod} is in section 2, but there is a shell command of the same
1223: name, whose man page is in section 1; to view the former, type
1224: @w{@kbd{M-x man @key{RET} chmod(2) @key{RET}}}.
1225: 
1226: @vindex Man-switches
1227: @kindex M-n @r{(Man mode)}
1228: @kindex M-p @r{(Man mode)}
1229:   If you do not specify a section, @kbd{M-x man} normally displays
1230: only the first man page found.  On some systems, the @code{man}
1231: program accepts a @samp{-a} command-line option, which tells it to
1232: display all the man pages for the specified topic.  To make use of
1233: this, change the value of the variable @code{Man-switches} to
1234: @samp{"-a"}.  Then, in the Man mode buffer, you can type @kbd{M-n} and
1235: @kbd{M-p} to switch between man pages in different sections.  The mode
1236: line shows how many manual pages are available.
1237: 
1238: @findex woman
1239: @cindex manual pages, on MS-DOS/MS-Windows
1240:   An alternative way of reading manual pages is the @kbd{M-x woman}
1241: command.  Unlike @kbd{M-x man}, it does not run any external programs
1242: to format and display the man pages; the formatting is done by Emacs,
1243: so it works on systems such as MS-Windows where the @command{man}
1244: program may be unavailable.  It prompts for a man page, and displays
1245: it in a buffer named @file{*WoMan @var{section} @var{topic}}.
1246: 
1247:   @kbd{M-x woman} computes the completion list for manpages the first
1248: time you invoke the command.  With a numeric argument, it recomputes
1249: this list; this is useful if you add or delete manual pages.
1250: 
1251:   If you type a name of a manual page and @kbd{M-x woman} finds that
1252: several manual pages by the same name exist in different sections, it
1253: pops up a window with possible candidates asking you to choose one of
1254: them.
1255: 
1256:   For more information about setting up and using @kbd{M-x woman}, see
1257: @ifinfo
1258: @ref{Top, WoMan, Browse UN*X Manual Pages WithOut Man, woman, The
1259: WoMan Manual}.
1260: @end ifinfo
1261: @ifnotinfo
1262: the WoMan Info manual, which is distributed with Emacs.

1428: 
1429: Semantic is a package that provides language-aware editing commands
1430: based on @code{source code parsers}.  This section provides a brief
1431: description of Semantic; for full details,
1432: @ifnottex
1433: see @ref{Top, Semantic,, semantic, Semantic}.

1439:   Most of the language-aware features in Emacs, such as Font Lock

1441: expressions and syntax tables.} that usually give good results but are
1442: never completely exact.  In contrast, the parsers used by Semantic
1443: have an exact understanding of programming language syntax.  This
1444: allows Semantic to provide search, navigation, and completion commands
1445: that are powerful and precise.

1447: @cindex Semantic mode

1452:

1454: parse each file you visit.  Currently, Semantic understands C, C++,
1455: Scheme, Javascript, Java, HTML, and Make.  Within each parsed buffer,
1456: the following commands are available:
1457: 
1458: @table @kbd

1464: @item C-c , J
1465: @kindex C-c , J
1466: Prompt for the name of a function defined in any file Emacs has
1467: parsed, and move point there (@code{semantic-complete-jump}).
1468: 
1469: @item C-c , @key{SPC}
1470: @kindex C-c , SPC
1471: Display a list of possible completions for the symbol at point
1472: (@code{semantic-complete-analyze-inline}).  This also activates a set
1473: of special key bindings for choosing a completion: @key{RET} accepts
1474: the current completion, @kbd{M-n} and @kbd{M-p} cycle through possible
1475: completions, @key{TAB} completes as far as possible and then cycles,

1477: 
1478: @item C-c , l
1479: @kindex C-c , l
1480: Display a list of the possible completions of the symbol at point, in

1545: replace @samp{\alpha} @dots{} @samp{\omega} and other math macros with
1546: their Unicode characters.  You may wish to use this in non-programming
1547: modes as well.  You can customize the mode by adding more entries to
1548: @code{prettify-symbols-alist}.  More elaborate customization is
1549: available via customizing @code{prettify-symbols-compose-predicate} if
1550: its default value @code{prettify-symbols-default-compose-p} is not
1551: appropriate.  There is also a global version,
1552: @code{global-prettify-symbols-mode}, which enables the mode in all
1553: buffers that support it.
1554: 
1555:   The symbol at point can be shown in its original form.  This is
1556: controlled by the variable @code{prettify-symbols-unprettify-at-point}:
1557: if non-@code{nil}, the original form of symbol at point will be
1558: restored for as long as point is at it.

1600: 
1601:   This section describes commands for moving point, in C mode and
1602: related modes.
1603: 
1604: @table @code
1605: @item C-M-a
1606: @itemx C-M-e
1607: @findex c-beginning-of-defun
1608: @findex c-end-of-defun
1609: Move point to the beginning or end of the current function or
1610: top-level definition.  In languages with enclosing scopes (such as
1611: C++'s classes) the @dfn{current function} is the immediate one,
1612: possibly inside a scope.  Otherwise it is the one defined by the least
1613: enclosing braces.  (By contrast, @code{beginning-of-defun} and
1614: @code{end-of-defun} search for braces in column zero.)  @xref{Moving
1615: by Defuns}.
1616: 
1617: @item C-c C-u
1618: @kindex C-c C-u @r{(C mode)}
1619: @findex c-up-conditional
1620: Move point back to the containing preprocessor conditional, leaving the
1621: mark behind.  A prefix argument acts as a repeat count.  With a negative
1622: argument, move point forward to the end of the containing
1623: preprocessor conditional.
1624: 
1625: @samp{#elif} is equivalent to @samp{#else} followed by @samp{#if}, so
1626: the function will stop at a @samp{#elif} when going backward, but not
1627: when going forward.
1628: 
1629: @item C-c C-p
1630: @kindex C-c C-p @r{(C mode)}
1631: @findex c-backward-conditional
1632: Move point back over a preprocessor conditional, leaving the mark
1633: behind.  A prefix argument acts as a repeat count.  With a negative
1634: argument, move forward.
1635: 
1636: @item C-c C-n
1637: @kindex C-c C-n @r{(C mode)}
1638: @findex c-forward-conditional
1639: Move point forward across a preprocessor conditional, leaving the mark
1640: behind.  A prefix argument acts as a repeat count.  With a negative
1641: argument, move backward.
1642: 
1643: @item M-a
1644: @kindex M-a (C mode)
1645: @findex c-beginning-of-statement
1646: Move point to the beginning of the innermost C statement
1647: (@code{c-beginning-of-statement}).  If point is already at the beginning
1648: of a statement, move to the beginning of the preceding statement.  With
1649: prefix argument @var{n}, move back @var{n} @minus{} 1 statements.
1650: 
1651: In comments or in strings which span more than one line, this command
1652: moves by sentences instead of statements.
1653: 
1654: @item M-e
1655: @kindex M-e (C mode)
1656: @findex c-end-of-statement
1657: Move point to the end of the innermost C statement or sentence; like
1658: @kbd{M-a} except that it moves in the other direction
1659: (@code{c-end-of-statement}).

1662: @node Electric C
1663: @subsection Electric C Characters
1664: 
1665:   In C mode and related modes, certain printing characters are
1666: @dfn{electric}---in addition to inserting themselves, they also
1667: reindent the current line, and optionally also insert newlines.  The
1668: electric characters are @kbd{@{}, @kbd{@}}, @kbd{:}, @kbd{#},
1669: @kbd{;}, @kbd{,}, @kbd{<}, @kbd{>}, @kbd{/}, @kbd{*}, @kbd{(}, and
1670: @kbd{)}.
1671: 
1672:   You might find electric indentation inconvenient if you are editing
1673: chaotically indented code.  If you are new to CC Mode, you might find
1674: it disconcerting.  You can toggle electric action with the command
1675: @kbd{C-c C-l}; when it is enabled, @samp{/l} appears in the mode line
1676: after the mode name:
1677: 
1678: @table @kbd
1679: @item C-c C-l
1680: @kindex C-c C-l @r{(C mode)}
1681: @findex c-toggle-electric-state
1682: Toggle electric action (@code{c-toggle-electric-state}).  With a
1683: positive prefix argument, this command enables electric action, with a
1684: negative one it disables it.
1685: @end table
1686: 
1687:   Electric characters insert newlines only when, in addition to the
1688: electric state, the @dfn{auto-newline} feature is enabled (indicated
1689: by @samp{/la} in the mode line after the mode name).  You can turn
1690: this feature on or off with the command @kbd{C-c C-a}:
1691: 
1692: @table @kbd
1693: @item C-c C-a
1694: @kindex C-c C-a @r{(C mode)}
1695: @findex c-toggle-auto-newline
1696: Toggle the auto-newline feature (@code{c-toggle-auto-newline}).  With a
1697: prefix argument, this command turns the auto-newline feature on if the
1698: argument is positive, and off if it is negative.
1699: @end table
1700: 
1701:   Usually the CC Mode style configures the exact circumstances in
1702: which Emacs inserts auto-newlines.  You can also configure this
1703: directly.  @xref{Custom Auto-newlines,,, ccmode, The CC Mode Manual}.

1705: @node Hungry Delete
1706: @subsection Hungry Delete Feature in C
1707: @cindex hungry deletion (C Mode)
1708: 
1709:   If you want to delete an entire block of whitespace at point, you
1710: can use @dfn{hungry deletion}.  This deletes all the contiguous
1711: whitespace either before point or after point in a single operation.
1712: @dfn{Whitespace} here includes tabs and newlines, but not comments or
1713: preprocessor commands.
1714: 
1715: @table @kbd
1716: @item C-c C-@key{DEL}
1717: @itemx C-c @key{DEL}
1718: @findex c-hungry-delete-backwards
1719: @kindex C-c C-DEL (C Mode)
1720: @kindex C-c DEL (C Mode)
1721: Delete the entire block of whitespace preceding point (@code{c-hungry-delete-backwards}).
1722: 
1723: @item C-c C-d
1724: @itemx C-c C-@key{Delete}
1725: @itemx C-c @key{Delete}
1726: @findex c-hungry-delete-forward
1727: @kindex C-c C-d (C Mode)
1728: @kindex C-c C-Delete (C Mode)
1729: @kindex C-c Delete (C Mode)
1730: Delete the entire block of whitespace after point (@code{c-hungry-delete-forward}).
1731: @end table
1732: 
1733:   As an alternative to the above commands, you can enable @dfn{hungry
1734: delete mode}.  When this feature is enabled (indicated by @samp{/h} in
1735: the mode line after the mode name), a single @key{DEL} deletes all
1736: preceding whitespace, not just one space, and a single @kbd{C-d}
1737: (but @emph{not} plain @key{Delete}) deletes all following whitespace.
1738: 
1739: @table @kbd
1740: @item M-x c-toggle-hungry-state
1741: @findex c-toggle-hungry-state
1742: Toggle the hungry-delete feature
1743: (@code{c-toggle-hungry-state}).  With a prefix argument,
1744: this command turns the hungry-delete feature on if the argument is
1745: positive, and off if it is negative.
1746: @end table
1747: 
1748: @vindex c-hungry-delete-key
1749:    The variable @code{c-hungry-delete-key} controls whether the
1750: hungry-delete feature is enabled.

1756: @item M-x c-context-line-break
1757: @findex c-context-line-break
1758: This command inserts a line break and indents the new line in a manner
