0010: @anchor{Glossary---Abbrev}
0011: @item Abbrev
0012: An abbrev is a text string that expands into a different text string
0013: when present in the buffer.  For example, you might define a few letters
0014: as an abbrev for a long phrase that you want to insert frequently.
0015: @xref{Abbrevs}.
0016: 
0017: @item Aborting
0018: Aborting means getting out of a recursive edit (q.v.).  The
0019: commands @kbd{C-]} and @kbd{M-x top-level} are used for this.
0020: @xref{Quitting}.
0021: 
0022: @item Active Region
0023: Setting the mark (q.v.@:) at a position in the text also activates it.
0024: When the mark is active, we call the region an active region.
0025: @xref{Mark}.
0026: 
0027: @item Alt
0028: Alt is the name of a modifier bit that a keyboard input character may
0029: have.  To make a character Alt, type it while holding down the @key{Alt}
0030: key.  Such characters are given names that start with @kbd{@key{Alt}-}
0031: (usually written @kbd{A-} for short).  (Note that many terminals have a
0032: key labeled @key{Alt} that is really a @key{META} key.)  @xref{User
0033: Input, Alt}.
0034: 
0035: @item Argument
0036: @xref{Glossary---Numeric Argument}.
0037: 
0038: @item @acronym{ASCII} character
0039: An @acronym{ASCII} character is either an @acronym{ASCII} control
0040: character or an @acronym{ASCII} printing character.  @xref{User Input}.
0041: 
0042: @item @acronym{ASCII} control character
0043: An @acronym{ASCII} control character is the Control version of an upper-case
0044: letter, or the Control version of one of the characters @samp{@@[\]^_?}.
0045: 
0046: @item @acronym{ASCII} printing character
0047: @acronym{ASCII} letters, digits, space, and the following punctuation
0048: characters: @samp{!@@#$%^&*()_-+=|\~`@{@}[]:;"'<>,.?/}.
0049: 
0050: @item Auto Fill Mode
0051: Auto Fill mode is a minor mode (q.v.@:) in which text that you insert is
0052: automatically broken into lines of a given maximum width.
0053: @xref{Filling}.
0054: 
0055: @item Auto Saving
0056: Auto saving is the practice of periodically saving the contents of an
0057: Emacs buffer in a specially-named file, so that the information will
0058: be preserved if the buffer is lost due to a system error or user error.
0059: @xref{Auto Save}.
0060: 
0061: @item Autoloading
0062: Emacs can automatically load Lisp libraries when a Lisp program requests a
0063: function from those libraries.  This is called ``autoloading''.
0064: @xref{Lisp Libraries}.
0065: 
0066: @item Backtrace
0067: A backtrace is a trace of a series of function calls showing how a
0068: program arrived at a certain point.  It is used mainly for finding and
0069: correcting bugs (q.v.).  Emacs can display a backtrace when it signals
0070: an error or when you type @kbd{C-g} (@pxref{Glossary---Quitting}).
0071: @xref{Checklist}.
0072: 
0073: @item Backup File
0074: A backup file records the contents that a file had before the current
0075: editing session.  Emacs makes backup files automatically to help you
0076: track down or cancel changes you later regret making.  @xref{Backup}.
0077: 
0078: @item Balancing Parentheses
0079: Emacs can balance parentheses (or other matching delimiters) either
0080: manually or automatically.  You do manual balancing with the commands
0081: to move over parenthetical groupings (@pxref{Moving by Parens}).
0082: Automatic balancing works by blinking or highlighting the delimiter
0083: that matches the one you just inserted, or inserting the matching
0084: delimiter for you (@pxref{Matching,,Matching Parens}).
0085:

0087: @item Balanced Expressions
0088: A balanced expression is a syntactically recognizable expression, such
0089: as a symbol, number, string constant, block, or parenthesized expression
0090: in C@.  @xref{Expressions,Balanced Expressions}.

0095: @item Base Buffer
0096: A base buffer is a buffer whose text is shared by an indirect buffer

0099: @item Bidirectional Text
0100: Some human languages, such as English, are written from left to right.
0101: Others, such as Arabic, are written from right to left.  Emacs
0102: supports both of these forms, as well as any mixture of them---this
0103: is ``bidirectional text''.  @xref{Bidirectional Editing}.
0104: 
0105: @item Bind
0106: To bind a key sequence means to give it a binding (q.v.).
0107: @xref{Rebinding}.
0108: 
0109: @anchor{Glossary---Binding}
0110: @item Binding
0111: A key sequence gets its meaning in Emacs by having a binding, which is a
0112: command (q.v.), a Lisp function that is run when you type that
0113: sequence.  @xref{Commands,Binding}.  Customization often involves
0114: rebinding a character to a different command function.  The bindings of
0115: all key sequences are recorded in the keymaps (q.v.).  @xref{Keymaps}.
0116: 
0117: @item Blank Lines
0118: Blank lines are lines that contain only whitespace.  Emacs has several
0119: commands for operating on the blank lines in the buffer.  @xref{Blank Lines}.
0120: 
0121: @item Bookmark
0122: Bookmarks are akin to registers (q.v.@:) in that they record positions
0123: in buffers to which you can return later.  Unlike registers, bookmarks
0124: persist between Emacs sessions.  @xref{Bookmarks}.
0125: 
0126: @item Border
0127: A border is a thin space along the edge of the frame, used just for
0128: spacing, not for displaying anything.  An Emacs frame has an ordinary
0129: external border, outside of everything including the menu bar, plus an
0130: internal border that surrounds the text windows, their scroll bars
0131: and fringes, and separates them from the menu bar and tool bar.  You
0132: can customize both borders with options and resources (@pxref{Borders
0133: X}).  Borders are not the same as fringes (q.v.).
0134: 
0135: @item Buffer
0136: The buffer is the basic editing unit; one buffer corresponds to one text
0137: being edited.  You normally have several buffers, but at any time you are
0138: editing only one, the current buffer, though several can be visible
0139: when you are using multiple windows or frames (q.v.).  Most buffers
0140: are visiting (q.v.@:) some file.  @xref{Buffers}.
0141: 
0142: @item Buffer Selection History
0143: Emacs keeps a buffer selection history that records how recently each
0144: Emacs buffer has been selected.  This is used for choosing a buffer to
0145: select.  @xref{Buffers}.

0147: @item Bug
0148: A bug is an incorrect or unreasonable behavior of a program, or
0149: inaccurate or confusing documentation.  Emacs developers treat bug
0150: reports, both in Emacs code and its documentation, very seriously and
0151: ask you to report any bugs you find.  @xref{Bugs}.

0155: right away when you press down on a mouse button.  @xref{Mouse Buttons}.

0174: @item Case Conversion
0175: Case conversion means changing text from upper case to lower case or
0176: vice versa.  @xref{Case}.

0178: @item Case Folding
0179: Case folding means ignoring the differences between case variants of
0180: the same letter: upper-case, lower-case, and title-case.  Emacs
0181: performs case folding by default in text search.  @xref{Lax Search}.

0183: @item Character
0184: Characters form the contents of an Emacs buffer.  Also, key sequences
0185: (q.v.@:) are usually made up of characters (though they may include
0186: other input events as well).  @xref{User Input}.
0187: 
0188: @item Character Folding
0189: Character folding means ignoring differences between similarly looking
0190: characters, such as between @code{a}, and @code{@:a} and @code{@'a}.
0191: Emacs performs character folding by default in text search.  @xref{Lax
0192: Search}.

0209: @item Clipboard
0210: A clipboard is a buffer provided by the window system for transferring
0211: text between applications.  On the X Window System, the clipboard is
0212: provided in addition to the primary selection (q.v.); on MS-Windows and Mac,
0213: the clipboard is used @emph{instead} of the primary selection.

0216: @item Coding System
0217: A coding system is an encoding for representing text characters in a
0218: file or in a stream of information.  Emacs has the ability to convert
0219: text to or from a variety of coding systems when reading or writing it.
0220: @xref{Coding Systems}.

0222: @item Command
0223: A command is a Lisp function specially defined to be able to serve as a
0224: key binding in Emacs.  When you type a key sequence (q.v.), its
0225: binding (q.v.@:) is looked up in the relevant keymaps (q.v.@:) to find
0226: the command to run.  @xref{Commands}.

0231: @item Command Name
0232: A command name is the name of a Lisp symbol that is a command
0233: (@pxref{Commands}).  You can invoke any command by its name using
0234: @kbd{M-x} (@pxref{M-x,M-x,Running Commands by Name}).

0236: @item Comment
0237: A comment is text in a program which is intended only for humans reading
0238: the program, and which is specially marked so that it will be ignored
0239: when the program is loaded or compiled.  Emacs offers special commands
0240: for creating, aligning and killing comments.  @xref{Comments}.

0242: @item Common Lisp
0243: Common Lisp is a dialect of Lisp (q.v.@:) much larger and more powerful
0244: than Emacs Lisp.  Emacs provides a subset of Common Lisp in the CL
0245: package.  @xref{Top, Common Lisp, Overview, cl, Common Lisp Extensions}.

0247: @anchor{Glossary---Compilation}
0248: @item Compilation
0249: Compilation is the process of creating an executable program from source
0250: code.  Emacs has commands for compiling files of Emacs Lisp code
0251: (@pxref{Byte Compilation,,, elisp, the Emacs Lisp
0252: Reference Manual}) and programs in C and other languages
0253: (@pxref{Compilation}).

0255: @item Complete Key
0256: A complete key is a key sequence that fully specifies one action to be
0257: performed by Emacs.  For example, @kbd{X} and @kbd{C-f} and @kbd{C-x m}
0258: are complete keys.  Complete keys derive their meanings from being bound
0259: (q.v.@:) to commands (q.v.).  Thus, @kbd{X} is conventionally bound to
0260: a command to insert @samp{X} in the buffer; @kbd{C-x m} is
0261: conventionally bound to a command to begin composing a mail message.
0262: @xref{Keys}.
0263: 
0264: @item Completion
0265: Completion is what Emacs does when it automatically expands an
0266: abbreviation for a name into the entire name.  Completion is done for
0267: minibuffer (q.v.@:) arguments when the set of possible valid inputs
0268: is known; for example, on command names, buffer names, and
0269: file names.  Completion usually occurs when @key{TAB}, @key{SPC} or
0270: @key{RET} is typed.  @xref{Completion}.
0271: 
0272: @anchor{Glossary---Continuation Line}
0273: @item Continuation Line
0274: When a line of text is longer than the width of the window, it
0275: normally (but see @ref{Glossary---Truncation}) takes up more than one
0276: screen line when displayed.  We say that the text line is continued, and all
0277: screen lines used for it after the first are called continuation
0278: lines.  @xref{Continuation Lines}.  A related Emacs feature is
0279: filling (q.v.).
0280: 
0281: @item Control Character
0282: A control character is a character that you type by holding down the
0283: @key{Ctrl} key.  Some control characters also have their own keys, so
0284: that you can type them without using @key{Ctrl}.  For example,
0285: @key{RET}, @key{TAB}, @key{ESC} and @key{DEL} are all control
0286: characters.  @xref{User Input}.
0287: 
0288: @item Copyleft
0289: A copyleft is a notice giving the public legal permission to
0290: redistribute and modify a program or other work of art, but requiring
0291: modified versions to carry similar permission.  Copyright is normally
0292: used to keep users divided and helpless; with copyleft we turn that
0293: around to empower users and encourage them to cooperate.
0294: 
0295: The particular form of copyleft used by the GNU project is called the
0296: GNU General Public License.  @xref{Copying}.
0297: 
0298: @item @key{Ctrl}
0299: The @key{Ctrl} or control key is what you hold down
0300: in order to enter a control character (q.v.).  @xref{Glossary---C-}.
0301: 
0302: @item Current Buffer
0303: The current buffer in Emacs is the Emacs buffer on which most editing
0304: commands operate.  You can select any Emacs buffer as the current one.
0305: @xref{Buffers}.
0306: 
0307: @item Current Line
0308: The current line is the line that point is on (@pxref{Point}).
0309: 
0310: @item Current Paragraph
0311: The current paragraph is the paragraph that point is in.  If point is
0312: between two paragraphs, the current paragraph is the one that follows
0313: point.  @xref{Paragraphs}.
0314: 
0315: @item Current Defun
0316: The current defun is the defun (q.v.@:) that point is in.  If point is
0317: between defuns, the current defun is the one that follows point.
0318: @xref{Defuns}.
0319: 
0320: @item Cursor
0321: The cursor is the rectangle on the screen which indicates the position
0322: (called point; q.v.@:) at which insertion and deletion takes place.
0323: The cursor is on or under the character that follows point.  Often
0324: people speak of ``the cursor'' when, strictly speaking, they mean
0325: ``point''.  @xref{Point,Cursor}.

0328: Customization is making minor changes in the way Emacs works, to
0329: reflect your preferences or needs.  It is often done by setting
0330: variables (@pxref{Variables}) or faces (@pxref{Face Customization}),
0331: or by rebinding key sequences (@pxref{Keymaps}).

0337: @anchor{Glossary---Daemon}
0338: @item Daemon
0339: A daemon is a standard term for a system-level process that runs in the
0340: background.  Daemons are often started when the system first starts up.
0341: When Emacs runs in daemon-mode, it does not

0345: @item Default Argument
0346: The default for an argument is the value that will be assumed if you
0347: do not specify one.  When the minibuffer is used to read an argument,
0348: the default argument is used if you just type @key{RET}.
0349: @xref{Minibuffer}.
0350: 
0351: @anchor{Glossary---Default}
0352: @item Default
0353: A default is the value that is used for a certain purpose when
0354: you do not explicitly specify a value to use.
0355: 
0356: @item Default Directory
0357: When you specify a file name that does not start with @samp{/} or @samp{~},
0358: it is interpreted relative to the current buffer's default directory.
0359: (On MS systems, file names that start with a drive letter
0360: @samp{@var{x}:} are treated as absolute, not relative.)
0361: @xref{Minibuffer File,Default Directory}.

0363: @item Defun
0364: A defun is a major definition at the top level in a program.  The name
0365: ``defun'' comes from Lisp, where most such definitions use the construct
0366: @code{defun}.  @xref{Defuns}.
0367: 
0368: @item @key{DEL}
0369: @key{DEL} is a character that runs the command to delete one character
0370: of text before the cursor.  It is typically either the @key{Delete}
0371: key or the @key{BACKSPACE} key, whichever one is easy to type.
0372: @xref{Erasing,DEL}.
0373: 
0374: @item Deletion
0375: Deletion means erasing text without copying it into the kill ring
0376: (q.v.).  The alternative is killing (q.v.).  @xref{Killing,Deletion}.
0377: 
0378: @anchor{Glossary---Deletion of Files}
0379: @item Deletion of Files
0380: Deleting a file means erasing it from the file system.
0381: (Note that some systems use the concept of a trash can, or recycle
0382: bin, to allow you to undelete files.)
0383: @xref{Misc File Ops,Misc File Ops,Miscellaneous File Operations}.
0384: 
0385: @item Deletion of Messages
0386: Deleting a message (in Rmail, and other mail clients) means flagging
0387: it to be eliminated from your mail file.  Until you expunge (q.v.@:)
0388: the Rmail file, you can still undelete the messages you have deleted.
0389: @xref{Rmail Deletion}.
0390: 
0391: @item Deletion of Windows
0392: Deleting a window means eliminating it from the screen.  Other windows
0393: expand to use up the space.  The text that was in the window is not
0394: lost, and you can create a new window with the same dimensions as the
0395: old if you wish.  @xref{Windows}.

0397: @item Directory

0399: you can place individual files or subdirectories.  They are sometimes
0400: referred to as ``folders''.  @xref{Directories}.

0405: to all the files within a certain directory.  @xref{Directory

0443: @item Echo Area
0444: The echo area is the bottom line of the screen, used for echoing the

0528: @item File Name
0529: @c This is fairly tautological...
0530: A file name is a name that refers to a file.  File names may be relative
0531: or absolute; the meaning of a relative file name depends on the current
0532: directory, but an absolute file name refers to the same file regardless
0533: of which directory is current.  On GNU and Unix systems, an absolute
0534: file name starts with a slash (the root directory) or with @samp{~/} or
0535: @samp{~@var{user}/} (a home directory).  On MS-Windows/MS-DOS, an
0536: absolute file name can also start with a drive letter and a colon, e.g.,
0537: @samp{@var{d}:}.

0539: Some people use the term ``pathname'' for file names, but we do not;

1094: @item Quoting
1095: Quoting means depriving a character of its usual special significance.
1096: The most common kind of quoting in Emacs is with @kbd{C-q}.  What
1097: constitutes special significance depends on the context and on
1098: convention.  For example, an ordinary character as an Emacs command
1099: inserts itself; so in this context, a special character is any character
1100: that does not normally insert itself (such as @key{DEL}, for example),
1101: and quoting it makes it insert itself as if it were not special.  Not
1102: all contexts allow quoting.  @xref{Inserting Text,Quoting}.

1297: @item String
1298: A string is a kind of Lisp data object that contains a sequence of
1299: characters.  Many Emacs variables are intended to have strings as
1300: values.  The Lisp syntax for a string consists of the characters in the
1301: string with a @samp{"} before and another @samp{"} after.  A @samp{"}
1302: that is part of the string must be written as @samp{\"} and a @samp{\}
1303: that is part of the string must be written as @samp{\\}.  All other
1304: characters, including newline, can be included just by writing them
1305: inside the string; however, backslash sequences as in C, such as
1306: @samp{\n} for newline or @samp{\241} using an octal character code, are
1307: allowed as well.
