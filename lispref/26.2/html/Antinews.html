Appendix A Emacs 25 Antinews

時代に逆らって生きるユーザーのために、以下はEmacsバージョン24.5へのダウングレードに関する情報です。Emacs 26.2機能の不在による結果としての偉大なる単純さをぜひ堪能してください。

A.1 Old Lisp Features in Emacs 25

  • 並行性の機能は削除されました。たとえ“概ね協調的(mostly cooperative)”と呼ばれる、同時に実行されるLispスレッドは1つだけの形式に限定されていたとはいえ、追加スレッドが存在しても正しく機能する必要があるLispプログラムにたいしてEmacsが明らかに複雑化してしまうからです。
  • ファイル属性の処理はfile-attribute-typeやfile-attribute-modification-timeのようなアクセサ関数を廃止することによって簡略化されました。常に序数値で個別の属性にアクセスして、眠りながらにして序数値を諳んじることができるのが真のLispプログラマーです。
  • 非同期DNS解決、接続、TLSストリームにたいするTLSネゴシエーション使用の削除によって、ネットワーク関連のコード初期の単純さを取り戻しました。もはやネットワーク通信を使用するLispプログラムを記述する際に、結果として生じる複雑性や興味深い競合状態を考慮する必要はありません。その直接的な帰結としてgnutls-bootの:complete-negotiationパラメーターが不要になり削除されました(これは単に非同期性の撤廃がEmacsを単純化する一例に過ぎない)。
  • 非ASCIIのURLをもつウェブサイトに安易にアクセスできないようにpuny.elライブラリーを削除しました。そのようなサイトは時を遡るしたがいますますレアになっていくので、それらのサポートに特化したライブラリーをもつのは不要な保守負担だと判断しました。
  • 外部タイムスタンプの変換をより挑戦的にするために時間変換関数current-time-string、current-time-zone、decode-time、format-time-string、set-time-zone-ruleはタイムゾーンルールとして整数オフセットを受け取らなくなります。さらにformat-time-stringは‘%q’を四半期に変換しなくなります。あなたでも簡単に行えるのでしょう。
  • フォーマット仕様の‘%2$’のようなフィールド数値が利用できなくなります。これらの使用によりコードの読み取りと理解が困難になること、そして普及したCライブラリーの同種機能がなくなる過去においては不正であろうと判断しました。
  • 行番号を表示するためのビルトイン機能が削除(Antinews in The GNU Emacs Manualを参照)されたので、独自に表示レイアウトを行うLispプログラムによるこの扱いにくい機能への対応が不要になるようにline-number-display-width、およびdisplay-line-numbers-disableプロパティにたいするサポートも削除しました。
  • regexpクラス[:blank:]のUnicode文字プロパティにたいするサポートを削除することにより正規表現が簡略化されました。その結果として当該クラスはスペースとタブだけにマッチするようになります。繰り返しますが、これは時の逆行によるUnicodeの重要性減少に合致しています。
  • 同様の理由により関数char-from-nameを削除しました。EmacsがUnicodeをサポートするかぎり(それほど長い間ではないが)、ucs-namesがリターンするUnicode文字の完全なリストへのアクセスで十分容易なはずです。
  • file-attributes、file-symlink-p、make-symbolic-linkのような引数としてファイル名を受け取る種々の関数は‘/:’でクォートされたファイル名にたいする特別なサポートを取り戻し、シンボリックリンクのターゲット内の‘~’はあなたが期待する通り(すなわちホームディレクトリー)に解釈されるようになりました。これらの関数の呼び出しが対話的か否かによる操作の差異による混乱は削除されました。
  • ファイルの作成やリネームを行ういくつかの関数は対象がディレクトリー名でないように見える場合にも、それらがディレクトリなら特別に処理するようになります。たとえばBがディレクトリーなら(rename-file "A" "B")はAをBにリネームしません。これはファイルの処理をより冒険的にするためです。
  • Emacsのメモリーマネージャーにより負荷を与えてEmacs改善のためのテストを行うために、format関数がより多くの場面で新たな文字列をリターンするようになりました。
  • 関数assocは3つ目のオプション引数を削除することにより簡略化されました。比較には常にequalを使用するようになりました。同様にalist-getは比較にassq、map-getとmap-putはeqlを常に使用します。
  • 数値比較と関数format、make-hash-table、min、max、logbは結果を予測可能性を減少させるために、内部で時折値を丸めるようになりました。
  • 関数ffloor、fceiling、ftruncate、froundが整数の引数を受け取るようになりました。それとは逆にdecode-charのような浮動小数点整数を受け取る関数は、整数の引数を受け取らなくなります。いずれの場合でも結果は時として愉快なほど無意味です。
  • GnuTLS暗号化関数がEmacsで利用できなくなります。このような関数にたいするニーズは低下しており、扱いにくいインターフェースが使用を困難にしていると判断しました。
  • ユーザー定義型のサポートを削除しました。またcl-defstructはrecordを使用しなくなります。これにより極めて少数の既存コードや過去のコードがrecordによって破壊される可能性を取り除きました。
  • 非推奨メッセージに煩わされることなくstring-as-unibyte、string-make-multibyte、および同種の関数を再び使用できるようになりました。これは更に遠い過去においてEmacsがマルチバイトテキストを削除するための準備です。
  • プログラマー自身の特異な手法を使用して、あるバージョン文字列が別のバージョン文字列より前かを判断することを奨励するためにstring-version-lessp関数を削除しました。
  • 関数read-colorがカラー名の表示時にそのカラーをバックグラウンドに使用しなくなりました。これはユーザーを驚かせるとともに、カラーに問題のある端末で奇妙で一貫性のない結果をもたらすと判断しました。
  • 昔には不要なのでテキスト端末の24ビットカラーのサポートをなくしました。
  • 観測可能な過去にたいするテストにおいてOSシンボルで十分であり、別APIの使用を習得するのは負担でもあるので関数file-name-case-insensitive-pを削除しました。
  • Emacs Lispハッカーににとって座って別のインタラクティブなQ&A関数を記述する必要や、それを特定のケースごとに最適化するのに優る喜びはないという事実の認識にもとづき関数read-multiple-choiceを削除しました。
  • 関数add-variable-watcherとそれに対応するデバッガコマンドdebug-on-variable-changeは削除されました。同じユースケースを網羅するには各停止ポイントで変数の値を調べれば十分なので、これらはデバッグを複雑にしてしまいます。ルールを単純化しましょう!
  • 関数mapcanがなくなりました。かわりにmapcarを使用して結果のリストを適切に処理してください。
  • lengthやmemberのような低レベルなリスト関数に循環リストを与えれば、Emacsのフリーズを引き起こすように無限ループできるようになりました。これは循環的ではない入力の際に、これらの関数を僅かに高速にするために役立つかもしれません。
  • write-region関数がファイル名ハンドラーにたいしてlockname引数を伝播しなくなりました。
  • Emacsがファイルアクセス中に別プロセスがファイルシステムを変更したことにより、file-attributes由来の奇妙な乱数値をリターンするLispプログラムを再び記述できるようになりました。これにより近い過去において興味深いアプリケーションをいくつか生み出すことができます。
  • Emacsコードでの任意のシンボリックリンクの処理をより挑戦的にするために関数file-attributes、file-symlink-p、make-symbolic-linkがローカルのシンボリックリンクのターゲットをあるケースにおいて暗黙に変更するようになりました。
  • エラーfile-missingは削除されました。そのようなエラーの処理はかわりにfile-errorカテゴリーにまとめられます。
  • 関数delete-directory再帰的に処理を行い、かつがdelete-directoryが取得する前に別のプロセスがディレクトリーを削除した場合にはエラーをシグナルするようになりました。
  • 入力メソッドのdutchはトルコ語のサポートも試みるようになりました(ただし不正確)。さらに‘IJ’や‘ij’を放置せずに特殊文字への変換も行います。
  • non-breakingハイフンおよびクォートの近似は独自のフェイスのかわりに、単にescape-glyphフェイスで表示されるようになります。これはより単純であり、ユーザーがこれらの文字を含むテキストを閲覧時に解決すべき楽しいパズルを提供するでしょう。
  • ユーザーオプションelectric-quote-context-sensitiveと変数electric-quote-inhibit-functionsによりelectric quotingがより単純になり、何か間違ったことを行えるようになります。
  • ユーザーオプションtext-quoting-styleは削除されて単なる変数になりました。
  • 関数file-name-quote、file-name-unquote、file-name-quoted-pを削除しました。ファイル名がクォート済みかどうかをチェックするコードの記述は容易であり、二重にクォートされたファイル名でも適切に記述されたLispコードでは問題が起こるはずがありません。
  • z-group、min-width、parent-frame、delete-before等のフレームパラメーターを削除しました。ウィンドウマネージャーの機能が次第に低下していかないことは明らかであり、Emacsがウィンドウマネージャーを置き換えるべきではありません。
  • モードラインとヘッダーラインのフォーマットはバッファーのみにもとづいてカスタマイズ可能であると判断してウィンドウパラメーターmode-line-formatとheader-line-formatを削除しました。
  • Emacsが安全ではないチャンネルを介したPOP3メールの取得を行う限定版のmovemail置換を通常はビルドするようになり、configure時のオプション --with-mailutilsは削除されました。これはセキュリティーが重要ではない場合のEmacsのセットアップを簡略化します。
  • configure時のオプション--enable-gcc-warnings=warn-onlyは削除されたので、ビルド時の警告は常に致命的になります。
  • configure時のオプション--disable-build-detailsを削除しました。この方法によりEmacsのビルドが一意かつ再現不能になりました。
  • ビルド番号を個別にemacs-build-numberに格納するかわりに変数emacs-versionに含めるようになりました。
  • EmacsをIRIXに移植しました。
  • いくつかのオプションと変数を削除してEmacsの単純化、および潜在的な信頼性を少し低下しました。これには--module-assertionsオプション、attempt-stack-overflow-recovery変数、attempt-orderly-shutdown-on-fatal-signal変数が含まれます。
  • 簡略化にたいする継続要求の一環として、他の多くの関数と変数が排除されました。
