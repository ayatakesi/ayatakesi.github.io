   156    'assume (COND)'.  COND should be free of side effects, as it may or
   157    may not be evaluated.
   158 
   159    'eassert (COND)' checks COND at runtime if ENABLE_CHECKING is
   160    defined and suppress_checking is false, and does nothing otherwise.
   161    Emacs dies if COND is checked and is false.  The suppress_checking
   162    variable is initialized to 0 in alloc.c.  Set it to 1 using a
   163    debugger to temporarily disable aborting on detected internal
   164    inconsistencies or error conditions.

   166    In some cases, a good compiler may be able to optimize away the
   167    eassert macro even if ENABLE_CHECKING is true, e.g., if XSTRING (x)

   170    test redundant.

   173    assume that COND is true afterwards, regardless of whether runtime

   176    for COND to call external functions or access volatile storage.  */

   553    instructions.
   554 
   555    First, there are already a couple of Lisp types that can be used if
   556    your new type does not need to be exposed to Lisp programs nor
   557    displayed to users.  These are Lisp_Misc_Ptr and PVEC_OTHER,

   560    might be to some low-level C object that contains auxiliary

   567    unsuitable for Lisp objects that can be exposed to users.

   575    collector frees them.

   979 /* Header of vector-like objects.  This documents the layout constraints on
   980    vectors and pseudovectors (objects of PVEC_xxx subtype).  It also prevents
   981    compilers from being fooled by Emacs's type punning: XSETPSEUDOVECTOR

   989   {
   990     /* The main member contains various pieces of information:
   991        - The MSB (ARRAY_MARK_FLAG) holds the gcmarkbit.
   992        - The next bit (PSEUDOVECTOR_FLAG) indicates whether this is a plain
